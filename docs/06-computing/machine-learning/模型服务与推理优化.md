## 一、核心概念

### 1.1 定义与原理

**模型服务(Model Serving)** 是将训练好的机器学习模型部署为可访问的API服务，供生产环境调用。

**推理优化(Inference Optimization)** 是通过技术手段降低模型推理延迟、提高吞吐量的过程。

**核心原理**：

- **动态批处理(Dynamic Batching)**：合并多个请求批量推理
- **模型量化(Quantization)**：降低数值精度减少计算量
- **算子融合(Operator Fusion)**：合并相邻算子减少内存访问
- **内存优化(Memory Optimization)**：高效管理激活值和权重

### 1.2 关键特性

| 特性 | 描述 | 收益 |
|------|------|------|
| **动态批处理** | 自动合并请求批次 | 吞吐量提升3-10x |
| **模型并行** | 多GPU分割模型 | 支持超大模型 |
| **多框架支持** | 统一服务多种模型格式 | 降低运维复杂度 |
| **GPU加速** | TensorRT/ONNX Runtime优化 | 延迟降低10-100x |
| **模型热更新** | 不停机更新模型版本 | 零中断部署 |
| **A/B测试** | 流量分割多版本 | 安全验证新模型 |

### 1.3 适用场景

| 场景 | 适用性 | 说明 |
|------|--------|------|
| 实时推荐系统 | ⭐⭐⭐⭐⭐ | 毫秒级延迟要求 |
| 自动驾驶感知 | ⭐⭐⭐⭐⭐ | 严格时延约束 |
| 语音识别服务 | ⭐⭐⭐⭐⭐ | 流式推理需求 |
| 大语言模型API | ⭐⭐⭐⭐⭐ | 高并发、大模型 |
| 图像分类服务 | ⭐⭐⭐⭐ | 可接受百毫秒延迟 |
| 离线批处理 | ⭐⭐ | 不需要实时优化 |

**自定义Handler**：

```python
from ts.torch_handler.base_handler import BaseHandler
import torch
import json

class CustomHandler(BaseHandler):
    def __init__(self):
        super().__init__()
        self.initialized = False

    def initialize(self, context):
        # 加载模型
        properties = context.system_properties
        model_dir = properties.get("model_dir")
        self.device = torch.device(
            "cuda:" + str(properties.get("gpu_id"))
            if torch.cuda.is_available() else "cpu"
        )

        # 加载模型权重
        model_path = f"{model_dir}/model.pth"
        self.model = torch.load(model_path, map_location=self.device)
        self.model.eval()

        # 加载标签映射
        with open(f"{model_dir}/labels.json") as f:
            self.labels = json.load(f)

        self.initialized = True

    def preprocess(self, data):
        # 输入预处理
        images = []
        for row in data:
            image = row.get("data") or row.get("body")
            images.append(self.transform(image))
        return torch.stack(images).to(self.device)

    def inference(self, data):
        with torch.no_grad():
            outputs = self.model(data)
            probs = torch.nn.functional.softmax(outputs, dim=1)
        return probs

    def postprocess(self, inference_output):
        # 后处理
        results = []
        for prob in inference_output:
            top5 = torch.topk(prob, 5)
            results.append({
                self.labels[str(i)]: float(p)
                for i, p in zip(top5.indices, top5.values)
            })
        return results
```

### 4.2 最佳实践

1. **模型预热**

   ```python
   # 在部署前预热模型缓存
   def warm_up(model, input_shape, iterations=100):
       dummy_input = torch.randn(*input_shape)
       for _ in range(iterations):
           _ = model(dummy_input)
   ```

2. **动态批处理调优**

   ```protobuf
   # 根据模型特性和流量模式调整
   dynamic_batching {
       # 计算密集型模型：较大批次
       preferred_batch_size: [ 8, 16, 32 ]

       # 延迟敏感：较小等待时间
       max_queue_delay_microseconds: 50

       # 设置最大批次保护
       max_queue_size: 100
   }
   ```

3. **模型缓存策略**

   ```python
   # 使用LRU缓存频繁请求
   from functools import lru_cache

   @lru_cache(maxsize=1000)
   def get_cached_inference(input_hash):
       return model.predict(input_hash)
   ```

4. **监控指标**


### 4.3 常见问题

**Q1: GPU内存不足？**
A:

- 使用模型分片（模型并行）
- 启用FP16/INT8量化
- 调整max_batch_size
- 使用梯度检查点技术

**Q2: 延迟过高？**
A:

- 启用TensorRT优化
- 调整动态批处理参数
- 使用CUDA Graphs减少CPU开销
- 考虑模型剪枝或蒸馏

**Q3: 冷启动慢？**
A:

- 配置minReplicas保持预热实例
- 使用模型序列化缓存（.plan文件）
- 优化模型加载顺序

**Q4: 版本切换导致服务中断？**
A:

```protobuf
# 使用版本策略实现平滑过渡
version_policy {
  all {}
}
# 配合负载均衡器逐步切换流量
```

## 五、形式化分析

### 5.1 批处理延迟模型

**延迟组成**：

```
总延迟 = 队列等待时间 + 批处理填充时间 + 推理时间 + 后处理时间

其中:
- 队列等待时间: ~指数分布，与负载相关
- 批处理填充时间: 由preferred_batch_size和max_queue_delay决定
- 推理时间: f(batch_size)，通常次线性增长

最优批大小 B* = argmin_B [L(B)]
其中 L(B) = λ/(B·μ) + B·t_inference
```

### 5.2 吞吐量优化

**吞吐量公式**：

```
吞吐量 = batch_size / (t_preprocess + t_inference + t_postprocess)

优化策略:
1. 异步预处理/后处理
2. 流水线并行
3. 增大batch_size（权衡延迟）
```
