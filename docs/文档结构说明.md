## 📋 文档组织原则

本项目文档按照**语义层次结构**组织，确保：

1. **主题聚焦**：所有文档围绕"分布式计算工作流系统"核心主题
2. **层次清晰**：按照从基础理论到实践应用的逻辑层次组织
3. **便于维护**：结构清晰，便于查找和更新
4. **语义明确**：目录名称和文件组织反映内容的语义关系

## 📝 文档命名规范

### 目录命名

- 使用数字前缀（00-09）表示语义层次
- 使用英文大写字母和连字符（如 `02-THEORY`）
- 子目录使用小写字母和连字符（如 `formal-verification`）

### 文件命名

- 使用中文描述性名称
- 专题文档使用 `[主题]专题文档.md` 格式
- 指南文档使用 `[主题]指南.md` 格式
- 报告文档使用 `[主题]报告.md` 格式

## 🔄 维护指南

### 添加新文档

1. 确定文档所属的语义层次
2. 放置在对应的层级目录中
3. 更新本说明文档
4. 更新 `00-INDEX/` 中的索引

### 移动文档

1. 检查文档的语义归属
2. 移动到正确的层级目录
3. 更新所有引用链接
4. 更新索引文档

### 归档文档

1. 确定文档是否需要归档
2. 移动到 `archive/` 对应目录
3. 更新归档清单
4. 在原始位置保留引用链接
