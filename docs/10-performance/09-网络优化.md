# 网络优化：TCP调优与零拷贝

## 核心概念

### 1. TCP协议栈性能

TCP是互联网的核心协议，理解其性能特性对高并发系统至关重要。

**TCP连接生命周期**：
```
┌─────────────────────────────────────────────────────────────┐
│                   TCP连接状态机                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   CLOSED                                                    │
│      │                                                      │
│      ▼ 发送SYN                                              │
│   SYN_SENT ───────────→ ESTABLISHED ←────────── SYN_RCVD   │
│      │                      │                      ▲        │
│      │                      │                      │        │
│      │         数据传输      │                      │        │
│      │                      ▼                      │        │
│      │              ┌──────────────┐               │        │
│      │              │ CLOSE_WAIT   │ (被动关闭)    │        │
│      │              │   /          │               │        │
│      │              │ FIN_WAIT_1   │ (主动关闭)    │        │
│      │              │      │       │               │        │
│      │              │      ▼       │               │        │
│      │              │ FIN_WAIT_2 ──┼──→ TIME_WAIT  │        │
│      │              │      │       │      │        │        │
│      │              │      ▼       │      │        │        │
│      │              │   CLOSING    │      │        │        │
│      │              │      │       │      │        │        │
│      │              └──────┼───────┘      │        │        │
│      │                     │              │        │        │
│      └─────────────────────┴──────────────┘        │        │
│                            │                       │        │
│                            ▼                       │        │
│                         CLOSED ←───────────────────┘        │
│                            (2MSL后)                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**TCP关键性能参数**：
```
┌─────────────────────────────────────────────────────────────┐
│                  TCP性能参数详解                           │
├──────────────┬──────────────────────────────────────────────┤
│   窗口大小    │  决定单连接吞吐量                           │
│              │  最大吞吐 = 窗口大小 / RTT                   │
│              │  1GBps * 100ms RTT = 需要12.5MB窗口          │
├──────────────┼──────────────────────────────────────────────┤
│   拥塞窗口    │  控制发送速率，防止网络拥塞                 │
│              │  慢启动、拥塞避免、快速恢复                  │
├──────────────┼──────────────────────────────────────────────┤
│   Nagle算法   │  小数据包合并，提高吞吐                     │
│              │  延迟敏感应用应禁用                          │
├──────────────┼──────────────────────────────────────────────┤
│   延迟确认    │  ACK延迟发送，减少ACK包数量                 │
│              │  可能增加响应延迟                            │
├──────────────┼──────────────────────────────────────────────┤
│   TIME_WAIT   │  确保最后的ACK到达，防止旧连接干扰          │
│              │  高并发短连接场景需优化                        │
└──────────────┴──────────────────────────────────────────────┘
```

### 2. 零拷贝技术

零拷贝技术减少数据在用户态和内核态之间的拷贝次数，显著提升IO性能。

**传统数据传输**：
```
磁盘 → 内核缓冲区 → 用户缓冲区 → Socket缓冲区 → 网卡
   1        2           3             4            5

拷贝次数：4次（1→2, 2→3, 3→4, 4→5）
上下文切换：4次（用户↔内核）
```

**sendfile零拷贝**：
```
磁盘 → 内核缓冲区 ──────────────────→ 网卡
               ↓（描述符传递）
           Socket缓冲区（仅元数据）

拷贝次数：2次（DMA完成，CPU不参与）
上下文切换：2次

sendfile系统调用流程：
1. 用户态调用sendfile(fd_in, fd_out, offset, count)
2. 内核态DMA将数据从磁盘读到内核缓冲区
3. 内核态通过socket缓冲区描述符直接发送
4. DMA将数据从内核缓冲区传到网卡
```

**mmap零拷贝**：
```
磁盘 → 内核缓冲区（映射到用户空间）
            ↓（直接访问，无需拷贝）
        用户空间（虚拟地址映射）
            ↓（修改后）
        内核自动同步到磁盘

适用场景：文件读写、共享内存
```

### 3. 网络模型演进

**IO模型对比**：

| 模型 | 描述 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|----------|
| 阻塞IO | 同步等待 | 简单 | 性能差 | 低并发 |
| 非阻塞IO | 轮询检查 | 非阻塞 | CPU浪费 | 不适用 |
| IO多路复用 | select/poll/epoll | 高效 | 编程复杂 | 高并发 |
| 信号驱动IO | 事件通知 | 异步感 | 不常用 | 特殊场景 |
| 异步IO | aio | 真异步 | 实现复杂 | 高性能IO |

**epoll工作原理**：
```
┌─────────────────────────────────────────────────────────────┐
│                    epoll事件驱动模型                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   1. epoll_create() 创建epoll实例                          │
│                                                             │
│   2. epoll_ctl() 注册感兴趣的事件                          │
│      - EPOLLIN (可读)                                       │
│      - EPOLLOUT (可写)                                      │
│      - EPOLLET (边缘触发)                                   │
│      - EPOLLONESHOT (单次触发)                              │
│                                                             │
│   3. epoll_wait() 等待事件                                 │
│      - 返回就绪的文件描述符列表                             │
│      - O(1)复杂度，与连接数无关                             │
│                                                             │
│   内核实现：                                                │
│   - 红黑树：存储所有监听的fd                                │
│   - 就绪链表：存储就绪的fd                                  │
│   - 回调机制：事件发生自动加入就绪链表                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**3. 应用层TCP优化（Go示例）**：
```go
package main

import (
    "net"
    "syscall"
    "time"
)

func optimizedListener() net.Listener {
    lc := net.ListenConfig{
        Control: func(network, address string, c syscall.RawConn) error {
            return c.Control(func(fd uintptr) {
                // 启用TCP Fast Open
                syscall.SetsockoptInt(int(fd), 
                    syscall.IPPROTO_TCP, syscall.TCP_FASTOPEN, 1)
                
                // 禁用Nagle算法（低延迟）
                syscall.SetsockoptInt(int(fd), 
                    syscall.IPPROTO_TCP, syscall.TCP_NODELAY, 1)
                
                // 启用Quick ACK
                syscall.SetsockoptInt(int(fd), 
                    syscall.IPPROTO_TCP, syscall.TCP_QUICKACK, 1)
                
                // 重用地址
                syscall.SetsockoptInt(int(fd), 
                    syscall.SOL_SOCKET, syscall.SO_REUSEADDR, 1)
            })
        },
    }
    
    ln, _ := lc.Listen(nil, "tcp", ":8080")
    return ln
}

func optimizedDial() net.Conn {
    dialer := &net.Dialer{
        Timeout:   5 * time.Second,
        KeepAlive: 30 * time.Second,
        Control: func(network, address string, c syscall.RawConn) error {
            return c.Control(func(fd uintptr) {
                // 禁用Nagle
                syscall.SetsockoptInt(int(fd), 
                    syscall.IPPROTO_TCP, syscall.TCP_NODELAY, 1)
            })
        },
    }
    
    conn, _ := dialer.Dial("tcp", "server:8080")
    return conn
}
```

// 传统方式（4次拷贝）
func serveFileTraditional(w http.ResponseWriter, r *http.Request, path string) {
    file, _ := os.Open(path)
    defer file.Close()
    
    data, _ := ioutil.ReadAll(file)  // 读到用户空间

// 零拷贝方式（sendfile）
func serveFileZeroCopy(w http.ResponseWriter, r *http.Request, path string) {
    file, _ := os.Open(path)
    defer file.Close()
    
    stat, _ := file.Stat()
    
    // Go 1.11+ 自动使用sendfile

}

// 手动sendfile（使用syscall）
func sendfileOut(fd int, file *os.File, offset *int64, count int) error {
    _, _, errno := syscall.Syscall6(
        syscall.SYS_SENDFILE,
        uintptr(fd),
        uintptr(file.Fd()),
        uintptr(unsafe.Pointer(offset)),
        uintptr(count),
        0,
        0,
    )
    if errno != 0 {
        return errno
    }
    return nil
}

**2. mmap实现（大文件处理）**：
```go
package main

import (
    "os"
    "syscall"
    "unsafe"
)

// MmapFile 内存映射文件
type MmapFile struct {
    data []byte
    file *os.File
}

func OpenMmap(path string) (*MmapFile, error) {
    file, err := os.OpenFile(path, os.O_RDWR, 0644)
    if err != nil {
        return nil, err
    }
    
    stat, err := file.Stat()
    if err != nil {
        file.Close()
        return nil, err
    }
    
    size := stat.Size()
    if size == 0 {
        file.Close()
        return nil, nil
    }
    
    data, err := syscall.Mmap(
        int(file.Fd()),
        0,
        int(size),
        syscall.PROT_READ|syscall.PROT_WRITE,
        syscall.MAP_SHARED,
    )
    if err != nil {
        file.Close()
        return nil, err
    }
    
    return &MmapFile{data: data, file: file}, nil
}

func (m *MmapFile) ReadAt(offset int64, p []byte) int {
    return copy(p, m.data[offset:])
}

func (m *MmapFile) WriteAt(offset int64, p []byte) int {
    return copy(m.data[offset:], p)
}

func (m *MmapFile) Close() error {
    syscall.Munmap(m.data)
    return m.file.Close()
}

// 使用示例
func processLargeFile(path string) error {
    mf, err := OpenMmap(path)
    if err != nil {
        return err
    }
    defer mf.Close()
    
    // 直接操作内存，无拷贝
    header := make([]byte, 1024)
    mf.ReadAt(0, header)
    
    // 修改数据
    mf.WriteAt(0, []byte("NEW HEADER"))
    
    // 自动同步到磁盘
    return nil
}
```

**3. splice实现（管道零拷贝）**：
```go
// splice在管道间直接传输数据，无需用户空间参与
// Linux 2.6.17+ 支持

func spliceCopy(dst, src int, length int) (int64, error) {
    // 创建管道
    pipe := make([]int, 2)
    if err := syscall.Pipe2(pipe, syscall.O_CLOEXEC); err != nil {
        return 0, err
    }
    defer syscall.Close(pipe[0])
    defer syscall.Close(pipe[1])
    
    var total int64
    for total < int64(length) {
        // 从src splice到管道
        n, _, errno := syscall.Syscall6(
            syscall.SYS_SPLICE,
            uintptr(src), 0,
            uintptr(pipe[1]), 0,
            uintptr(length-int(total)),
            syscall.SPLICE_F_MOVE,
        )
        if errno != 0 {
            return total, errno
        }
        if n == 0 {
            break
        }
        
        // 从管道splice到dst
        m, _, errno := syscall.Syscall6(
            syscall.SYS_SPLICE,
            uintptr(pipe[0]), 0,
            uintptr(dst), 0,
            n,
            syscall.SPLICE_F_MOVE,
        )
        if errno != 0 {
            return total, errno
        }
        
        total += int64(m)
    }
    
    return total, nil
}
```

### 高性能网络编程

**1. epoll实现**：
```go
package main

import (
    "syscall"
    "unsafe"
)

// Epoll封装
type Epoll struct {
    fd     int
    events []syscall.EpollEvent
}

func NewEpoll() (*Epoll, error) {
    fd, err := syscall.EpollCreate1(syscall.EPOLL_CLOEXEC)
    if err != nil {
        return nil, err
    }
    return &Epoll{
        fd:     fd,
        events: make([]syscall.EpollEvent, 1024),
    }, nil
}

func (e *Epoll) Add(fd int, events uint32) error {
    ev := syscall.EpollEvent{
        Events: events,
        Fd:     int32(fd),
    }
    return syscall.EpollCtl(e.fd, syscall.EPOLL_CTL_ADD, fd, &ev)
}

func (e *Epoll) Mod(fd int, events uint32) error {
    ev := syscall.EpollEvent{
        Events: events,
        Fd:     int32(fd),
    }
    return syscall.EpollCtl(e.fd, syscall.EPOLL_CTL_MOD, fd, &ev)
}

func (e *Epoll) Del(fd int) error {
    return syscall.EpollCtl(e.fd, syscall.EPOLL_CTL_DEL, fd, nil)
}

func (e *Epoll) Wait(timeout int) ([]syscall.EpollEvent, error) {
    n, err := syscall.EpollWait(e.fd, e.events, timeout)
    if err != nil {
        return nil, err
    }
    return e.events[:n], nil
}

func (e *Epoll) Close() error {
    return syscall.Close(e.fd)
}

// 事件循环
func eventLoop(epoll *Epoll, handler func(fd int, events uint32)) {
    for {
        events, err := epoll.Wait(-1)
        if err != nil {
            continue
        }
        
        for _, ev := range events {
            handler(int(ev.Fd), ev.Events)
        }
    }
}
```

**2. 连接池实现**：
```go
type ConnPool struct {
    addr     string
    maxConn  int
    idleConn chan net.Conn
    active   int32
}

func NewConnPool(addr string, maxConn int) *ConnPool {
    return &ConnPool{
        addr:     addr,
        maxConn:  maxConn,
        idleConn: make(chan net.Conn, maxConn),
    }
}

func (p *ConnPool) Get() (net.Conn, error) {
    select {
    case conn := <-p.idleConn:
        return conn, nil
    default:
        if atomic.LoadInt32(&p.active) < int32(p.maxConn) {
            atomic.AddInt32(&p.active, 1)
            return net.Dial("tcp", p.addr)
        }
        // 等待空闲连接
        return <-p.idleConn, nil
    }
}

func (p *ConnPool) Put(conn net.Conn) {
    select {
    case p.idleConn <- conn:
    default:
        conn.Close()
        atomic.AddInt32(&p.active, -1)
    }
}
```
