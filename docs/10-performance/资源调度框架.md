## 📋 执行摘要

资源调度框架是分布式系统的核心基础设施，负责将计算任务高效地分配到集群资源上。本文档深入分析YARN、Mesos、Kubernetes和Omega四种主流调度架构的设计理念、实现机制和适用场景。

---

## 一、核心概念

### 1.1 定义与原理

**资源调度**是指在分布式集群中，根据任务需求和资源可用性，将计算任务分配到合适的计算节点上的过程。

**核心挑战**：
- **多租户隔离**：不同用户/应用间的资源隔离
- **资源利用率**：最大化集群资源使用效率
- **调度延迟**：快速响应任务调度请求
- **公平性与优先级**：平衡不同用户的资源需求

### 1.2 关键特性

- **可扩展性**：支持大规模集群（10K+节点）
- **容错性**：节点故障时的任务恢复
- **灵活性**：支持多种应用类型
- **资源隔离**：CPU、内存、网络、磁盘IO隔离

### 1.3 适用场景

| 场景 | 适用性 | 说明 |
|------|--------|------|
| 大数据批处理 | ⭐⭐⭐⭐⭐ | YARN + Hadoop/Spark生态 |
| 容器化微服务 | ⭐⭐⭐⭐⭐ | Kubernetes事实标准 |
| 混合负载调度 | ⭐⭐⭐⭐ | Mesos的多框架支持 |
| 高性能计算 | ⭐⭐⭐⭐ | Slurm、PBS等专业调度器 |
| 低延迟流处理 | ⭐⭐⭐⭐ | 需要专用调度优化 |

```java
// ResourceManager - 全局资源管理
public class ResourceManager {
    // 调度器：选择哪个应用获得资源
    private Scheduler scheduler;
    
    // 应用管理器：管理所有ApplicationMaster
    private ApplicationsManager appsManager;
    
    // 资源分配决策
    public Allocation allocate(ApplicationId appId, 
                               ResourceRequest request) {
        // 1. 检查应用资源配额
        if (!checkQuota(appId, request)) {
            return Allocation.EMPTY;
        }
        
        // 2. 调度器选择合适的节点
        NodeId node = scheduler.selectNode(request);
        
        // 3. 生成Container分配
        Container container = Container.newInstance(
            containerId, node, request.getResource());
        
        return new Allocation(Collections.singletonList(container));
    }
}

// ApplicationMaster - 单个应用的任务协调
public class ApplicationMaster {
    private AMRMClientAsync resourceManager;
    private NMClientAsync nodeManager;
    
    public void run() {
        // 1. 向RM注册
        resourceManager.registerApplicationMaster(
            amHost, amRpcPort, trackingUrl);
        
        // 2. 请求资源
        List<ResourceRequest> requests = createResourceRequests();
        resourceManager.addContainerRequest(requests);
        
        // 3. 获取分配的资源并启动任务
        while (!completed) {
            AllocateResponse response = resourceManager.allocate(progress);
            for (Container container : response.getAllocatedContainers()) {
                launchTask(container);
            }
        }
        
        // 4. 注销
        resourceManager.unregisterApplicationMaster(status);
    }
}
```

**Mesos资源分配流程**：

```java
// Framework调度器
public class HadoopScheduler implements Scheduler {
    
    @Override
    public void resourceOffers(SchedulerDriver driver, 
                               List<Offer> offers) {
        for (Offer offer : offers) {
            // 检查offer资源是否满足需求
            if (satisfiesRequirements(offer, pendingTasks)) {
                // 创建任务
                List<TaskInfo> tasks = new ArrayList<>();
                for (TaskRequest request : pendingTasks) {
                    if (fits(offer, request)) {
                        TaskInfo task = TaskInfo.newBuilder()
                            .setName(request.getName())
                            .setTaskId(TaskID.newBuilder()
                                .setValue(UUID.randomUUID().toString()))
                            .setSlaveId(offer.getSlaveId())
                            .addResources(createCpuResource(request.getCpus()))
                            .addResources(createMemResource(request.getMem()))
                            .setExecutor(getExecutorInfo())
                            .build();
                        tasks.add(task);
                    }
                }
                // 接受offer并启动任务
                driver.launchTasks(offer.getId(), tasks);
            } else {
                // 拒绝offer
                driver.declineOffer(offer.getId());
            }
        }
    }
}
```

**Kubernetes调度算法**：

```go
// 调度主流程
func (sched *Scheduler) scheduleOne(ctx context.Context) {
    // 1. 获取待调度Pod
    podInfo := sched.NextPod()
    
    // 2. 过滤阶段 (Predicates)
    feasibleNodes, err := sched.findNodesThatFitPod(ctx, podInfo.Pod)
    if len(feasibleNodes) == 0 {
        // 无可行节点，Pod进入不可调度状态
        sched.recordSchedulingFailure(...)
        return
    }
    
    // 3. 打分阶段 (Priorities)
    priorityList, err := sched.prioritizeNodes(ctx, podInfo.Pod, feasibleNodes)
    
    // 4. 选择最优节点
    host := sched.selectHost(priorityList)
    
    // 5. 绑定
    err := sched.bind(ctx, podInfo.Pod, host)
}

// 关键Predicates（过滤条件）
var predicates = []string{
    "PodFitsResources",      // 资源充足
    "PodFitsHost",          // 节点名称匹配
    "PodFitsHostPorts",     // 端口不冲突
    "MatchNodeSelector",    // 节点选择器匹配
    "NoDiskConflict",       // 磁盘无冲突
    "PodToleratesNodeTaints", // 容忍污点
    "CheckNodeLabelPresence", // 节点标签
}

// 关键Priorities（打分策略）
var priorities = []string{
    "LeastRequestedPriority",    // 优先选择空闲资源多的节点
    "BalancedResourceAllocation", // CPU/内存均衡
    "SelectorSpreadPriority",    // 分散Pod
    "NodeAffinityPriority",      // 节点亲和性
    "TaintTolerationPriority",   // 污点容忍
    "ImageLocalityPriority",     // 镜像本地性
}
```

**Omega调度流程**：

```python
class OmegaScheduler:
    def __init__(self, cell_state):
        self.local_cell_state = cell_state.copy()  # 本地副本
        self.scheduler_type = self.get_type()       # batch/service/ml
    
    def schedule_task(self, task):
        # 1. 基于本地副本做调度决策
        placement = self.select_node(task, self.local_cell_state)
        
        # 2. 尝试提交到共享状态
        success = self.cell_state.commit_transaction(
            read_version=self.local_cell_state.version,
            changes=[
                AddTask(task, placement.node_id)
            ]
        )
        
        if success:
            # 3a. 提交成功，更新本地副本
            self.local_cell_state.apply_changes([AddTask(task, placement.node_id)])
            return placement
        else:
            # 3b. 冲突，获取最新状态重试
            self.local_cell_state = self.cell_state.get_full_copy()
            return self.schedule_task(task)  # 重试

# 不同调度器的专用策略
class BatchScheduler(OmegaScheduler):
    """批处理调度器：优先考虑数据局部性"""
    def select_node(self, task, cell_state):
        candidates = cell_state.get_nodes_with_data(task.input_data)
        return min(candidates, key=lambda n: n.current_load)

class ServiceScheduler(OmegaScheduler):
    """服务调度器：优先考虑分布和故障域"""
    def select_node(self, task, cell_state):
        candidates = cell_state.get_feasible_nodes(task)
        # 分散到不同故障域
        return select_best_for_ha(candidates, task.existing_pods)
```

## 五、形式化分析

### 5.1 调度问题建模

**定义**：
- $J = \{j_1, j_2, ..., j_n\}$：作业集合
- $M = \{m_1, m_2, ..., m_m\}$：机器集合
- $r_{ij}$：作业$j_i$在机器$m_j$上的资源需求
- $c_{ij}$：作业$j_i$在机器$m_j$上的执行成本

**优化目标**：
$$\min \sum_{i=1}^{n} \sum_{j=1}^{m} c_{ij} x_{ij}$$

**约束**：
$$\sum_{i=1}^{n} r_{ij} x_{ij} \leq R_j, \quad \forall j \in M$$
$$\sum_{j=1}^{m} x_{ij} = 1, \quad \forall i \in J$$
$$x_{ij} \in \{0, 1\}$$

### 5.2 复杂度分析

**通用调度问题是NP-hard的**

- 单调度器：$O(n \cdot m)$ 每次调度决策
- 双层调度：$O(n \cdot m)$ 框架级 + $O(1)$ 本地决策
- 共享状态：$O(n \cdot m)$ 但可并行，冲突重试概率$p$
