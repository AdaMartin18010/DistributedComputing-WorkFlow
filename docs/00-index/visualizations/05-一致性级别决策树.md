## 🔍 详细级别定义

### 1. 线性一致性 (Linearizability)

```mermaid
sequenceDiagram
    participant C1 as Client A
    participant C2 as Client B
    participant S as 存储系统

    Note over C1,C2: 时间轴
    C1->>S: Write(x=1)
    S-->>C1: OK
    C2->>S: Read(x)
    S-->>C2: x=1
    Note right of C2: 必定读到最新值
```

**特征**：

- 所有操作看起来都是原子的
- 任何读取都能看到最新的写入
- 等价于单副本系统

**实现系统**：etcd, ZooKeeper, Spanner, CockroachDB

### 2. 顺序一致性 (Sequential Consistency)

```mermaid
sequenceDiagram
    participant P1 as 进程P1
    participant P2 as 进程P2
    participant Mem as 共享内存

    P1->>Mem: Write(x=1)
    P1->>Mem: Write(y=2)
    P2->>Mem: Read(y)
    Mem-->>P2: 2
    P2->>Mem: Read(x)
    Mem-->>P2: 1
    Note right of P2: 所有进程看到相同顺序
```

**与线性一致性区别**：不要求实时性，只要求全局顺序一致

### 3. 因果一致性 (Causal Consistency)

```mermaid
graph TB
    subgraph "进程A"
        A1[Write x=1] --> A2[Write y=x+1]
    end

    subgraph "进程B"
        B1[Read y=2] --> B2[Write z=y*2]
    end

    subgraph "进程C"
        C1[Read x=1] --> C2[Write w=5]
    end

    A1 -.因果.-> A2
    A2 -.因果.-> B1
    B1 -.因果.-> B2

    style A2 fill:#ffe1b3
    style B1 fill:#ffe1b3
```

**依赖追踪**：

- 读取-写入依赖
- 写入-写入依赖
- 传递性因果

### 4. 会话一致性 (Session Consistency)

```mermaid
sequenceDiagram
    participant Client
    participant Primary as 主副本
    participant Replica as 从副本

    Note over Client,Replica: 会话绑定到主副本
    Client->>Primary: Write x=1
    Primary-->>Client: OK
    Client->>Primary: Read x
    Primary-->>Client: x=1
    Note right of Client: 同一会话保证
```

### 5. 最终一致性 (Eventual Consistency)

```mermaid
stateDiagram-v2
    [*] --> 副本A写入: Write
    副本A写入 --> 复制进行中
    复制进行中 --> 副本B更新: 异步复制
    副本B更新 --> 最终一致: 收敛
    最终一致 --> [*]

    note right of 复制进行中
        短暂不一致窗口
    end note
```

## 🎯 业务场景匹配

| 业务场景 | 推荐一致性 | 代表系统 | 理由 |
|----------|------------|----------|------|
| 银行转账 | 线性一致性 | Spanner | 资金准确性至上 |
| 库存扣减 | 线性一致性/顺序一致 | etcd | 避免超卖 |
| 社交点赞 | 因果一致性 | COPS | 因果关系重要 |
| 用户评论 | 会话一致性 | DynamoDB | 用户自己可见 |
| 商品列表 | 最终一致性 | Cassandra | 高可用优先 |
| 日志记录 | 最终一致性 | Kafka | 吞吐优先 |
| 配置管理 | 线性一致性 | etcd | 配置必须准确 |
| 推荐结果 | 最终一致性 | Redis | 非关键数据 |
