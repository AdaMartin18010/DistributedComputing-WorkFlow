# 流处理语义详解 - Exactly-Once

## 1. 流处理语义概述

流处理系统中的语义定义了数据处理的一致性保证级别。根据故障恢复时的行为，可分为三种语义级别。

```
┌─────────────────────────────────────────────────────────────────────┐
│                   流处理语义对比                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  At-Most-Once (最多一次)                                           │
│  ┌─────────┐      故障            ┌─────────┐                      │
│  │ Event 1 │────────────────────▶ │ 丢失    │                      │
│  │ Event 2 │────────┐             └─────────┘                      │
│  │ Event 3 │        └──────────▶ │  处理  │                        │
│  └─────────┘                     └─────────┘                       │
│                                                                     │
│  特点：不保证数据不丢失，最简单但最不安全                            │
│  适用：日志收集、监控指标（可容忍丢失）                              │
│                                                                     │
│  ───────────────────────────────────────────────────────────────── │
│                                                                     │
│  At-Least-Once (至少一次)                                          │
│  ┌─────────┐      故障            ┌─────────┐                      │
│  │ Event 1 │────────────────────▶ │ 重放    │                      │
│  │ Event 2 │────────┐    ┌──────▶│ Event 1 │                      │
│  │ Event 3 │        └────┘        │ Event 2 │  ← 重复处理          │
│  └─────────┘                     └─────────┘                       │
│                                                                     │
│  特点：保证数据不丢失，但可能重复处理                                │
│  适用：大多数流处理场景（需幂等处理或去重）                          │
│                                                                     │
│  ───────────────────────────────────────────────────────────────── │
│                                                                     │
│  Exactly-Once (精确一次)                                           │
│  ┌─────────┐      故障            ┌─────────┐                      │
│  │ Event 1 │────────────────────▶ │ 重放    │                      │
│  │ Event 2 │────────┐    ┌──────▶│ Event 1 │                      │
│  │ Event 3 │        └────┘        │ Event 2 │                      │
│  └─────────┘         ▲           └────┬────┘                       │
│                      │                │                            │
│                      └──── 检查点 ────┘                            │
│                                                                     │
│  特点：保证数据不丢失且不重复，最强一致性                            │
│  适用：金融交易、计费系统、订单处理                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 2. Exactly-Once 实现原理

```
┌─────────────────────────────────────────────────────────────────────┐
│                Exactly-Once 实现机制                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  1. 幂等性生产者 (Idempotent Producer)                              │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  Producer                                                   │   │
│  │  ┌─────────┐  PID: 12345  Sequence: 0  ──▶  ┌─────────┐    │   │
│  │  │ Event 1 │───────────────────────────────▶│ Topic-A │    │   │
│  │  └─────────┘                              └─────────┘    │   │
│  │       │                                                   │   │
│  │       │  网络故障，重试                                   │   │
│  │       ▼                                                   │   │
│  │  ┌─────────┐  PID: 12345  Sequence: 0  ──▶  ┌─────────┐    │   │
│  │  │ Event 1 │───────────────────────────────▶│  去重    │    │   │
│  │  └─────────┘                              └─────────┘    │   │
│  │                                             (重复丢弃)    │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  2. 事务机制 (Transactions)                                         │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  事务边界                                                   │   │
│  │  ┌─────────┐  Begin TX  ┌─────────┐  Commit TX  ┌─────────┐│   │
│  │  │ 消费    │───────────▶│ 处理    │────────────▶│ 生产    ││   │
│  │  │ Offset  │            │ Event   │             │ Output  ││   │
│  │  └─────────┘            └─────────┘             └─────────┘│   │
│  │       │                       │                       │     │   │
│  │       └───────────────────────┴───────────────────────┘     │   │
│  │                        原子提交                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  3. 两阶段提交 (2PC)                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  Phase 1: Prepare                                           │   │
│  │  Coordinator ──▶ Participant 1 (Ready?)                    │   │
│  │            ──▶ Participant 2 (Ready?)                      │   │
│  │            ──▶ Participant N (Ready?)                      │   │
│  │                                                              │   │
│  │  Phase 2: Commit                                            │   │
│  │  Coordinator ──▶ All Participants (Commit!)                │   │
│  │                                                              │   │
│  │  或 Abort                                                   │   │
│  │  Coordinator ──▶ All Participants (Abort!)                 │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 3. Flink Exactly-Once 实现

### 3.1 Checkpoint 机制

```java
import org.apache.flink.streaming.api.environment.CheckpointConfig;
import org.apache.flink.streaming.api.environment.StreamExecutionEnvironment;

StreamExecutionEnvironment env = StreamExecutionEnvironment.getExecutionEnvironment();

// 启用 Checkpoint
env.enableCheckpointing(60000);  // 每 60 秒

// 设置 Checkpointing 模式
env.getCheckpointConfig().setCheckpointingMode(CheckpointingMode.EXACTLY_ONCE);

// Checkpoint 配置
env.getCheckpointConfig().setMinPauseBetweenCheckpoints(30000);
env.getCheckpointConfig().setCheckpointTimeout(600000);
env.getCheckpointConfig().setMaxConcurrentCheckpoints(1);

// 启用外部化 Checkpoint
env.getCheckpointConfig().enableExternalizedCheckpoints(
    ExternalizedCheckpointCleanup.RETAIN_ON_CANCELLATION);

// 设置状态后端
env.setStateBackend(new HashMapStateBackend());
env.getCheckpointConfig().setCheckpointStorage("hdfs:///flink/checkpoints");

// 精确一次语义配置
env.getCheckpointConfig().setTolerableCheckpointFailureNumber(3);
```

### 3.2 两阶段提交 Sink

```java
import org.apache.flink.api.common.state.ListState;
import org.apache.flink.api.common.state.ListStateDescriptor;
import org.apache.flink.runtime.state.FunctionInitializationContext;
import org.apache.flink.runtime.state.FunctionSnapshotContext;
import org.apache.flink.streaming.api.checkpoint.CheckpointedFunction;
import org.apache.flink.streaming.api.functions.sink.TwoPhaseCommitSinkFunction;

public class ExactlyOnceKafkaSink extends TwoPhaseCommitSinkFunction<String, KafkaTransaction, Void> {

    private transient KafkaProducer<String, String> producer;

    public ExactlyOnceKafkaSink() {
        super(TypeInformation.of(String.class).createSerializer(new ExecutionConfig()),
              TypeInformation.of(Void.class).createSerializer(new ExecutionConfig()));
    }

    @Override
    protected void invoke(KafkaTransaction transaction, String value, Context context) {
        // 将数据添加到当前事务
        transaction.producer.send(new ProducerRecord<>("output-topic", value));
    }

    @Override
    protected KafkaTransaction beginTransaction() {
        // 开始新事务
        KafkaProducer<String, String> producer = createProducer();
        producer.initTransactions();
        producer.beginTransaction();
        return new KafkaTransaction(producer);
    }

    @Override
    protected void preCommit(KafkaTransaction transaction) {
        // 预提交：刷写数据
        transaction.producer.flush();
    }

    @Override
    protected void commit(KafkaTransaction transaction) {
        // 正式提交事务
        transaction.producer.commitTransaction();
        transaction.producer.close();
    }

    @Override
    protected void abort(KafkaTransaction transaction) {
        // 回滚事务
        transaction.producer.abortTransaction();
        transaction.producer.close();
    }
}
```

## 4. Kafka Exactly-Once 配置

### 4.1 生产者配置

```java
Properties props = new Properties();
props.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
props.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG,
          StringSerializer.class.getName());
props.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG,
          StringSerializer.class.getName());

// 启用幂等性
props.put(ProducerConfig.ENABLE_IDEMPOTENCE_CONFIG, "true");
props.put(ProducerConfig.ACKS_CONFIG, "all");
props.put(ProducerConfig.RETRIES_CONFIG, Integer.MAX_VALUE);
props.put(ProducerConfig.MAX_IN_FLIGHT_REQUESTS_PER_CONNECTION, "5");

// 事务配置
props.put(ProducerConfig.TRANSACTIONAL_ID_CONFIG, "my-transactional-id");
props.put(ProducerConfig.TRANSACTION_TIMEOUT_CONFIG, "60000");

KafkaProducer<String, String> producer = new KafkaProducer<>(props);

// 事务方式发送
producer.initTransactions();

try {
    producer.beginTransaction();

    producer.send(new ProducerRecord<>("topic1", "key1", "value1"));
    producer.send(new ProducerRecord<>("topic2", "key2", "value2"));

    // 发送消费位点
    producer.sendOffsetsToTransaction(
        consumer.position(consumer.assignment()),
        consumer.groupMetadata()
    );

    producer.commitTransaction();
} catch (Exception e) {
    producer.abortTransaction();
    throw e;
}
```

## 5. Spark Structured Streaming Exactly-Once

```python
from pyspark.sql import SparkSession

spark = SparkSession.builder \
    .appName("ExactlyOnce") \
    .getOrCreate()

# Kafka Source 配置
df = spark.readStream \
    .format("kafka") \
    .option("kafka.bootstrap.servers", "localhost:9092") \
    .option("subscribe", "input-topic") \
    .option("startingOffsets", "latest") \
    .option("failOnDataLoss", "false") \
    .load()

# 处理逻辑
processed = df.selectExpr("CAST(value AS STRING)")

# Exactly-Once 输出
query = processed.writeStream \
    .format("kafka") \
    .option("kafka.bootstrap.servers", "localhost:9092") \
    .option("topic", "output-topic") \
    .option("checkpointLocation", "/path/to/checkpoint") \
    .start()

query.awaitTermination()
```

## 7. 幂等性设计模式

```java
// 幂等性写入示例
public class IdempotentWriter {

    private final Map<String, Long> processedIds = new ConcurrentHashMap<>();

    public void write(Event event) {
        String id = event.getId();
        long timestamp = event.getTimestamp();

        // 检查是否已处理
        Long existingTimestamp = processedIds.get(id);
        if (existingTimestamp != null && existingTimestamp >= timestamp) {
            // 已处理过更新的版本，忽略
            return;
        }

        // 执行写入
        doWrite(event);

        // 记录已处理
        processedIds.put(id, timestamp);
    }

    // 基于数据库唯一约束
    public void writeWithUpsert(Event event) {
        // INSERT ... ON CONFLICT (id) DO UPDATE
        String sql = """
            INSERT INTO events (id, data, timestamp)
            VALUES (?, ?, ?)
            ON CONFLICT (id) DO UPDATE
            SET data = EXCLUDED.data,
                timestamp = EXCLUDED.timestamp
            WHERE EXCLUDED.timestamp > events.timestamp
            """;
        jdbcTemplate.update(sql, event.getId(), event.getData(), event.getTimestamp());
    }
}
```

## 8. 语义选择指南

| 场景 | 推荐语义 | 实现方式 |
|------|---------|----------|
| 日志收集 | At-Most-Once | 异步发送，不重试 |
| 监控指标 | At-Least-Once | 重试 + 幂等 |
| 实时推荐 | At-Least-Once | 重试 + 去重 |
| 订单处理 | Exactly-Once | 事务 + 幂等 |
| 金融交易 | Exactly-Once | 2PC + 幂等 |
| 计费系统 | Exactly-Once | 事务 + 对账 |

## 10. 总结

Exactly-Once 语义是流处理中最强的一致性保证，但其实现复杂且有一定性能开销。在选择语义时，应根据业务场景和可接受的数据重复程度进行权衡。

关键要点：

1. **幂等性**是实现 Exactly-Once 的基础
2. **事务机制**保证多操作的原子性
3. **Checkpoint** 提供故障恢复能力
4. **两阶段提交**实现端到端一致性
5. 根据业务需求选择合适的语义级别
