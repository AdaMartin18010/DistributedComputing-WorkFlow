# 批量处理：Pipeline与Batch

## 核心概念

### 1. 批量处理的优势

批量处理通过合并多个操作，减少网络往返和系统调用次数。

**性能对比**：

```
┌─────────────────────────────────────────────────────────────┐
│                   批量处理优势                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   逐条处理（1000条）：                                       │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  for i := 0; i < 1000; i++ {                        │  │
│   │      send(request[i]) ──→ 等待响应 ──→ 处理下一条   │  │
│   │  }                                                  │  │
│   │                                                     │  │
│   │  总时间 = 1000 × (网络RTT + 处理时间)                │  │
│   │  假设RTT=1ms，总时间=1000ms                          │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   批量处理（100条一批）：                                     │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  for i := 0; i < 10; i++ {                          │  │
│   │      send(batch[i]) ──→ 等待响应 ──→ 处理下批       │  │
│   │  }                                                  │  │
│   │                                                     │  │
│   │  总时间 = 10 × (网络RTT + 处理时间)                  │  │
│   │  假设RTT=1ms，总时间=10ms                            │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   收益：100倍性能提升！                                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**适用场景**：

| 场景 | 说明 | 批量策略 |
|------|------|----------|
| 数据库插入 | 批量INSERT | 500-1000条/批 |
| Redis操作 | Pipeline | 100-1000命令/批 |
| HTTP请求 | 合并请求 | 根据API限制 |
| 文件写入 | Buffer写入 | 4KB-64KB/批 |
| 消息发送 | 批量发送 | 100条/批或100ms |

## 实践方法

### 数据库批量操作

**1. JDBC批量插入**：

```java
public void batchInsert(List<Order> orders) {
    String sql = "INSERT INTO orders (id, user_id, amount) VALUES (?, ?, ?)";

    try (Connection conn = dataSource.getConnection();
         PreparedStatement ps = conn.prepareStatement(sql)) {

        int batchSize = 1000;
        int count = 0;

        for (Order order : orders) {
            ps.setLong(1, order.getId());
            ps.setLong(2, order.getUserId());
            ps.setBigDecimal(3, order.getAmount());
            ps.addBatch();

            if (++count % batchSize == 0) {
                ps.executeBatch();
                ps.clearBatch();
            }
        }

        // 执行剩余
        ps.executeBatch();
    }
}

// 使用rewriteBatchedStatements=true
// jdbc:mysql://localhost/mydb?rewriteBatchedStatements=true
```

### Redis Pipeline

```java
public void batchGet(List<String> keys) {
    List<Object> results = redisTemplate.executePipelined(
        new RedisCallback<Object>() {
            @Override
            public Object doInRedis(RedisConnection connection) {
                for (String key : keys) {
                    connection.get(key.getBytes());
                }
                return null;
            }
        }
    );
    // results包含所有key的value
}

// 批量写入
public void batchSet(Map<String, String> kvMap) {
    redisTemplate.executePipelined(
        new RedisCallback<Object>() {
            @Override
            public Object doInRedis(RedisConnection connection) {
                kvMap.forEach((k, v) -> {
                    connection.set(k.getBytes(), v.getBytes());
                });
                return null;
            }
        }
    );
}
```

### 批量处理框架

**1. 批量写入器**：

```go
type BatchWriter struct {
    buffer   []Record
    maxSize  int
    flushInterval time.Duration
    writer   Writer
    mu       sync.Mutex
    timer    *time.Timer
}

func NewBatchWriter(writer Writer, maxSize int, flushInterval time.Duration) *BatchWriter {
    bw := &BatchWriter{
        buffer:  make([]Record, 0, maxSize),
        maxSize: maxSize,
        flushInterval: flushInterval,
        writer:  writer,
    }
    go bw.flushLoop()
    return bw
}

func (bw *BatchWriter) Write(record Record) error {
    bw.mu.Lock()
    defer bw.mu.Unlock()

    bw.buffer = append(bw.buffer, record)

    if len(bw.buffer) >= bw.maxSize {
        return bw.flush()
    }

    return nil
}

func (bw *BatchWriter) flush() error {
    if len(bw.buffer) == 0 {
        return nil
    }

    batch := bw.buffer
    bw.buffer = make([]Record, 0, bw.maxSize)

    return bw.writer.WriteBatch(batch)
}

func (bw *BatchWriter) flushLoop() {
    ticker := time.NewTicker(bw.flushInterval)
    defer ticker.Stop()

    for range ticker.C {
        bw.mu.Lock()
        bw.flush()
        bw.mu.Unlock()
    }
}
```

## 案例数据

### 数据导入优化

**场景**：导入100万条订单数据

**逐条插入**：

```
时间：30分钟
数据库负载：高
网络开销：大
```

**批量插入（1000条/批）**：

```
时间：30秒
数据库负载：中
网络开销：小
性能提升：60倍
```

**优化配置**：

```sql
-- 临时禁用索引、外键检查
SET FOREIGN_KEY_CHECKS=0;
SET UNIQUE_CHECKS=0;
SET AUTOCOMMIT=0;

-- 批量插入
INSERT INTO orders ...;
INSERT INTO orders ...;
...

COMMIT;

-- 恢复检查
SET FOREIGN_KEY_CHECKS=1;
SET UNIQUE_CHECKS=1;
```

## 优化前后对比

| 场景 | 逐条处理 | 批量处理 | 改进 |
|------|----------|----------|------|
| DB插入100万条 | 30min | 30s | 60倍↑ |
| Redis 1000次 | 1000ms | 10ms | 100倍↑ |
| HTTP 100请求 | 10s | 500ms | 20倍↑ |
| 文件写入1MB | 1000次调用 | 1次调用 | 1000倍↑ |

---

**总结**：批量处理是提升性能的有效手段。通过合并操作减少网络往返和系统调用，可以获得数量级的性能提升。需要根据具体场景选择合适的批量大小，平衡延迟和吞吐量。
