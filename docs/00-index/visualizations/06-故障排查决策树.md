## 🎯 主决策树：系统为什么慢？

```mermaid
graph TD
    A[🔴 系统性能问题] --> B{网络延迟高?}

    B -->|是| C{延迟类型?}
    B -->|否| D{CPU使用率高?}

    %% 网络问题分支
    C -->|跨可用区| E[检查网络拓扑]
    C -->|随机抖动| F[检查网络拥塞]
    C -->|持续高延迟| G[检查带宽限制]

    E --> E1["解决方案：<br/>• 就近部署<br/>• 专线连接<br/>• 数据本地性优化"]
    F --> F1["解决方案：<br/>• 流量整形<br/>• QoS优先级<br/>• 拥塞控制调优"]
    G --> G1["解决方案：<br/>• 升级带宽<br/>• 压缩传输<br/>• 批量请求"]

    %% CPU问题分支
    D -->|是| H{CPU使用模式?}
    D -->|否| I{内存使用率高?}

    H -->|单核满载| J[检查热点/锁竞争]
    H -->|多核均衡| K[检查计算密集型任务]
    H -->|间歇性峰值| L[检查GC/STW]

    J --> J1["解决方案：<br/>• 分片打散热点<br/>• 优化锁粒度<br/>• 无锁数据结构"]
    K --> K1["解决方案：<br/>• 水平扩容<br/>• 异步化改造<br/>• 算法优化"]
    L --> L1["解决方案：<br/>• 调整GC策略<br/>• 增加堆内存<br/>• 对象池优化"]

    %% 内存问题分支
    I -->|是| M{内存使用模式?}
    I -->|否| N{磁盘IO高?}

    M -->|持续增长| O[检查内存泄漏]
    M -->|频繁交换| P[检查内存不足]
    M -->|缓存占用大| Q[检查缓存命中率]

    O --> O1["解决方案：<br/>• Heap Dump分析<br/>• 引用链追踪<br/>• 修复泄漏点"]
    P --> P1["解决方案：<br/>• 增加物理内存<br/>• 限制进程内存<br/>• 优化数据缓存"]
    Q --> Q1["解决方案：<br/>• 调整缓存策略<br/>• 增大缓存容量<br/>• 优化缓存Key"]

    %% IO问题分支
    N -->|是| R{IO类型?}
    N -->|否| S{第三方依赖慢?}

    R -->|随机读高| T[检查索引缺失]
    R -->|顺序写慢| U[检查磁盘类型]
    R -->|fsync延迟| V[检查同步策略]

    T --> T1["解决方案：<br/>• 添加索引<br/>• 查询优化<br/>• 读写分离"]
    U --> U1["解决方案：<br/>• SSD替换HDD<br/>• RAID配置<br/>• 预分配文件"]
    V --> V1["解决方案：<br/>• 批量刷盘<br/>• 异步刷盘<br/>• 调内核参数"]

    %% 依赖问题分支
    S -->|是| W{依赖类型?}
    S -->|否| X[深入应用层分析]

    W -->|数据库慢| Y[检查慢查询/连接池]
    W -->|缓存慢| Z[检查热Key/大Key]
    W -->|下游服务| AA[检查超时/重试]

    Y --> Y1["解决方案：<br/>• SQL优化<br/>• 连接池调优<br/>• 分库分表"]
    Z --> Z1["解决方案：<br/>• Key拆分<br/>• 本地缓存<br/>• 读写分离"]
    AA --> AA1["解决方案：<br/>• 熔断降级<br/>• 超时设置<br/>• 异步解耦"]
```

## 🌲 子决策树：详细诊断路径

### 网络问题诊断

```mermaid
graph TD
    A[网络问题] --> B{是否丢包?}
    B -->|是| C{丢包位置?}
    B -->|否| D[检查延迟分布]

    C -->|网卡| E[检查Ring Buffer]
    C -->|交换机| F[检查端口拥塞]
    C -->|中间网络| G[使用MTR追踪]

    D --> H{P99延迟?}
    H -->|>100ms| I[检查长连接池]
    H -->|<10ms| J[正常范围]
    H -->|抖动大| K[检查干扰因素]
```

### 数据库慢查询诊断

```mermaid
graph TD
    A[数据库慢] --> B{读慢还是写慢?}

    B -->|读慢| C{查询类型?}
    B -->|写慢| D{写入模式?}

    C -->|单条查询| E[检查索引/执行计划]
    C -->|聚合查询| F[检查统计信息]
    C -->|范围扫描| G[检查分区策略]

    D -->|随机写| H[检查磁盘IOPS]
    D -->|批量写| I[检查事务大小]
    D -->|更新冲突| J[检查锁等待]
```

## 📊 关键指标检查清单

### RED 指标检查

| 指标 | 健康阈值 | 检查工具 | 常见原因 |
|------|----------|----------|----------|
| Request Rate | 符合预期 | Prometheus | 突发流量/爬虫 |
| Error Rate | < 0.1% | Grafana | 依赖故障/代码Bug |
| Duration P99 | < SLA | APM | 上面决策树分析 |

### USE 指标检查

| 资源 | 利用率 | 饱和度 | 错误 |
|------|--------|--------|------|
| CPU | < 70% | Load < CPU*2 | 无 |
| 内存 | < 80% | Swap使用 = 0 | OOM次数 = 0 |
| 磁盘 | < 70% | IO Wait < 20% | 磁盘错误 = 0 |
| 网络 | < 70% | 重传率 < 1% | 丢包 = 0 |

## 🔧 常用诊断命令

```bash
# 网络诊断
ping -c 100 $HOST          # 基础连通性
mtr --report $HOST          # 路径分析
ss -tan | grep ESTAB | wc -l  # 连接数统计
nicstat 1 10               # 网卡统计

# CPU诊断
top -H -p $PID             # 线程级CPU
perf top -p $PID           # 热点函数
mpstat -P ALL 1            # 核级统计

# 内存诊断
vmstat 1 10                # 内存/交换
pmap -x $PID | tail -1     # 进程内存
jmap -histo $PID           # Java对象统计

# IO诊断
iostat -xz 1 10            # 磁盘IO
iotop -p $PID              # 进程IO
strace -e trace=file $CMD  # 系统调用追踪
```
