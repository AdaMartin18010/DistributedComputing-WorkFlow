# 零拷贝技术：sendfile与mmap

## 核心概念

### 1. 传统IO的数据拷贝

传统IO涉及多次数据拷贝和上下文切换，是性能瓶颈之一。

**传统文件传输**：

```
┌─────────────────────────────────────────────────────────────┐
│                   传统文件传输流程                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   磁盘文件 ──→ 用户请求下载                                 │
│                                                             │
│   步骤1: read() 系统调用                                    │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  磁盘 ──DMA→ 内核缓冲区 ──CPU拷贝→ 用户缓冲区       │  │
│   │         0ms           1ms                          │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   步骤2: write() 系统调用                                   │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  用户缓冲区 ──CPU拷贝→ Socket缓冲区 ──DMA→ 网卡    │  │
│   │         1ms              0ms                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   总计：                                                     │
│   - 数据拷贝：4次（2次DMA，2次CPU）                         │
│   - 上下文切换：4次（用户↔内核）                            │
│   - CPU参与：2次拷贝                                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 零拷贝技术

零拷贝技术通过减少数据拷贝次数，显著提升IO性能。

**sendfile零拷贝**：

```
┌─────────────────────────────────────────────────────────────┐
│                   sendfile零拷贝                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   sendfile(fd_in, fd_out, offset, count)                    │
│                                                             │
│   步骤：                                                     │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                     │  │
│   │  磁盘 ──DMA→ 内核缓冲区 ──→ 网卡                   │  │
│   │         0ms            0ms（描述符传递）             │  │
│   │                                                     │  │
│   │  Socket缓冲区只存储文件描述符和长度信息              │  │
│   │                                                     │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   总计：                                                     │
│   - 数据拷贝：2次（都是DMA）                                │
│   - 上下文切换：2次                                         │
│   - CPU参与：0次（数据不经过CPU）                           │
│                                                             │
│   性能提升：2-3倍                                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**mmap零拷贝**：

```
┌─────────────────────────────────────────────────────────────┐
│                   mmap内存映射                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   mmap() 将文件映射到进程地址空间                           │
│                                                             │
│   磁盘文件 ←──────────→ 物理内存                           │
│              ↑        ↑                                     │
│              │        │                                     │
│         用户空间 ←──→ 内核空间                             │
│              （共享同一物理页）                              │
│                                                             │
│   特点：                                                     │
│   - 用户直接读写内存，无需系统调用                          │
│   - 内核自动同步修改到磁盘                                  │
│   - 适合大文件随机访问                                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 实践方法

### sendfile应用

**1. Nginx静态文件**：

```nginx
# nginx.conf
server {
    location /static/ {
        root /var/www;
        sendfile on;           # 启用sendfile
        tcp_nopush on;         # 优化大包发送
        tcp_nodelay on;        # 优化小包发送
    }
}
```

**2. Java NIO零拷贝**：

```java
public class ZeroCopyExample {

    public void transferFile(String sourcePath, String destPath)
            throws IOException {
        try (FileChannel source = new FileInputStream(sourcePath).getChannel();
             FileChannel dest = new FileOutputStream(destPath).getChannel()) {

            // 零拷贝传输
            source.transferTo(0, source.size(), dest);
        }
    }

    // 网络传输
    public void sendFile(SocketChannel socketChannel, String filePath)
            throws IOException {
        try (FileChannel fileChannel = new FileInputStream(filePath).getChannel()) {
            fileChannel.transferTo(0, fileChannel.size(), socketChannel);
        }
    }
}
```

**3. Go sendfile**：

```go
func serveFile(w http.ResponseWriter, r *http.Request, path string) {
    file, err := os.Open(path)
    if err != nil {
        http.Error(w, err.Error(), 500)
        return
    }
    defer file.Close()

    stat, err := file.Stat()
    if err != nil {
        http.Error(w, err.Error(), 500)
        return
    }

    // Go自动使用sendfile
    http.ServeContent(w, r, stat.Name(), stat.ModTime(), file)
}
```

### mmap应用

**1. Java内存映射文件**：

```java
public class MmapExample {

    public void readLargeFile(String path) throws IOException {
        try (RandomAccessFile file = new RandomAccessFile(path, "r");
             FileChannel channel = file.getChannel()) {

            // 映射文件到内存
            MappedByteBuffer buffer = channel.map(
                FileChannel.MapMode.READ_ONLY, 0, channel.size());

            // 直接读写内存
            while (buffer.hasRemaining()) {
                byte b = buffer.get();
                // 处理数据
            }
        }
    }

    // 大文件处理
    public void processLargeFile(String path) throws IOException {
        int chunkSize = 1024 * 1024 * 100;  // 100MB

        try (RandomAccessFile file = new RandomAccessFile(path, "r");
             FileChannel channel = file.getChannel()) {

            long fileSize = channel.size();
            long position = 0;

            while (position < fileSize) {
                long remaining = fileSize - position;
                long mapSize = Math.min(chunkSize, remaining);

                MappedByteBuffer buffer = channel.map(
                    FileChannel.MapMode.READ_ONLY, position, mapSize);

                processBuffer(buffer);

                position += mapSize;
            }
        }
    }
}
```

## 案例数据

### 文件传输性能对比

| 方式 | 1GB文件传输时间 | CPU使用率 | 内存使用 |
|------|-----------------|-----------|----------|
| 传统IO | 10秒 | 80% | 1GB |
| sendfile | 3秒 | 10% | 几乎为0 |
| mmap | 3秒 | 15% | 按需加载 |

### Kafka零拷贝应用

Kafka使用sendfile和mmap实现高性能日志传输：

```
生产者 → 写入页缓存(mmap) → 异步刷盘
消费者 ← sendfile ← 页缓存 ← 磁盘

特点：
- 生产者：mmap写入，用户态操作
- 消费者：sendfile传输，零拷贝
- 全程CPU不参与数据搬运
```

|------|--------|--------|------|

| CPU使用 | 80% | 10% | 87%↓ |
| 内存占用 | 1GB | 几乎为0 | 100%↓ |

---

**总结**：零拷贝技术是高性能IO的核心。通过sendfile和mmap，可以大幅减少CPU拷贝和内存占用，显著提升文件传输和网络IO性能。对于高吞吐量的文件服务、消息队列等场景，零拷贝是必选项。
