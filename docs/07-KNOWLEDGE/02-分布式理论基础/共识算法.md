# 共识算法

## 概述

**共识算法（Consensus Algorithm）** 是分布式系统中用于让多个节点就某个值达成一致的算法。它是分布式系统的核心问题之一，广泛应用于分布式数据库、分布式存储、区块链等系统。

共识问题需要满足三个性质：

1. **一致性（Agreement）**：所有正确节点决定相同的值
2. **有效性（Validity）**：决定的值必须是某个节点提议的值
3. **终止性（Termination）**：所有正确节点最终都会决定

## 共识算法分类

| 分类维度 | 类型 |
|---------|------|
| **故障模型** | 崩溃容错（Crash Fault Tolerance）、拜占庭容错（Byzantine Fault Tolerance） |
| **网络模型** | 同步、异步、部分同步 |
| **领导者** | 有领导者（Leader-based）、无领导者（Leaderless） |
| **投票机制** | 多数派、加权投票、随机选择 |

## Paxos算法

### 基本Paxos

**提出者**：Leslie Lamport（1989年提出，1998年发表）

**核心思想**：使用多数派决策和两阶段提交达成共识。

**角色**：

- **提议者（Proposer）**：提出值的节点
- **接受者（Acceptor）**：接受或拒绝提议的节点
- **学习者（Learner）**：学习已决定值的节点

**算法伪代码**：

```python
class Proposer:
    def __init__(self, id, acceptors):
        self.id = id
        self.acceptors = acceptors
        self.proposal_num = 0

    def propose(self, value):
        # Phase 1: Prepare
        self.proposal_num = (self.proposal_num + 1, self.id)
        promises = []

        for acceptor in self.acceptors:
            promise = acceptor.prepare(self.proposal_num)
            if promise.accepted:
                promises.append(promise)

        if len(promises) < majority:
            return Failure

        # 选择已接受的最大编号的值，如果没有则使用自己的值
        chosen_value = value
        for promise in promises:
            if promise.has_accepted_value:
                if promise.proposal_num > chosen_value_proposal_num:
                    chosen_value = promise.accepted_value

        # Phase 2: Accept
        accepts = []
        for acceptor in self.acceptors:
            accept = acceptor.accept(self.proposal_num, chosen_value)
            if accept.accepted:
                accepts.append(accept)

        if len(accepts) >= majority:
            return Success(chosen_value)
        return Failure

class Acceptor:
    def __init__(self):
        self.max_promised = None
        self.accepted_proposal = None
        self.accepted_value = None

    def prepare(self, proposal_num):
        if self.max_promised is None or proposal_num > self.max_promised:
            self.max_promised = proposal_num
            return Promise(accepted=True,
                          has_accepted_value=self.accepted_value is not None,
                          accepted_value=self.accepted_value,
                          proposal_num=self.accepted_proposal)
        return Promise(accepted=False)

    def accept(self, proposal_num, value):
        if proposal_num >= self.max_promised:
            self.accepted_proposal = proposal_num
            self.accepted_value = value
            return Accept(accepted=True)
        return Accept(accepted=False)
```

**安全性保证**：

- 所有节点决定相同的值
- 决定的值必须是提议的值

**复杂度分析**：

- **时间复杂度**：$O(1)$（两阶段）
- **消息复杂度**：$O(n)$（每个阶段需要与多数节点通信）
- **容错能力**：可容忍少数节点故障（$n \ge 2f + 1$）

### Multi-Paxos

**优化**：支持一系列值的共识，避免重复准备阶段。

```python
class MultiPaxos:
    def __init__(self, nodes):
        self.nodes = nodes
        self.leader = None
        self.log = []  # 日志条目列表
        self.next_index = 0

    def append(self, value):
        # Leader模式下跳过准备阶段
        if self.is_leader:
            entry = LogEntry(index=self.next_index, value=value)
            self.log.append(entry)
            # 复制到多数节点
            return self.replicate(entry)
        else:
            # 转发给Leader
            return self.forward_to_leader(value)

    def replicate(self, entry):
        # 发送AppendEntries到所有Follower
        acks = 1  # 自己的确认
        for node in self.nodes:
            if node != self:
                if node.append_entries(entry):
                    acks += 1

        if acks >= majority:
            entry.committed = True
            return Success
        return Failure
```

### Paxos变种

| 变种 | 特点 | 优化点 |
|-----|------|--------|
| **Fast Paxos** | 快速路径优化 | 正常情况下只需要一轮消息 |
| **Cheap Paxos** | 减少节点数 | 正常情况需要更少节点 |
| **Flexible Paxos** | 灵活的Quorum | 分离读写Quorum以提高性能 |

## Raft算法

### 核心思想

**提出者**：Diego Ongaro & John Ousterhout（2014年）

Raft算法将共识问题分解为三个子问题：

1. **领导者选举（Leader Election）**
2. **日志复制（Log Replication）**
3. **安全性（Safety）**

**设计目标**：比Paxos更容易理解和实现。

### 节点角色

```python
enum Role:
    LEADER      # 处理所有客户端请求
    FOLLOWER    # 被动接收领导者更新
    CANDIDATE   # 参与选举
```

### 领导者选举

```python
class RaftNode:
    def __init__(self, id, peers):
        self.id = id
        self.peers = peers
        self.role = Role.FOLLOWER
        self.current_term = 0
        self.voted_for = None
        self.log = []

        # 超时定时器
        self.election_timeout = random(150, 300)  # 毫秒
        self.heartbeat_interval = 50  # 毫秒

    def start_election(self):
        """成为候选者并开始选举"""
        self.role = Role.CANDIDATE
        self.current_term += 1
        self.voted_for = self.id

        votes = 1  # 自己的投票

        for peer in self.peers:
            response = peer.request_vote(
                term=self.current_term,
                candidate_id=self.id,
                last_log_index=len(self.log) - 1,
                last_log_term=self.log[-1].term if self.log else 0
            )

            if response.vote_granted:
                votes += 1

        if votes > len(self.peers) / 2:
            self.become_leader()

    def request_vote(self, term, candidate_id, last_log_index, last_log_term):
        """处理投票请求"""
        if term < self.current_term:
            return VoteResponse(term=self.current_term, vote_granted=False)

        if term > self.current_term:
            self.current_term = term
            self.role = Role.FOLLOWER
            self.voted_for = None

        if (self.voted_for is None or self.voted_for == candidate_id) and \
           self.is_log_up_to_date(last_log_index, last_log_term):
            self.voted_for = candidate_id
            return VoteResponse(term=self.current_term, vote_granted=True)

        return VoteResponse(term=self.current_term, vote_granted=False)

    def become_leader(self):
        """成为领导者"""
        self.role = Role.LEADER
        # 初始化next_index和match_index
        self.next_index = {peer: len(self.log) for peer in self.peers}
        self.match_index = {peer: 0 for peer in self.peers}
        # 开始发送心跳
        self.start_heartbeat()
```

### 日志复制

```python
    def append_entries(self, term, leader_id, prev_log_index,
                       prev_log_term, entries, leader_commit):
        """处理AppendEntries RPC"""
        if term < self.current_term:
            return AppendResponse(term=self.current_term, success=False)

        self.current_term = term
        self.role = Role.FOLLOWER

        # 检查日志匹配
        if prev_log_index >= len(self.log) or \
           self.log[prev_log_index].term != prev_log_term:
            return AppendResponse(term=self.current_term, success=False)

        # 追加新条目
        for i, entry in enumerate(entries):
            index = prev_log_index + 1 + i
            if index < len(self.log):
                if self.log[index].term != entry.term:
                    # 删除冲突条目及后续所有条目
                    self.log = self.log[:index]
                    self.log.append(entry)
            else:
                self.log.append(entry)

        # 更新commit_index
        if leader_commit > self.commit_index:
            self.commit_index = min(leader_commit, len(self.log) - 1)

        return AppendResponse(term=self.current_term, success=True)

    def replicate_log(self):
        """领导者复制日志到Follower"""
        for peer in self.peers:
            next_idx = self.next_index[peer]
            entries = self.log[next_idx:]

            response = peer.append_entries(
                term=self.current_term,
                leader_id=self.id,
                prev_log_index=next_idx - 1,
                prev_log_term=self.log[next_idx - 1].term if next_idx > 0 else 0,
                entries=entries,
                leader_commit=self.commit_index
            )

            if response.success:
                self.match_index[peer] = len(self.log) - 1
                self.next_index[peer] = len(self.log)
                self.update_commit_index()
            else:
                self.next_index[peer] -= 1

    def update_commit_index(self):
        """更新commit_index"""
        for n in range(len(self.log) - 1, self.commit_index, -1):
            if self.log[n].term == self.current_term:
                # 统计复制的节点数
                count = 1  # 自己
                for peer in self.peers:
                    if self.match_index[peer] >= n:
                        count += 1

                if count > len(self.peers) / 2:
                    self.commit_index = n
                    break
```

### Raft安全性保证

| 性质 | 表述 |
|-----|------|
| **选举安全性** | 每个任期最多一个领导者 |
| **日志匹配** | 如果两个日志条目有相同的索引和任期，则它们包含相同的命令 |
| **领导者完整性** | 已提交的日志条目在所有更高任期的领导者中 |
| **状态机安全性** | 如果节点已应用某索引的日志条目，则其他节点不会在该索引应用不同的条目 |

### Raft正确性证明概要

**定理**：Raft算法保证安全性和活性。

**安全性证明**：

1. **选举安全性**：
   - 候选者需要获得多数投票才能成为领导者
   - 由于多数集合的交集非空，不可能有两个候选者同时获得多数投票

2. **日志匹配**：
   - 领导者只在多数节点确认后才提交日志条目
   - 通过归纳法证明：如果两个日志在索引i处相同，则在所有小于i的索引处也相同

3. **领导者完整性**：
   - 新领导者必须包含已提交的日志条目
   - 选举限制：只有日志足够新的节点才能成为领导者

**活性证明**：

- 大多数节点正确时，最终会选出领导者
- 领导者会复制日志到大多数节点
- 由于大多数节点正确，最终会达成共识

## Zab协议

**全称**：ZooKeeper Atomic Broadcast

**特点**：

- 为ZooKeeper设计的共识协议
- 保证全局顺序性
- 支持崩溃恢复

**阶段**：

1. **发现（Discovery）**：领导者选举和同步
2. **同步（Synchronization）**：数据同步
3. **广播（Broadcast）**：消息广播

```python
class ZabNode:
    def __init__(self, id):
        self.id = id
        self.epoch = 0  # 领导者周期
        self.counter = 0  # 消息计数器
        self.history = []  # 提议历史

    def propose(self, value):
        """领导者提议"""
        self.counter += 1
        zxid = (self.epoch, self.counter)  # (epoch, counter)
        proposal = Proposal(zxid, value)
        self.history.append(proposal)

        # 发送给所有Follower
        for follower in self.followers:
            follower.receive_proposal(proposal)

        return proposal

    def commit(self, zxid):
        """提交提议"""
        # 等待多数确认
        acks = 1  # 自己
        for follower in self.followers:
            if follower.ack(zxid):
                acks += 1

        if acks > len(self.followers) / 2:
            # 广播提交
            for follower in self.followers:
                follower.receive_commit(zxid)
            return True
        return False
```

## PBFT（Practical Byzantine Fault Tolerance）

**提出者**：Castro & Liskov（1999年）

**适用场景**：拜占庭故障（恶意节点）

**容错要求**：$n \ge 3f + 1$（容忍 $f$ 个拜占庭节点）

**算法阶段**：

```python
class PBFTNode:
    def __init__(self, id, nodes, f):
        self.id = id
        self.nodes = nodes
        self.f = f  # 最大容错数
        self.view = 0  # 视图号
        self.sequence = 0  # 序列号

    def client_request(self, request):
        """处理客户端请求（主节点）"""
        if not self.is_primary:
            return

        self.sequence += 1

        # 1. Pre-prepare阶段
        pre_prepare_msg = PrePrepare(
            view=self.view,
            sequence=self.sequence,
            digest=hash(request)
        )
        self.broadcast(pre_prepare_msg)

        # 2. Prepare阶段
        self.wait_for_prepares(pre_prepare_msg)

        # 3. Commit阶段
        self.wait_for_commits(pre_prepare_msg)

        # 执行请求
        self.execute(request)

        # 4. Reply阶段
        return Reply(result=self.execute(request))

    def receive_pre_prepare(self, msg):
        """处理Pre-prepare消息"""
        if self.validate_pre_prepare(msg):
            prepare_msg = Prepare(
                view=msg.view,
                sequence=msg.sequence,
                digest=msg.digest,
                node_id=self.id
            )
            self.broadcast(prepare_msg)

    def wait_for_prepares(self, msg):
        """等待2f个Prepare消息"""
        prepares = []
        for node in self.nodes:
            if node.id != self.id:
                prepare = node.receive_prepare(msg)
                if prepare and prepare.digest == msg.digest:
                    prepares.append(prepare)

        if len(prepares) >= 2 * self.f:
            # 进入Prepared状态
            self.prepared.add(msg.sequence)

            # 发送Commit消息
            commit_msg = Commit(
                view=msg.view,
                sequence=msg.sequence,
                node_id=self.id
            )
            self.broadcast(commit_msg)

    def wait_for_commits(self, msg):
        """等待2f+1个Commit消息"""
        commits = []
        for node in self.nodes:
            if node.id != self.id:
                commit = node.receive_commit(msg)
                if commit:
                    commits.append(commit)

        if len(commits) >= 2 * self.f:
            # 进入Committed状态，可以执行
            return True
        return False
```

**复杂度分析**：

- **消息复杂度**：$O(n^2)$
- **时间复杂度**：$O(1)$（无视图切换）
- **容错能力**：可容忍 $f$ 个拜占庭节点（$n \ge 3f + 1$）

## Paxos vs Raft 详细对比

| 对比维度 | Paxos | Raft |
|---------|-------|------|
| **核心思想** | 多数派投票 | 强领导者 + 日志复制 |
| **角色复杂度** | 三种角色（Proposer、Acceptor、Learner） | 三种状态（Leader、Follower、Candidate） |
| **日志管理** | 隐式，只关注单值共识 | 显式，维护完整日志 |
| **成员变更** | 复杂 | 支持联合共识 |
| **实现难度** | 高 | 低 |
| **性能优化** | 需要额外优化（Fast Paxos） | Leader模式天然优化 |
| **调试难度** | 高 | 低 |
| **工业应用** | Chubby、Spanner | etcd、Consul、TiKV |
