## 📐 思维表征标准

### 思维导图标准

**格式**：使用Mermaid mindmap语法

**内容要求**：

- 根节点：理论模型名称
- 一级节点：主要知识领域（概念、定义、定理、算法、工具、应用）
- 二级节点：具体知识点
- 三级节点：详细说明

**示例结构**：

```mermaid
mindmap
  root((理论模型))
    核心概念
      概念1
      概念2
    形式化定义
      定义1
      定义2
    性质定理
      定理1
      定理2
```

### 多维知识对比矩阵标准

**格式**：Markdown表格

**维度要求**：

- 至少3个对比维度
- 每个维度有明确的评分标准
- 包含对比说明和适用场景

**示例结构**：

| 理论模型 | 表达能力 | 复杂度 | 工具支持 | 适用场景 |
|---------|---------|--------|---------|---------|
| 模型A | ⭐⭐⭐⭐⭐ | 高 | 丰富 | 场景1, 场景2 |
| 模型B | ⭐⭐⭐⭐ | 中 | 中等 | 场景2, 场景3 |

### 论证决策树标准

**格式**：使用Mermaid flowchart语法

**结构要求**：

- 根节点：决策起点
- 中间节点：判断条件
- 叶子节点：决策结果
- 每个分支有明确的判断标准

**示例结构**：

```mermaid
flowchart TD
    A[决策起点] --> B{判断条件1}
    B -->|是| C[结果1]
    B -->|否| D{判断条件2}
    D -->|是| E[结果2]
    D -->|否| F[结果3]
```

### 概念属性关系图标准

**格式**：使用Mermaid graph语法

**元素要求**：

- 节点：概念、属性
- 边：关系类型（包含、依赖、等价、蕴含等）
- 标签：关系说明

**示例结构**：

```mermaid
graph TD
    A[概念1] -->|包含| B[属性1]
    A -->|依赖| C[概念2]
    C -->|蕴含| D[属性2]
```

### 形式化证明流程图标准

**格式**：使用Mermaid flowchart语法

**步骤要求**：

- 清晰的证明步骤
- 每个步骤的输入输出
- 关键引理和定理的引用
- 证明策略说明
