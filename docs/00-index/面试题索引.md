## 📖 答题技巧

### 面试题答题框架 (4S法)

```
【1. 需求分析】(2分钟)
   - 功能性需求：核心功能是什么？
   - 非功能性需求：QPS？延迟？一致性要求？

【2. 流量估算】(1分钟)
   - DAU、峰值QPS、数据存储量

【3. 高层设计】(5分钟)
   - 画出系统架构图
   - 说明各组件职责

【4. 详细设计】(10分钟)
   - 核心流程：数据如何流动？
   - 数据模型：关键表结构
   - 关键问题：如何解决一致性、高并发等问题？

【5. 扩展讨论】(2分钟)
   - 如何扩展支持10倍增长？
   - 单点故障如何处理？
```

### 加分关键词

| 类别 | 关键词 |
|:---|:---|
| **架构** | 微服务、事件驱动、CQRS、分层架构 |
| **一致性** | 最终一致性、BASE、CAP、幂等性 |
| **性能** | 缓存、异步、削峰、限流、降级 |
| **可用性** | 熔断、舱壁、故障转移、多活 |
| **扩展** | 无状态、分片、复制、负载均衡 |
