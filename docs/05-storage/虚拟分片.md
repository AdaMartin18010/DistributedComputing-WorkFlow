## 📋 执行摘要

虚拟分片（Virtual Sharding）是一种逻辑分片与物理节点解耦的架构设计，通过引入虚拟分片层实现数据的逻辑划分。这种设计使得扩容时只需迁移虚拟分片而非实际数据，大大简化了扩容操作并减少了数据迁移量。

## 一、核心概念

### 1.1 虚拟分片架构

```
┌─────────────────────────────────────────────────────────────┐
│                    虚拟分片架构                              │
├─────────────────────────────────────────────────────────────┤
│                                                            │
│  三层架构：                                                  │
│  ┌───────────────────────────────────────────────────────┐ │
│  │  1. 应用层 (Application)                               │ │
│  │     ├─ 按虚拟分片路由                                  │ │
│  │     └─ shard_key → virtual_shard_id                   │ │
│  ├───────────────────────────────────────────────────────┤ │
│  │  2. 路由层 (Router)                                    │ │
│  │     ├─ 虚拟分片到物理节点映射表                          │ │
│  │     ├─ virtual_shard_id → physical_node               │ │
│  │     └─ 支持动态重新映射                                 │ │
│  ├───────────────────────────────────────────────────────┤ │
│  │  3. 存储层 (Storage)                                   │ │
│  │     ├─ 物理节点1: 包含虚拟分片0,1,2,3                  │ │
│  │     ├─ 物理节点2: 包含虚拟分片4,5,6,7                  │ │
│  │     └─ 物理节点3: 包含虚拟分片8,9,10,11                │ │
│  └───────────────────────────────────────────────────────┘ │
│                                                            │
│  对比：                                                      │ │
│  ┌───────────────────────────────────────────────────────┐ │
│  │  传统分片                   虚拟分片                     │ │
│  │  ├─ 分片数 = 物理节点数    ├─ 虚拟分片数 >> 物理节点数   │ │
│  │  ├─ 扩容需迁移数据         ├─ 扩容只迁移虚拟分片映射      │ │
│  │  ├─ 数据迁移量大           ├─ 数据迁移量小               │ │
│  │  └─ 扩容需停服或双写       └─ 可在线平滑扩容             │ │
│  └───────────────────────────────────────────────────────┘ │
│                                                            │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 虚拟分片数量设计

| 场景 | 虚拟分片数 | 物理节点数 | 比例 |
|------|------------|------------|------|
| 小型系统 | 256 | 4 | 64:1 |
| 中型系统 | 1024 | 16 | 64:1 |
| 大型系统 | 4096 | 64 | 64:1 |

## 二、技术细节

### 2.1 映射算法

```
┌─────────────────────────────────────────────────────────────┐
│                    虚拟分片映射算法                          │
├─────────────────────────────────────────────────────────────┤
│                                                            │
│  1. 路由计算：                                               │ │
│  ┌───────────────────────────────────────────────────────┐ │
│  │  virtual_shard_id = hash(shard_key) % VIRTUAL_SHARD_COUNT│ │
│  │                                                       │ │
│  │  physical_node = lookup_mapping(virtual_shard_id)     │ │
│  │                                                       │ │
│  │  例:                                                  │ │
│  │  ├─ VIRTUAL_SHARD_COUNT = 1024                        │ │
│  │  ├─ user_id=12345                                     │ │
│  │  ├─ hash(12345) = 987654321                           │ │
│  │  ├─ virtual_shard_id = 987654321 % 1024 = 177         │ │
│  │  └─ lookup(177) → Node 3                              │ │
│  └───────────────────────────────────────────────────────┘ │
│                                                            │
│  2. 映射表设计：                                             │ │
│  ┌───────────────────────────────────────────────────────┐ │
│  │  CREATE TABLE shard_mapping (                         │ │
│  │      virtual_shard_id INT PRIMARY KEY,                │ │
│  │      physical_node_id INT NOT NULL,                   │ │
│  │      status ENUM('ACTIVE', 'MIGRATING'),              │ │
│  │      version INT DEFAULT 1,                           │ │
│  │      updated_at TIMESTAMP                             │ │
│  │  );                                                   │ │
│  │                                                       │ │
│  │  -- 索引加速查询                                      │ │
│  │  CREATE INDEX idx_node ON shard_mapping(physical_node_id);│ │
│  └───────────────────────────────────────────────────────┘ │
│                                                            │
│  3. 扩容流程：                                               │ │
│  ┌───────────────────────────────────────────────────────┐ │
│  │  Step 1: 添加新物理节点                                │ │
│  │  Step 2: 计算需要迁移的虚拟分片                          │ │
│  │  Step 3: 逐批迁移虚拟分片（背景任务）                    │ │
│  │          ├─ 双写阶段: 新旧节点都写入                     │ │
│  │          ├─ 数据同步: 复制历史数据                       │ │
│  │          ├─ 校验: 确认数据一致                          │ │
│  │          └─ 切换: 更新映射表，切读取流量                 │ │
│  │  Step 4: 清理旧节点数据（延迟删除）                      │ │
│  └───────────────────────────────────────────────────────┘ │
│                                                            │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 扩容示例

```
┌─────────────────────────────────────────────────────────────┐
│                    虚拟分片扩容示例                          │
├─────────────────────────────────────────────────────────────┤
│                                                            │
│  扩容前：                                                    │
│  ┌───────────────────────────────────────────────────────┐ │
│  │  Physical Node A: Virtual Shards [0-511]              │ │
│  │  Physical Node B: Virtual Shards [512-1023]           │ │
│  └───────────────────────────────────────────────────────┘ │
│                                                            │
│  添加节点 C 后：                                             │
│  ┌───────────────────────────────────────────────────────┐ │
│  │  目标：每个节点约341个虚拟分片（均匀分布）               │ │
│  │                                                       │ │
│  │  迁移计划：                                            │ │
│  │  ├─ Node A 保留: [0-341]                              │ │
│  │  ├─ Node A 迁出: [342-511] → Node C                   │ │
│  │  ├─ Node B 保留: [512-853]                            │ │
│  │  └─ Node B 迁出: [854-1023] → Node C                  │ │
│  │                                                       │ │
│  │  Node C 接收: [342-511] ∪ [854-1023] = 340 shards     │ │
│  │                                                       │ │
│  │  数据迁移比例：约 33% 的数据需要迁移                    │ │
│  └───────────────────────────────────────────────────────┘ │
│                                                            │
│  对比传统Hash：                                              │
│  ├─ Hash % 2 → Hash % 3 需迁移约 66% 数据                  │ │
│  ├─ 虚拟分片只需迁移 33% 数据                               │ │
│  └─ 虚拟分片越多，迁移比例越低                              │ │
│                                                            │
└─────────────────────────────────────────────────────────────┘
```

## 三、实践指南

### 3.1 代码实现

```java
@Component
public class VirtualShardRouter {

    private static final int VIRTUAL_SHARD_COUNT = 1024;

    @Autowired
    private ShardMappingRepository mappingRepo;

    private final ConcurrentHashMap<Integer, Integer> cache =
        new ConcurrentHashMap<>();

    /**
     * 计算虚拟分片ID
     */
    public int getVirtualShardId(String shardKey) {
        int hash = Hashing.murmur3_32().hashString(shardKey, UTF_8).asInt();
        return Math.abs(hash) % VIRTUAL_SHARD_COUNT;
    }

    /**
     * 获取物理节点
     */
    public String getPhysicalNode(String shardKey) {
        int virtualShardId = getVirtualShardId(shardKey);

        // 本地缓存
        Integer nodeId = cache.get(virtualShardId);
        if (nodeId != null) {
            return "node-" + nodeId;
        }

        // 查询映射表
        ShardMapping mapping = mappingRepo.findById(virtualShardId)
            .orElseThrow();

        cache.put(virtualShardId, mapping.getPhysicalNodeId());
        return "node-" + mapping.getPhysicalNodeId();
    }

    /**
     * 批量迁移虚拟分片（管理接口）
     */
    @Transactional
    public void migrateVirtualShards(List<Integer> shardIds, int targetNodeId) {
        for (int shardId : shardIds) {
            // 1. 更新状态为MIGRATING
            mappingRepo.updateStatus(shardId, "MIGRATING");

            // 2. 执行数据迁移（异步任务）
            dataMigrationService.migrate(shardId, targetNodeId);

            // 3. 更新映射
            mappingRepo.updateNode(shardId, targetNodeId, "ACTIVE");

            // 4. 刷新缓存
            cache.remove(shardId);
        }
    }
}
```
