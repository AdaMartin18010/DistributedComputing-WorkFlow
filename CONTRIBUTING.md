# 贡献指南

感谢您对本项目的关注！我们欢迎各种形式的贡献。

## 如何贡献

### 报告问题

如果您发现了问题或有改进建议，请：

1. 检查[常见问题解答](docs/11-faq/常见问题解答.md)是否已有解答
2. 在GitHub Issues中搜索是否已有相关问题
3. 如果没有，请创建新的Issue，包含：
   - 问题描述
   - 复现步骤
   - 预期行为
   - 实际行为
   - 环境信息

### 提交代码

1. **Fork项目**

   ```bash
   git clone https://github.com/your-username/DistributedComputing-WorkFlow.git
   cd DistributedComputing-WorkFlow
   ```

2. **创建分支**

   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **进行修改**
   - 遵循项目代码风格
   - 添加必要的注释
   - 更新相关文档

4. **提交更改**

   ```bash
   git add .
   git commit -m "描述您的更改"
   ```

5. **推送并创建Pull Request**

   ```bash
   git push origin feature/your-feature-name
   ```

### 改进文档

文档改进同样重要！您可以：

1. 修正拼写和语法错误
2. 改进文档结构
3. 添加更多示例
4. 翻译文档
5. 补充缺失的内容

### 添加案例

我们欢迎添加更多企业实践案例：

1. 确保案例真实可靠
2. 包含详细的业务场景描述
3. 提供性能数据和关键指标
4. 说明技术选型理由

## 代码规范

### 文档规范

- 使用Markdown格式
- 遵循现有文档结构
- 保持一致的格式风格
- 添加必要的图表和代码示例

### 提交信息规范

提交信息应清晰描述更改内容：

```text
类型: 简短描述

详细描述（可选）

相关Issue: #123
```

**类型**：

- `feat`: 新功能
- `fix`: 修复问题
- `docs`: 文档更新
- `style`: 格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 其他更改

## 审查流程

1. 提交Pull Request后，项目维护者会进行审查
2. 可能需要根据反馈进行修改
3. 审查通过后，更改会被合并到主分支

## 行为准则

- 尊重所有贡献者
- 接受建设性批评
- 关注对项目最有利的事情
- 对其他社区成员表示同理心

## 联系方式

如有问题，请通过以下方式联系：

- GitHub Issues
- Pull Request评论
- 项目讨论区

感谢您的贡献！
