### 关系类型说明

#### 强关系（强度：强）

- **生成**：概念A生成概念B，B是A的直接产物
- **满足**：概念A满足概念B，A具有B的性质
- **包含**：概念A包含概念B，B是A的组成部分
- **基于**：概念A基于概念B，A建立在B的基础上
- **依赖**：概念A依赖概念B，A的实现需要B
- **互斥**：概念A和B互斥，不能同时满足
- **假设**：概念A假设概念B，A的前提是B
- **强于**：概念A强于概念B，A的性质比B更强
- **保证**：概念A保证概念B，A确保B成立
- **组成**：概念A由概念B组成，B是A的组成部分

#### 中关系（强度：中）

- **选择**：概念A选择概念B，A可以选择B
- **绕过**：概念A绕过概念B，A避免了B的限制

## 📊 查询示例

### 查询1：查找概念的所有属性

```sql
SELECT a.属性名称, a.属性类型, a.属性值, a.说明
FROM 属性表 a
WHERE a.所属概念 = '状态（State）'
```

**结果**：

- 变量赋值（结构属性）
- 完整快照（结构属性）

### 查询2：查找概念的所有关系

```sql
SELECT r.关系类型, r.目标概念, r.关系说明
FROM 关系表 r
WHERE r.源概念 = '状态（State）'
```

**结果**：

- 生成 → 动作
- 包含 → 变量赋值
- 包含 → 完整快照

### 查询3：查找概念的所有依赖

```sql
SELECT r.源概念, r.关系类型, r.目标概念
FROM 关系表 r
WHERE r.关系类型 = '依赖' AND r.源概念 = 'CTL'
```

**结果**：

- CTL → 依赖 → Kripke结构
