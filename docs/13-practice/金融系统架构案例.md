## 一、核心概念

### 1.1 金融系统架构定义

金融系统是最高级别的分布式系统，对数据一致性、系统可用性、安全合规有极致要求。其核心特征包括：

- **强一致性**：资金操作必须保证ACID特性
- **高可用性**：5个9（99.999%）可用性要求
- **安全合规**：满足央行、银保监会等监管要求
- **可追溯性**：所有操作全程留痕、可审计

### 1.2 关键特性

- **双活架构**：异地多活部署，RPO≈0，RTO<分钟级
- **分户记账**：精细化的科目体系和账户体系
- **实时风控**：毫秒级风险识别和拦截
- **数据加密**：全链路加密传输和存储

### 1.3 适用场景

| 场景 | 适用性 | 说明 |
|------|--------|------|
| 银行核心系统 | ⭐⭐⭐⭐⭐ | 存款、贷款、支付结算 |
| 证券交易系统 | ⭐⭐⭐⭐⭐ | 股票、基金交易 |
| 保险核心系统 | ⭐⭐⭐⭐ | 保单、理赔、精算 |
| 第三方支付 | ⭐⭐⭐⭐ | 钱包、快捷支付 |
| 供应链金融 | ⭐⭐⭐⭐ | 产业金融、贸易融资 |

### 3.3 一致性保障

#### 3.3.1 资金操作原则

| 原则 | 说明 | 实现方式 |
|------|------|----------|
| 先记账后付款 | 确保付款方余额充足 | 本地事务 |
| 先收款后记账 | 确保资金已到账 | 外部确认 |
| 借贷必相等 | 会计恒等式 | 交易级校验 |
| 当日事当日毕 | 日终必须平衡 | 批量检查 |

#### 3.3.2 TCC分布式事务

```
TCC模式应用于跨行转账：

Try阶段：
┌─────────┐    ┌─────────┐    ┌─────────┐
│  冻结   │───→│  校验   │───→│  预扣   │
│付款方资金│    │收款方账户│    │  头寸   │
└─────────┘    └─────────┘    └─────────┘
     │              │              │
     ▼              ▼              ▼
  冻结成功       校验通过       头寸充足

Confirm阶段（全部成功）：
实际扣减付款方资金 ──→ 实际增加收款方资金 ──→ 释放冻结

Cancel阶段（任一步失败）：
解冻付款方资金 ──→ 回滚所有操作
```

#### 4.2.1 风控规则示例

```java
// 规则DSL定义
public class FraudRules {

    // 规则1：高频交易检测
    Rule highFrequencyRule = Rule.builder()
        .name("high_frequency_trading")
        .when(ctx -> {
            String userId = ctx.getUserId();
            // 1分钟内交易次数>5
            return getTradeCount(userId, 60) > 5;
        })
        .then(ctx -> {
            ctx.addRiskScore(50);
            ctx.setAction(Action.REVIEW);
        })
        .build();

    // 规则2：异地登录交易
    Rule geoLocationRule = Rule.builder()
        .name("abnormal_location")
        .when(ctx -> {
            Location lastLoc = getLastLoginLocation(ctx.getUserId());
            Location currentLoc = ctx.getLocation();
            // 距离超过500公里且时间小于2小时
            return distance(lastLoc, currentLoc) > 500
                && timeDiff(lastLoc, currentLoc) < 2 * 3600;
        })
        .then(ctx -> {
            ctx.addRiskScore(80);
            ctx.setAction(Action.CHALLENGE);
        })
        .build();

    // 规则3：黑名单命中
    Rule blacklistRule = Rule.builder()
        .name("blacklist_hit")
        .when(ctx -> {
            return blacklistService.contains(
                ctx.getUserId(),
                ctx.getDeviceId(),
                ctx.getCardNo()
            );
        })
        .then(ctx -> {
            ctx.addRiskScore(100);
            ctx.setAction(Action.REJECT);
        })
        .build();
}
```

#### 4.2.2 Flink实时计算

```java
// 实时特征计算
public class RiskFeatureJob {

    public static void main(String[] args) {
        StreamExecutionEnvironment env =
            StreamExecutionEnvironment.getExecutionEnvironment();

        // 交易流
        DataStream<Transaction> transactions = env
            .addSource(new KafkaSource<>())
            .assignTimestampsAndWatermarks(
                WatermarkStrategy.forBoundedOutOfOrderness(
                    Duration.ofSeconds(5)
                )
            );

        // 滑动窗口统计：用户近1小时交易金额
        DataStream<Feature> hourAmountFeature = transactions
            .keyBy(Transaction::getUserId)
            .window(SlidingEventTimeWindows.of(
                Time.hours(1),
                Time.minutes(1)
            ))
            .aggregate(new SumAggregate())
            .map(result -> new Feature("hour_amount", result));

        // 会话窗口：检测设备共用
        DataStream<Alert> deviceSharingAlert = transactions
            .keyBy(Transaction::getDeviceId)
            .window(EventTimeSessionWindows.withDynamicGap(
                (element) -> Time.minutes(30)
            ))
            .process(new DeviceSharingDetectFunction());

        env.execute("Risk Feature Calculation");
    }
}
```

## 六、系统对比

### 6.1 金融系统vs普通电商

| 维度 | 金融系统 | 电商系统 |
|------|----------|----------|
| 一致性要求 | 强一致性(ACID) | 最终一致性 |
| 可用性要求 | 5个9(99.999%) | 3-4个9 |
| 数据保留 | 永久保留 | 通常3-7年 |
| 审计要求 | 全量审计追踪 | 关键操作审计 |
| 监管要求 | 严格监管 | 相对宽松 |
| 灾备RTO | <分钟级 | 小时级 |
| 灾备RPO | ≈0 | 分钟级 |

### 6.2 技术选型

| 组件类型 | 选型 | 理由 |
|----------|------|------|
| 数据库 | DB2/Oracle | 金融级稳定性，支持强一致性 |
| 中间件 | IBM MQ | 金融行业标准，高可靠 |
| 事务 | TCC/Saga | 支持长时间事务 |
| 缓存 | Redis集群 | 高性能，支持持久化 |
| 计算 | Flink | 实时计算能力强 |
| 存储 | 对象存储 | 不可篡改，满足合规 |
