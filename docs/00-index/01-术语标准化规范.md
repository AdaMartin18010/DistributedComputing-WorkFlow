# 术语定义标准模板

> 📋 分布式计算知识库术语标准化规范 | 版本: 1.0

---

## 目的

本文档定义了分布式计算知识库中所有术语的标准化描述格式，确保术语定义的一致性、完整性和可追溯性。

---

## 术语定义标准模板

每个术语必须包含以下8个核心部分：

### 1. 中文名称 (Chinese Name)

- **标准中文译名**：使用业界公认的标准译名
- **命名原则**：
  - 优先使用学术文献中的标准译法
  - 次选主流技术文档中的通用译法
  - 避免生造词和过度意译
  - 保持与相关术语的一致性

### 2. 英文名称 (English Name)

- **英文原名**：使用粗体表示，如 **Consensus**
- **缩写形式**：如有常用缩写，用括号标注，如 **MVCC** (Multi-Version Concurrency Control)
- **大小写规范**：
  - 专有名词首字母大写
  - 普通术语小写（除非缩写）
  - 保留原始文献的大小写习惯

### 3. 定义（Definition）

- **字数限制**：100-200字
- **内容要求**：
  - 严格、准确的学术定义
  - 说明该术语在分布式系统中的特定含义
  - 避免循环定义
  - 使用已定义的术语解释新概念
- **写作风格**：
  - 第三人称客观叙述
  - 使用定义性语言（"是..."、"指..."）
  - 避免比喻和类比

### 4. 属性（Attributes）

每个术语必须标注以下属性：

#### 4.1 类型 (Type)

| 类型值 | 说明 | 示例 |
|:---|:---|:---|
| 概念 | 抽象的理论概念 | 一致性、分区容忍性 |
| 算法 | 具体的计算方法 | Paxos、Raft |
| 协议 | 通信或协调规则 | 2PC、 gossip协议 |
| 系统 | 完整的系统实现 | Spanner、ZooKeeper |
| 模型 | 理论或计算模型 | FLP模型、状态机 |
| 属性 | 系统特性 | 可用性、容错性 |
| 机制 | 实现技术 | 心跳机制、租约机制 |
| 模式 | 设计模式 | 主从模式、无状态模式 |

#### 4.2 层次 (Level)

| 层次值 | 说明 | 示例 |
|:---|:---|:---|
| 理论 | 纯理论研究 | CAP定理、FLP不可能性 |
| 算法 | 算法层面 | 共识算法、选举算法 |
| 实现 | 具体实现 | etcd、Kafka |
| 应用 | 工程应用 | 微服务、容器编排 |

#### 4.3 相关主题 (Topics)

- 使用主题域标签进行分类
- 可多选，用逗号分隔
- 标准主题域见"主题域分类表"

### 5. 形式化描述（Formal Specification）

- **适用情况**：算法、协议、模型类术语
- **表示方式**：
  - 数学符号表示（集合论、逻辑、代数）
  - 伪代码（复杂算法）
  - 状态转换图（状态机）
  - 时序图（协议交互）
- **要求**：与定义保持一致，可用于形式化验证

### 6. 示例（Examples）

- **具体内容**：
  - 具体系统或场景中的应用实例
  - 代码片段（如适用）
  - 时序示例
- **要求**：
  - 示例必须与定义相符
  - 覆盖典型用例
  - 标注出处（如来自特定论文/系统）

### 7. 相关术语（Related Terms）

必须包含以下关系类型：

- **同义词**：完全等价的不同名称
- **近义词**：含义相近但有细微差别
- **反义词**：含义相反的术语
- **上位词**：更宽泛的概念
- **下位词**：更具体的概念
- **关联词**：经常一起使用的术语

### 8. 引用（References）

- **权威出处**：
  - 学术论文（第一出处）
  - 技术标准（IEEE/ISO等）
  - 经典教材
  - 官方文档
- **引用格式**：
  - 作者, 年份, 论文/书籍标题
  - 链接到知识库内相关专题文档

## 质量控制清单

在提交新术语定义前，请确认：

- [ ] 中文名称符合标准译法
- [ ] 英文名称拼写正确
- [ ] 定义字数在100-200字范围内
- [ ] 定义准确、无歧义
- [ ] 属性（类型、层次、主题）已标注
- [ ] 形式化描述与文字定义一致（如适用）
- [ ] 示例具体且可验证
- [ ] 相关术语关系完整
- [ ] 引用来源权威可追溯
- [ ] 无与现有术语定义的冲突
