## Temporal

### 概述

Temporal 是一个持久化执行平台，采用**代码即工作流**（Workflow as Code）的理念，支持多种编程语言。

### 1. Hello World 示例

#### 活动定义 (Go)

```go
// activities.go
package app

import (
    "context"
    "fmt"
)

// GreeterActivity 实现简单的问候活动
// 这是 Temporal 中最基本的 Activity 定义
type GreeterActivity struct{}

// Greet 执行问候逻辑
// 参数 ctx: Activity 上下文，用于取消信号和心跳
// 参数 name: 要问候的对象名称
// 返回: 问候语和可能的错误
func (a *GreeterActivity) Greet(ctx context.Context, name string) (string, error) {
    // 模拟一些业务逻辑处理
    greeting := fmt.Sprintf("Hello, %s! Welcome to Temporal!", name)

    // 可以在这里添加日志、指标采集等
    fmt.Printf("Activity executed: greeting=%s\n", greeting)

    return greeting, nil
}
```

#### 工作流定义 (Go)

```go
// workflow.go
package app

import (
    "fmt"
    "time"

    "go.temporal.io/sdk/workflow"
)

// HelloWorldWorkflow 是最简单的工作流示例
// 演示基本的 Activity 调用和结果处理
func HelloWorldWorkflow(ctx workflow.Context, name string) (string, error) {
    // 创建 Activity 选项
    // StartToCloseTimeout: Activity 从开始到完成的最大时间
    ao := workflow.ActivityOptions{
        StartToCloseTimeout: 10 * time.Second,
    }
    ctx = workflow.WithActivityOptions(ctx, ao)

    var result string

    // 执行 Activity
    // ExecuteActivity 是异步的，会立即返回 Future
    // Get 方法阻塞等待 Activity 完成
    err := workflow.ExecuteActivity(ctx, "GreeterActivity.Greet", name).Get(ctx, &result)
    if err != nil {
        return "", fmt.Errorf("activity execution failed: %w", err)
    }

    return result, nil
}
```

#### Worker 启动代码 (Go)

```go
// main.go
package main

import (
    "log"

    "go.temporal.io/sdk/client"
    "go.temporal.io/sdk/worker"

    "your-module/app"
)

func main() {
    // 创建 Temporal 客户端连接
    // 默认连接 localhost:7233
    c, err := client.NewClient(client.Options{})
    if err != nil {
        log.Fatalf("Failed to create client: %v", err)
    }
    defer c.Close()

    // 创建 Worker
    // "hello-world-queue" 是任务队列名称
    w := worker.New(c, "hello-world-queue", worker.Options{})

    // 注册工作流和活动
    w.RegisterWorkflow(app.HelloWorldWorkflow)
    w.RegisterActivity(&app.GreeterActivity{})

    // 启动 Worker（阻塞调用）
    err = w.Run(worker.InterruptCh())
    if err != nil {
        log.Fatalf("Failed to start worker: %v", err)
    }
}
```

#### 客户端启动代码 (Go)

```go
// starter.go
package main

import (
    "context"
    "fmt"
    "log"

    "go.temporal.io/sdk/client"
)

func main() {
    c, err := client.NewClient(client.Options{})
    if err != nil {
        log.Fatalf("Failed to create client: %v", err)
    }
    defer c.Close()

    // 启动工作流执行
    workflowOptions := client.StartWorkflowOptions{
        ID:        "hello-world-workflow",
        TaskQueue: "hello-world-queue",
    }

    we, err := c.ExecuteWorkflow(context.Background(), workflowOptions,
        "HelloWorldWorkflow", "Temporal User")
    if err != nil {
        log.Fatalf("Failed to start workflow: %v", err)
    }

    // 获取工作流结果
    var result string
    err = we.Get(context.Background(), &result)
    if err != nil {
        log.Fatalf("Failed to get workflow result: %v", err)
    }

    fmt.Printf("Workflow result: %s\n", result)
}
```

### 2. 复杂订单处理工作流

```go
// order_workflow.go
package app

import (
    "fmt"
    "time"

    "go.temporal.io/sdk/workflow"
)

// OrderState 表示订单状态
type OrderState string

const (
    OrderStatePending    OrderState = "PENDING"
    OrderStatePaid       OrderState = "PAID"
    OrderStateInventory  OrderState = "INVENTORY_CHECKED"
    OrderStateShipped    OrderState = "SHIPPED"
    OrderStateCompleted  OrderState = "COMPLETED"
    OrderStateCancelled  OrderState = "CANCELLED"
)

// OrderRequest 订单请求结构
type OrderRequest struct {
    OrderID    string
    CustomerID string
    Items      []OrderItem
    TotalAmount float64
}

type OrderItem struct {
    ProductID string
    Quantity  int
    Price     float64
}

// OrderResult 订单处理结果
type OrderResult struct {
    OrderID   string
    State     OrderState
    Message   string
    Timestamp time.Time
}

// OrderProcessingWorkflow 实现完整的订单处理Saga
// 包含：支付 -> 库存扣减 -> 物流安排 -> 通知
// 任意步骤失败时执行补偿操作
func OrderProcessingWorkflow(ctx workflow.Context, req OrderRequest) (*OrderResult, error) {
    // 定义 Saga 补偿操作列表
    var compensations []func() error

    // 确保补偿操作在工作流取消或失败时执行
    defer func() {
        if ctx.Err() != nil {
            // 执行所有补偿操作
            for i := len(compensations) - 1; i >= 0; i-- {
                _ = compensations[i]()
            }
        }
    }()

    // 步骤1: 处理支付
    paymentID, err := processPayment(ctx, req)
    if err != nil {
        return nil, fmt.Errorf("payment failed: %w", err)
    }
    // 注册支付补偿
    compensations = append(compensations, func() error {
        return refundPayment(ctx, paymentID)
    })

    // 步骤2: 检查并扣减库存
    reservationID, err := reserveInventory(ctx, req)
    if err != nil {
        return nil, fmt.Errorf("inventory reservation failed: %w", err)
    }
    compensations = append(compensations, func() error {
        return releaseInventory(ctx, reservationID)
    })

    // 步骤3: 安排物流
    shipmentID, err := arrangeShipment(ctx, req)
    if err != nil {
        return nil, fmt.Errorf("shipment arrangement failed: %w", err)
    }
    compensations = append(compensations, func() error {
        return cancelShipment(ctx, shipmentID)
    })

    // 步骤4: 发送通知
    err = sendNotification(ctx, req.CustomerID, "Order confirmed!")
    if err != nil {
        // 通知失败不阻塞订单完成
        workflow.GetLogger(ctx).Error("Failed to send notification", "error", err)
    }

    return &OrderResult{
        OrderID:   req.OrderID,
        State:     OrderStateCompleted,
        Message:   "Order processed successfully",
        Timestamp: workflow.Now(ctx),
    }, nil
}

// processPayment 处理支付
func processPayment(ctx workflow.Context, req OrderRequest) (string, error) {
    ctx = workflow.WithActivityOptions(ctx, workflow.ActivityOptions{
        StartToCloseTimeout: 30 * time.Second,
        // 配置重试策略
        RetryPolicy: &temporal.RetryPolicy{
            InitialInterval:    time.Second,
            BackoffCoefficient: 2.0,
            MaximumInterval:    time.Minute,
            MaximumAttempts:    3,
        },
    })

    var paymentID string
    err := workflow.ExecuteActivity(ctx, "PaymentActivity.Process", req).Get(ctx, &paymentID)
    return paymentID, err
}

// refundPayment 退款补偿
func refundPayment(ctx workflow.Context, paymentID string) error {
    ctx = workflow.WithActivityOptions(ctx, workflow.ActivityOptions{
        StartToCloseTimeout: 30 * time.Second,
    })

    return workflow.ExecuteActivity(ctx, "PaymentActivity.Refund", paymentID).Get(ctx, nil)
}

// reserveInventory 预留库存
func reserveInventory(ctx workflow.Context, req OrderRequest) (string, error) {
    ctx = workflow.WithActivityOptions(ctx, workflow.ActivityOptions{
        StartToCloseTimeout: 10 * time.Second,
    })

    var reservationID string
    err := workflow.ExecuteActivity(ctx, "InventoryActivity.Reserve", req.Items).Get(ctx, &reservationID)
    return reservationID, err
}

// releaseInventory 释放库存
func releaseInventory(ctx workflow.Context, reservationID string) error {
    ctx = workflow.WithActivityOptions(ctx, workflow.ActivityOptions{
        StartToCloseTimeout: 10 * time.Second,
    })

    return workflow.ExecuteActivity(ctx, "InventoryActivity.Release", reservationID).Get(ctx, nil)
}

// arrangeShipment 安排物流
func arrangeShipment(ctx workflow.Context, req OrderRequest) (string, error) {
    ctx = workflow.WithActivityOptions(ctx, workflow.ActivityOptions{
        StartToCloseTimeout: 1 * time.Minute,
    })

    var shipmentID string
    err := workflow.ExecuteActivity(ctx, "ShipmentActivity.Arrange", req).Get(ctx, &shipmentID)
    return shipmentID, err
}

// cancelShipment 取消物流
func cancelShipment(ctx workflow.Context, shipmentID string) error {
    ctx = workflow.WithActivityOptions(ctx, workflow.ActivityOptions{
        StartToCloseTimeout: 30 * time.Second,
    })

    return workflow.ExecuteActivity(ctx, "ShipmentActivity.Cancel", shipmentID).Get(ctx, nil)
}

// sendNotification 发送通知
func sendNotification(ctx workflow.Context, customerID string, message string) error {
    ctx = workflow.WithActivityOptions(ctx, workflow.ActivityOptions{
        StartToCloseTimeout: 10 * time.Second,
    })

    return workflow.ExecuteActivity(ctx, "NotificationActivity.Send", customerID, message).Get(ctx, nil)
}
```

### 3. 并行处理和定时器示例

```go
// parallel_workflow.go
package app

import (
    "fmt"
    "time"

    "go.temporal.io/sdk/workflow"
)

// ParallelProcessingWorkflow 演示并行Activity执行
// 场景：同时处理多个独立的数据处理任务
func ParallelProcessingWorkflow(ctx workflow.Context, taskIDs []string) (map[string]string, error) {
    ctx = workflow.WithActivityOptions(ctx, workflow.ActivityOptions{
        StartToCloseTimeout: 5 * time.Minute,
    })

    // 为每个任务创建一个 Future
    futures := make([]workflow.Future, len(taskIDs))
    for i, taskID := range taskIDs {
        // 启动并行Activity执行
        futures[i] = workflow.ExecuteActivity(ctx, "DataProcessingActivity.Process", taskID)
    }

    // 等待所有任务完成
    results := make(map[string]string)
    for i, future := range futures {
        var result string
        if err := future.Get(ctx, &result); err != nil {
            return nil, fmt.Errorf("task %s failed: %w", taskIDs[i], err)
        }
        results[taskIDs[i]] = result
    }

    return results, nil
}

// WorkflowWithTimer 演示定时器使用
// 场景：在指定时间内完成任务，否则超时
func WorkflowWithTimer(ctx workflow.Context, timeout time.Duration) (string, error) {
    ctx = workflow.WithActivityOptions(ctx, workflow.ActivityOptions{
        StartToCloseTimeout: timeout + time.Minute,
    })

    // 创建一个 Selector 用于处理并发事件
    selector := workflow.NewSelector(ctx)

    var activityResult string
    activityFuture := workflow.ExecuteActivity(ctx, "LongRunningActivity.Execute")

    // 设置超时定时器
    timerFuture := workflow.NewTimer(ctx, timeout)

    // 等待 Activity 完成或超时
    selector.AddFuture(activityFuture, func(f workflow.Future) {
        _ = f.Get(ctx, &activityResult)
    })
    selector.AddFuture(timerFuture, func(f workflow.Future) {
        // 超时处理
        activityResult = "TIMEOUT"
    })

    // 等待任一条件满足
    selector.Select(ctx)

    if activityResult == "TIMEOUT" {
        // 尝试取消 Activity
        // 注意：实际的取消依赖于 Activity 实现是否监听 ctx.Done()
        return "", fmt.Errorf("activity timed out after %v", timeout)
    }

    return activityResult, nil
}

// ApprovalWorkflow 演示人工审批工作流
func ApprovalWorkflow(ctx workflow.Context, requestID string) (string, error) {
    ctx = workflow.WithActivityOptions(ctx, workflow.ActivityOptions{
        StartToCloseTimeout: 24 * time.Hour, // 足够长以等待人工操作
    })

    // 发送审批请求
    err := workflow.ExecuteActivity(ctx, "ApprovalActivity.SendRequest", requestID).Get(ctx, nil)
    if err != nil {
        return "", err
    }

    // 等待审批信号
    // Signal 是 Temporal 中实现外部事件触发的主要方式
    approvalCh := workflow.GetSignalChannel(ctx, "approval-response")

    // 同时设置超时
    timer := workflow.NewTimer(ctx, 48*time.Hour)

    selector := workflow.NewSelector(ctx)
    var approvalResult string

    selector.AddReceive(approvalCh, func(c workflow.ReceiveChannel, more bool) {
        var response ApprovalResponse
        c.Receive(ctx, &response)
        approvalResult = response.Decision
    })

    selector.AddFuture(timer, func(f workflow.Future) {
        approvalResult = "TIMEOUT"
    })

    selector.Select(ctx)

    // 记录审批结果
    _ = workflow.ExecuteActivity(ctx, "ApprovalActivity.RecordResult", requestID, approvalResult).Get(ctx, nil)

    return approvalResult, nil
}

type ApprovalResponse struct {
    Decision string // "APPROVED" or "REJECTED"
    Comment  string
    Approver string
}
```

## Apache Airflow

### 概述

Airflow 是 Python 编写的编排平台，使用**有向无环图（DAG）**定义工作流。

### 1. Hello World DAG

```python
# hello_world_dag.py
"""
Airflow Hello World DAG 示例

最佳实践:
- 使用 context manager 定义 DAG（Airflow 2.0+ 推荐）
- 设置合理的 default_args 避免重复配置
- 使用 task_id 命名规范：小写 + 下划线
"""

from datetime import datetime, timedelta
from airflow import DAG
from airflow.operators.python import PythonOperator
from airflow.utils.task_group import TaskGroup

# 默认参数配置
# owner: DAG 所有者
# depends_on_past: 是否依赖上一个运行周期
# retries: 失败重试次数
# retry_delay: 重试间隔
default_args = {
    'owner': 'airflow',
    'depends_on_past': False,
    'email': ['admin@example.com'],
    'email_on_failure': False,
    'email_on_retry': False,
    'retries': 1,
    'retry_delay': timedelta(minutes=5),
}

# 使用 context manager 定义 DAG
# catchup=False: 不补跑历史任务
# tags: 用于 Web UI 分类
with DAG(
    'hello_world_dag',
    default_args=default_args,
    description='A simple hello world DAG',
    schedule_interval=timedelta(days=1),  # 每日执行
    start_date=datetime(2024, 1, 1),
    catchup=False,
    tags=['example', 'hello-world'],
) as dag:

    # 定义 Python 函数任务
    def greet(**context):
        """
        问候任务

        **context 包含:
        - ds: 执行日期 (YYYY-MM-DD)
        - execution_date: 执行时间戳
        - dag_run: 当前 DAG 运行实例
        - task_instance: 当前任务实例
        """
        print(f"Hello from Airflow!")
        print(f"Execution date: {context['ds']}")
        print(f"DAG ID: {context['dag'].dag_id}")

        # 可以推送 XCom 供下游任务使用
        context['ti'].xcom_push(key='greeting', value='Hello World!')

        return 'Greeting completed'

    # 创建任务
    # task_id: 唯一标识符
    # python_callable: 要执行的函数
    hello_task = PythonOperator(
        task_id='hello_task',
        python_callable=greet,
    )

    def process_result(**context):
        """从上游任务获取 XCom 并处理"""
        ti = context['ti']

        # 拉取上游任务的 XCom
        greeting = ti.xcom_pull(task_ids='hello_task', key='greeting')
        print(f"Received from upstream: {greeting}")

        return f"Processed: {greeting}"

    process_task = PythonOperator(
        task_id='process_task',
        python_callable=process_result,
    )

    # 定义任务依赖关系
    # 使用 >> 操作符表示依赖
    hello_task >> process_task
```

### 2. 复杂 ETL 工作流

```python
# complex_etl_dag.py
"""
复杂 ETL 工作流示例

演示功能:
- TaskGroup: 任务分组组织
- BranchPythonOperator: 条件分支
- Sensor: 等待外部条件
- 动态任务生成
"""

from datetime import datetime, timedelta
from airflow import DAG
from airflow.operators.python import PythonOperator, BranchPythonOperator
from airflow.operators.empty import EmptyOperator
from airflow.sensors.filesystem import FileSensor
from airflow.utils.task_group import TaskGroup
from airflow.providers.postgres.operators.postgres import PostgresOperator
import json

# 连接配置
POSTGRES_CONN_ID = 'postgres_default'
S3_CONN_ID = 'aws_default'

default_args = {
    'owner': 'data-engineering',
    'depends_on_past': True,  # 依赖于上一次运行
    'retries': 2,
    'retry_delay': timedelta(minutes=5),
}

with DAG(
    'complex_etl_pipeline',
    default_args=default_args,
    description='Complex ETL pipeline with data quality checks',
    schedule_interval='0 2 * * *',  # 每天凌晨 2 点
    start_date=datetime(2024, 1, 1),
    catchup=False,
    max_active_runs=1,  # 限制同时运行的实例数
    tags=['etl', 'production'],
) as dag:

    # 开始节点
    start = EmptyOperator(task_id='start')

    # 1. 等待源数据文件
    # 使用 FileSensor 轮询检查文件是否存在
    wait_for_source_file = FileSensor(
        task_id='wait_for_source_file',
        filepath='/data/incoming/daily_{{ ds }}.csv',
        fs_conn_id='fs_default',
        poke_interval=300,  # 每 5 分钟检查一次
        timeout=3600,       # 最多等待 1 小时
        mode='reschedule',  # 不占用 worker slot
    )

    # 2. 数据提取分组
    with TaskGroup('extract_group', tooltip='Data extraction tasks') as extract_group:

        def extract_from_db(**context):
            """从数据库提取数据"""
            from airflow.providers.postgres.hooks.postgres import PostgresHook

            pg_hook = PostgresHook(postgres_conn_id=POSTGRES_CONN_ID)
            sql = """
                SELECT * FROM raw_data
                WHERE created_date = '{{ ds }}'
            """
            records = pg_hook.get_records(sql)

            # 保存到 XCom
            context['ti'].xcom_push(key='db_records', value=len(records))
            return f"Extracted {len(records)} records from database"

        extract_db = PythonOperator(
            task_id='extract_from_database',
            python_callable=extract_from_db,
        )

        def extract_from_api(**context):
            """从 API 提取数据"""
            import requests

            # 实际生产中使用 Airflow Connection 存储凭证
            response = requests.get(
                'https://api.example.com/data',
                params={'date': context['ds']}
            )
            response.raise_for_status()
            data = response.json()

            context['ti'].xcom_push(key='api_records', value=len(data))
            return f"Extracted {len(data)} records from API"

        extract_api = PythonOperator(
            task_id='extract_from_api',
            python_callable=extract_from_api,
        )

        # 并行提取
        [extract_db, extract_api]

    # 3. 数据质量检查
    def check_data_quality(**context):
        """
        检查数据质量，决定流程走向
        返回 task_id 决定分支
        """
        ti = context['ti']

        db_count = ti.xcom_pull(task_ids='extract_group.extract_from_database', key='db_records')
        api_count = ti.xcom_pull(task_ids='extract_group.extract_from_api', key='api_records')

        total = (db_count or 0) + (api_count or 0)

        # 设置检查阈值
        if total == 0:
            return 'skip_processing'  # 无数据，跳过处理
        elif total < 100:
            return 'process_light_load'  # 数据量小，轻量处理
        else:
            return 'process_full_load'   # 数据量大，完整处理

    data_quality_check = BranchPythonOperator(
        task_id='data_quality_check',
        python_callable=check_data_quality,
    )

    # 4. 不同处理路径
    skip_processing = EmptyOperator(task_id='skip_processing')

    def process_light(**context):
        """轻量处理逻辑"""
        print("Processing light load...")
        return "Light processing completed"

    process_light_load = PythonOperator(
        task_id='process_light_load',
        python_callable=process_light,
    )

    # 5. 完整处理（使用 TaskGroup 组织复杂流程）
    with TaskGroup('process_full_load', tooltip='Full data processing') as process_full_load:

        # 清洗
        clean_data = PostgresOperator(
            task_id='clean_data',
            postgres_conn_id=POSTGRES_CONN_ID,
            sql="""
                INSERT INTO staging.cleaned_data
                SELECT
                    id,
                    TRIM(name) as name,
                    NULLIF(email, '') as email,
                    created_at
                FROM raw_data
                WHERE created_date = '{{ ds }}'
                  AND id IS NOT NULL;
            """,
        )

        # 转换
        transform_data = PostgresOperator(
            task_id='transform_data',
            postgres_conn_id=POSTGRES_CONN_ID,
            sql="""
                INSERT INTO analytics.daily_summary
                SELECT
                    '{{ ds }}' as date,
                    COUNT(*) as total_records,
                    COUNT(DISTINCT email) as unique_emails
                FROM staging.cleaned_data
                WHERE DATE(created_at) = '{{ ds }}';
            """,
        )

        clean_data >> transform_data

    # 6. 聚合节点
    join_branch = EmptyOperator(
        task_id='join_branch',
        trigger_rule='none_failed_min_one_success',  # 任意分支成功即继续
    )

    # 7. 数据加载到数仓
    def load_to_warehouse(**context):
        """加载到数据仓库"""
        # 这里可以使用 S3Hook、SnowflakeHook 等
        print("Loading data to warehouse...")
        return "Data loaded successfully"

    load_data = PythonOperator(
        task_id='load_to_warehouse',
        python_callable=load_to_warehouse,
    )

    # 8. 生成报告
    def generate_report(**context):
        """生成处理报告"""
        report = {
            'execution_date': context['ds'],
            'dag_id': context['dag'].dag_id,
            'run_id': str(context['run_id']),
        }
        print(f"Report: {json.dumps(report, indent=2)}")
        return report

    generate_report_task = PythonOperator(
        task_id='generate_report',
        python_callable=generate_report,
    )

    end = EmptyOperator(task_id='end')

    # 定义完整流程
    start >> wait_for_source_file >> extract_group >> data_quality_check

    # 分支定义
    data_quality_check >> [skip_processing, process_light_load, process_full_load]

    # 聚合
    [skip_processing, process_light_load, process_full_load] >> join_branch

    join_branch >> load_data >> generate_report_task >> end
```

### 3. 动态任务映射（Dynamic Task Mapping）

```python
# dynamic_tasks_dag.py
"""
动态任务映射示例

演示 Airflow 2.3+ 的 Dynamic Task Mapping 功能
用于在运行时生成动态数量的任务实例
"""

from datetime import datetime
from airflow import DAG
from airflow.operators.python import PythonOperator
from airflow.decorators import task, task_group
from airflow.operators.empty import EmptyOperator

with DAG(
    'dynamic_task_mapping',
    start_date=datetime(2024, 1, 1),
    schedule_interval=None,
    catchup=False,
) as dag:

    start = EmptyOperator(task_id='start')

    @task
    def get_files_to_process(**context):
        """
        动态获取需要处理的文件列表
        实际场景中可能从数据库或 API 获取
        """
        # 模拟动态发现文件
        execution_date = context['ds']
        files = [
            f'/data/{execution_date}/file_001.csv',
            f'/data/{execution_date}/file_002.csv',
            f'/data/{execution_date}/file_003.csv',
        ]
        return files

    @task
    def process_file(file_path: str):
        """处理单个文件"""
        print(f"Processing file: {file_path}")
        # 实际处理逻辑
        return {'file': file_path, 'status': 'processed'}

    @task
    def aggregate_results(results: list):
        """聚合所有处理结果"""
        success_count = sum(1 for r in results if r['status'] == 'processed')
        print(f"Total files: {len(results)}, Successful: {success_count}")
        return {'total': len(results), 'success': success_count}

    # 动态任务映射
    files = get_files_to_process()
    processed = process_file.expand(file_path=files)
    summary = aggregate_results(processed)

    start >> files >> processed >> summary
```

```yaml
metadata:
```

```yaml
            dependencies: [checkout]

            template: security-scan
            dependencies: [checkout]

          # 3. 构建（所有测试通过后）
          - name: build
            template: build-image
            dependencies: [unit-tests, integration-tests, security-scan]

          # 4. 部署到开发环境
          - name: deploy-dev
            template: deploy
            arguments:
              parameters:
                - name: environment
                  value: dev
                - name: image-tag
                  value: "{{tasks.build.outputs.parameters.image-tag}}"

          - name: workspace
            mountPath: /workspace

        parameters:
          - name: revision
            valueFrom:

    - name: run-tests
      inputs:
        parameters:
          - name: test-type
          - name: coverage-threshold
            default: "0"

                exit 1
              fi
            else
              npm run test:integration
            fi
        volumeMounts:
          - name: workspace
            mountPath: /workspace

    # 安全扫描模板
    - name: security-scan
      container:
        image: aquasec/trivy:latest
        command: [sh, -c]
        args:
          - |
            cd /workspace
            # 扫描依赖漏洞
            trivy filesystem --exit-code 1 --severity HIGH,CRITICAL .
        volumeMounts:
          - name: workspace
            mountPath: /workspace

    # 构建镜像模板
    - name: build-image
      inputs:
        artifacts:
          - name: source
            path: /workspace
      container:
        image: gcr.io/kaniko-project/executor:latest
        command: [/kaniko/executor]
        args:
          - --context=/workspace
          - --dockerfile=/workspace/Dockerfile
          - --destination=registry.example.com/app:{{workflow.name}}
          - --cache=true

      outputs:
        parameters:
          - name: image-tag
            value: "{{workflow.name}}"

    # 部署模板
    - name: deploy
      inputs:
        parameters:
          - name: environment
          - name: image-tag
      container:
        image: bitnami/kubectl:latest
        command: [sh, -c]
        args:
          - |

              -n {{inputs.parameters.environment}}

            # 等待部署完成
            kubectl rollout status deployment/app \
              -n {{inputs.parameters.environment}} \
              --timeout=300s

            echo "Deployment completed successfully!"

    # 带人工批准的部署
    - name: deploy-with-approval
      inputs:
        parameters:
          - name: environment
          - name: image-tag
      steps:
        # 第1步：等待人工批准
        - - name: wait-for-approval
            template: approval-suspend
        # 第2步：执行部署
        - - name: deploy
            template: deploy
            arguments:
              parameters:
                - name: environment
                  value: "{{inputs.parameters.environment}}"
                - name: image-tag
                  value: "{{inputs.parameters.image-tag}}"

    # 暂停模板（用于人工批准）
    - name: approval-suspend
      suspend: {}
```

```java
// GreetDelegate.java
package com.example.workflow;

import org.camunda.bpm.engine.delegate.DelegateExecution;
import org.camunda.bpm.engine.delegate.JavaDelegate;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.stereotype.Component;

/**
 * 问候任务委托类
 *
 * 实现 JavaDelegate 接口，用于执行服务任务
 * 被 Spring 管理，可通过依赖注入使用其他服务
 */
@Component("greetDelegate")
public class GreetDelegate implements JavaDelegate {

    private static final Logger LOGGER = LoggerFactory.getLogger(GreetDelegate.class);

    @Override
    public void execute(DelegateExecution execution) throws Exception {
        // 从流程变量获取输入
        String name = (String) execution.getVariable("name");
        if (name == null || name.isEmpty()) {
            name = "World";
        }

        String greeting = "Hello, " + name + "!";

        // 记录日志
        LOGGER.info("Executing greeting: {} for process instance {}",
            greeting, execution.getProcessInstanceId());

        // 设置流程变量供后续使用
        execution.setVariable("greeting", greeting);
        execution.setVariable("timestamp", System.currentTimeMillis());
    }
}
```

### 2. 复杂订单流程（包含人工任务和网关）

```xml

<bpmn:definitions xmlns:bpmn="http://www.omg.org/spec/BPMN/20100524/MODEL"

                      camunda:delegateExpression="${orderValidationDelegate}">

      <bpmn:outgoing>toAutoApproval</bpmn:outgoing>
    </bpmn:exclusiveGateway>

    <!-- 经理审批（人工任务） -->
    <bpmn:userTask id="ManagerApproval" name="Manager Approval"
                   camunda:assignee="manager"
                   camunda:dueDate="${dateTime().plusHours(24).toDate()}">
      <bpmn:incoming>toManagerApproval</bpmn:incoming>
      <bpmn:outgoing>toProcessPayment</bpmn:outgoing>
    </bpmn:userTask>

    <!-- 自动审批路径 -->
    <bpmn:serviceTask id="AutoApproval" name="Auto Approve"
                      camunda:expression="${orderService.autoApprove(execution)}">
      <bpmn:incoming>toAutoApproval</bpmn:incoming>
      <bpmn:outgoing>toProcessPayment</bpmn:outgoing>
    </bpmn:userTask>

    <!-- 并行网关：拆分 -->
    <bpmn:parallelGateway id="ParallelSplit" name="Split">
      <bpmn:incoming>toProcessPayment</bpmn:incoming>
      <bpmn:outgoing>toPayment</bpmn:outgoing>
      <bpmn:outgoing>toInventory</bpmn:outgoing>
    </bpmn:parallelGateway>

    <!-- 支付处理 -->
    <bpmn:serviceTask id="ProcessPayment" name="Process Payment"
                      camunda:delegateExpression="${paymentDelegate}">
      <bpmn:incoming>toPayment</bpmn:incoming>
      <bpmn:outgoing>fromPayment</bpmn:outgoing>
    </bpmn:serviceTask>

    <!-- 库存处理 -->
    <bpmn:serviceTask id="ReserveInventory" name="Reserve Inventory"
                      camunda:delegateExpression="${inventoryDelegate}">
      <bpmn:incoming>toInventory</bpmn:incoming>
      <bpmn:outgoing>fromInventory</bpmn:outgoing>
    </bpmn:serviceTask>

    <!-- 并行网关：合并 -->
    <bpmn:parallelGateway id="ParallelJoin" name="Join">
      <bpmn:incoming>fromPayment</bpmn:incoming>
      <bpmn:incoming>fromInventory</bpmn:incoming>
      <bpmn:outgoing>toNotify</bpmn:outgoing>
    </bpmn:parallelGateway>

    <!-- 发送通知 -->
    <bpmn:sendTask id="SendNotification" name="Send Confirmation Email"
                   camunda:delegateExpression="${notificationDelegate}">
      <bpmn:incoming>toNotify</bpmn:incoming>
      <bpmn:outgoing>toEnd</bpmn:outgoing>
    </bpmn:sendTask>

    <!-- 结束 -->
    <bpmn:endEvent id="OrderCompleted" name="Order Completed">
      <bpmn:incoming>toEnd</bpmn:incoming>
    </bpmn:endEvent>

    <!-- 流程连线 -->
    <bpmn:sequenceFlow id="toValidate" sourceRef="OrderReceived" targetRef="ValidateOrder"/>
    <bpmn:sequenceFlow id="toCheckAmount" sourceRef="ValidateOrder" targetRef="AmountCheck"/>

    <!-- 条件流程 -->
    <bpmn:sequenceFlow id="toManagerApproval" sourceRef="AmountCheck" targetRef="ManagerApproval">
      <bpmn:conditionExpression xsi:type="bpmn:tFormalExpression">
        ${orderAmount > 1000}
      </bpmn:conditionExpression>
    </bpmn:sequenceFlow>

    <bpmn:sequenceFlow id="toAutoApproval" sourceRef="AmountCheck" targetRef="AutoApproval">
      <bpmn:conditionExpression xsi:type="bpmn:tFormalExpression">
        ${orderAmount <= 1000}
      </bpmn:conditionExpression>
    </bpmn:sequenceFlow>

    <bpmn:sequenceFlow id="toProcessPayment" sourceRef="ManagerApproval" targetRef="ParallelSplit"/>
    <bpmn:sequenceFlow id="toProcessPayment2" sourceRef="AutoApproval" targetRef="ParallelSplit"/>
    <bpmn:sequenceFlow id="toPayment" sourceRef="ParallelSplit" targetRef="ProcessPayment"/>
    <bpmn:sequenceFlow id="toInventory" sourceRef="ParallelSplit" targetRef="ReserveInventory"/>
    <bpmn:sequenceFlow id="fromPayment" sourceRef="ProcessPayment" targetRef="ParallelJoin"/>
    <bpmn:sequenceFlow id="fromInventory" sourceRef="ReserveInventory" targetRef="ParallelJoin"/>
    <bpmn:sequenceFlow id="toNotify" sourceRef="ParallelJoin" targetRef="SendNotification"/>
    <bpmn:sequenceFlow id="toEnd" sourceRef="SendNotification" targetRef="OrderCompleted"/>

  </bpmn:process>

</bpmn:definitions>
```

```java
@Component

    @Autowired
    private PaymentService paymentService;

    @Override
    public void execute(DelegateExecution execution) throws Exception {
        Order order = (Order) execution.getVariable("validatedOrder");

            PaymentResult result = paymentService.charge(order);

        }
    }

        if (paymentId != null) {
```

```java
// OrderProcessController.java
package com.example.workflow.api;

import org.camunda.bpm.engine.RuntimeService;
import org.camunda.bpm.engine.runtime.ProcessInstance;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;

import java.util.HashMap;
import java.util.Map;

/**
 * 订单流程 REST API
 */
@RestController
@RequestMapping("/api/orders")
public class OrderProcessController {

    @Autowired
    private RuntimeService runtimeService;

    /**
     * 启动订单流程
     */
    @PostMapping("/{orderId}/process")
    public ResponseEntity<Map<String, Object>> startOrderProcess(
            @PathVariable String orderId,
            @RequestBody OrderRequest request) {

        // 准备流程变量
        Map<String, Object> variables = new HashMap<>();
        variables.put("orderId", orderId);
        variables.put("customerId", request.getCustomerId());
        variables.put("items", request.getItems());
        variables.put("priority", request.getPriority());

        // 启动流程实例
        ProcessInstance instance = runtimeService
            .startProcessInstanceByKey("OrderProcess", orderId, variables);

        // 返回响应
        Map<String, Object> response = new HashMap<>();
        response.put("processInstanceId", instance.getId());
        response.put("processDefinitionId", instance.getProcessDefinitionId());
        response.put("status", "STARTED");

        return ResponseEntity.ok(response);
    }

    /**
     * 查询流程状态
     */
    @GetMapping("/{orderId}/status")
    public ResponseEntity<Map<String, Object>> getOrderStatus(@PathVariable String orderId) {
        ProcessInstance instance = runtimeService
            .createProcessInstanceQuery()
            .processInstanceBusinessKey(orderId)
            .singleResult();

        if (instance == null) {
            return ResponseEntity.notFound().build();
        }

        Map<String, Object> response = new HashMap<>();
        response.put("processInstanceId", instance.getId());
        response.put("isSuspended", instance.isSuspended());
        response.put("variables", runtimeService.getVariables(instance.getId()));

        return ResponseEntity.ok(response);
    }

    /**
     * 完成人工任务
     */
    @PostMapping("/{orderId}/approve")
    public ResponseEntity<Void> approveOrder(
            @PathVariable String orderId,
            @RequestBody ApprovalRequest request) {

        // 查询任务
        var task = taskService.createTaskQuery()
            .processInstanceBusinessKey(orderId)
            .taskDefinitionKey("ManagerApproval")
            .singleResult();

        if (task == null) {
            return ResponseEntity.notFound().build();
        }

        // 设置变量并完成任务
        Map<String, Object> variables = new HashMap<>();
        variables.put("approved", request.isApproved());
        variables.put("approvalComment", request.getComment());

        taskService.complete(task.getId(), variables);

        return ResponseEntity.ok().build();
    }
}
```

## 对比总结

| 特性 | Temporal | Airflow | Argo | Camunda |
|------|----------|---------|------|---------|
| **编程模型** | 代码优先 (Go/Java/TS/Python) | Python DAG 定义 | YAML + 容器 | BPMN 2.0 + Java/Expression |
| **部署方式** | 独立服务 | 独立服务/Python 环境 | Kubernetes 原生 | Java 应用/独立服务 |
| **持久化** | 内置（多数据库支持） | 需要数据库（PostgreSQL/MySQL） | 依赖 Kubernetes/etcd | 需要数据库 |
| **横向扩展** | 优秀（Worker 水平扩展） | 良好（Celery Executor） | 优秀（K8s 原生） | 良好 |
| **人工任务** | 通过 Signal 实现 | 通过 Sensor/外部触发器 | 通过 Suspend | 原生支持（User Task） |
| **定时调度** | Cron Workflow | 核心功能（Cron 表达式） | Cron Workflow | 通过 Job Executor |
| **适用场景** | 微服务编排、长事务 | 数据管道、ETL | CI/CD、K8s 原生 | 业务流程管理、审批 |
| **学习曲线** | 中等 | 平缓 | 中等 | 平缓（有 BPMN 基础） |
| **生态工具** | 丰富（UI、CLI、多种语言） | 非常丰富 | 增长中 | 成熟（Cockpit、Tasklist） |

### 选择建议

1. **选择 Temporal 当**：
   - 需要长时间运行的可靠工作流
   - 需要多语言 SDK 支持
   - 微服务编排是主要场景
   - 需要强大的故障恢复能力

2. **选择 Airflow 当**：
   - 数据工程/ETL 是主要场景
   - 团队熟悉 Python
   - 需要丰富的数据源集成
   - 定时调度需求复杂

3. **选择 Argo 当**：
   - 已经使用 Kubernetes
   - CI/CD 是主要场景
   - 希望云原生方式定义工作流
   - 需要容器化任务执行

4. **选择 Camunda 当**：
   - 业务流程需要可视化建模
   - 人工审批是核心场景
   - 需要 BPMN 2.0 标准兼容
   - 业务分析师需要参与流程设计

## 最佳实践

### Temporal 最佳实践

1. **Activity 幂等性**：所有 Activity 应该设计为幂等的
2. **心跳机制**：长运行 Activity 使用心跳报告进度
3. **工作流简洁**：工作流代码保持简洁，业务逻辑放在 Activity
4. **版本控制**：使用 `workflow.GetVersion` 处理工作流变更
5. **错误处理**：区分可重试错误和不可重试错误

### Airflow 最佳实践

1. **幂等性**：所有任务应该是幂等的
2. **任务粒度**：保持任务小而聚焦，便于重试
3. **连接管理**：使用 Airflow Connections 管理凭证
4. **XCom**：谨慎使用 XCom，大数据量使用外部存储
5. **SLA 和告警**：为关键任务配置 SLA

### Argo 最佳实践

1. **资源限制**：始终设置容器资源限制
2. **重试策略**：配置合理的重试和超时
3. **RBAC**：使用最小权限的服务账户
4. **工件管理**：使用 S3/MinIO 管理中间数据
5. **模板复用**：使用 WorkflowTemplate 共享通用模板

### Camunda 最佳实践

1. **服务任务**：保持服务任务轻量，复杂逻辑委托给外部服务
2. **错误边界**：使用错误边界事件处理异常
3. **补偿机制**：使用补偿处理器实现 Saga 模式
4. **历史清理**：配置历史数据清理策略
5. **流程版本**：使用版本控制管理流程变更
