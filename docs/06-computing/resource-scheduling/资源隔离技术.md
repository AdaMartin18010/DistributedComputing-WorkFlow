# 资源隔离技术详解

## 1. 概述

资源隔离是分布式系统中确保多租户环境下应用稳定运行的关键技术。通过限制和隔离 CPU、内存、磁盘 I/O、网络等资源使用，防止单个应用影响整个系统的稳定性。

```
│                                                                     │

│  ─────────────────────────────────────────────────────────          │
│                                                                     │
│  物理隔离          独立机器            整机                          │
│       │                                                            │

│  操作系统隔离      Cgroup/Namespace   进程级                         │
│       │                                                            │

│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 2. Linux Cgroup

Cgroup（Control Group）是 Linux 内核提供的资源限制和统计机制。

```
┌─────────────────────────────────────────────────────────────────────┐
│                   Cgroup 子系统                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐  ┌────────────┐ │
│  │   cpu       │  │   memory    │  │   blkio     │  │   pids     │ │
│  │             │  │             │  │             │  │            │ │
│  │ - shares    │  │ - limit     │  │ - read_bps  │  │ - max      │ │
│  │ - quota     │  │ - swap      │  │ - write_bps │  │            │ │
│  │ - period    │  │ - swappiness│  │ - read_iops │  │            │ │
│  │ - cpusets   │  │ - oom_control│  │ - write_iops│  │            │ │
│  └─────────────┘  └─────────────┘  └─────────────┘  └────────────┘ │
│                                                                     │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐                 │

```

### 2.1 CPU 限制

```bash
# 创建 cgroup
mkdir -p /sys/fs/cgroup/cpu/myapp

# 设置 CPU 权重（相对值）
echo 512 > /sys/fs/cgroup/cpu/myapp/cpu.shares

# 设置 CPU 配额（绝对限制）
# 每 100ms 周期内，最多使用 50ms CPU
echo 50000 > /sys/fs/cgroup/cpu/myapp/cpu.cfs_quota_us
echo 100000 > /sys/fs/cgroup/cpu/myapp/cpu.cfs_period_us

# 绑定到特定 CPU 核心
echo 0-3 > /sys/fs/cgroup/cpuset/myapp/cpuset.cpus

# 将进程加入 cgroup
echo $PID > /sys/fs/cgroup/cpu/myapp/tasks
```

### 2.2 内存限制

```bash
# 创建 cgroup
mkdir -p /sys/fs/cgroup/memory/myapp

# 设置内存限制（1GB）
echo 1073741824 > /sys/fs/cgroup/memory/myapp/memory.limit_in_bytes

# 设置 swap 限制（不限制）
echo -1 > /sys/fs/cgroup/memory/myapp/memory.memsw.limit_in_bytes

# 设置 swappiness
echo 10 > /sys/fs/cgroup/memory/myapp/memory.swappiness

# 启用 OOM killer
echo 1 > /sys/fs/cgroup/memory/myapp/memory.oom_control

# 将进程加入 cgroup
echo $PID > /sys/fs/cgroup/memory/myapp/tasks
```

### 2.3 磁盘 I/O 限制

```bash
# 创建 cgroup
mkdir -p /sys/fs/cgroup/blkio/myapp

# 限制读带宽（10MB/s）
echo "8:0 10485760" > /sys/fs/cgroup/blkio/myapp/blkio.throttle.read_bps_device

# 限制写带宽（5MB/s）
echo "8:0 5242880" > /sys/fs/cgroup/blkio/myapp/blkio.throttle.write_bps_device

# 限制 IOPS
echo "8:0 100" > /sys/fs/cgroup/blkio/myapp/blkio.throttle.read_iops_device
echo "8:0 50" > /sys/fs/cgroup/blkio/myapp/blkio.throttle.write_iops_device
```

Namespace 提供了进程级别的资源隔离，使进程只能看到特定的系统资源。

### 3.1 创建 Namespace

```bash
# 创建新的 PID 和 Mount Namespace
unshare --pid --mount --fork /bin/bash

# 创建 Network Namespace
ip netns add myns
ip netns exec myns ip link list

# 创建完整的容器环境（所有 Namespace）
unshare --pid --net --mount --ipc --uts --fork /bin/bash
```

### 3.2 在 YARN 中的应用

```bash
# YARN 使用 Cgroup 进行资源隔离
# yarn-site.xml 配置
<property>
    <name>yarn.nodemanager.linux-container-executor.cgroups.mount</name>
    <value>true</value>
</property>

<property>
    <name>yarn.nodemanager.linux-container-executor.cgroups.hierarchy</name>
    <value>/yarn</value>
</property>

<property>
    <name>yarn.nodemanager.resource.cpu.enabled</name>
    <value>true</value>
</property>

<property>
    <name>yarn.nodemanager.linux-container-executor.cgroups.strict-resource-usage</name>
    <value>true</value>
</property>
```

## 4. Docker 资源隔离

Docker 是 Cgroup 和 Namespace 的高级封装。

```bash
# CPU 限制
docker run -it \
    --cpu-shares 512 \
    --cpu-quota 50000 \
    --cpu-period 100000 \
    --cpuset-cpus 0-3 \
    ubuntu

# 内存限制
docker run -it \
    --memory 1g \
    --memory-swap 1g \
    --memory-swappiness 10 \
    ubuntu

# I/O 限制
docker run -it \
    --device-read-bps /dev/sda:10mb \
    --device-write-bps /dev/sda:5mb \
    ubuntu

# Pid 限制
docker run -it --pids-limit 100 ubuntu
```

### 4.1 Docker Compose 配置

```yaml
version: '3.8'
services:
  app:
    image: myapp:latest
    deploy:
      resources:
        limits:
          cpus: '2.0'
          memory: 2G
        reservations:
          cpus: '1.0'
          memory: 1G
    pid_limit: 100
    blkio_config:
      weight: 300
      device_read_bps:
        - path: /dev/sda
          rate: '10mb'
      device_write_bps:
        - path: /dev/sda
          rate: '5mb'
```

## 5. Kubernetes 资源隔离

### 5.1 Resource Quota

```yaml
apiVersion: v1
kind: ResourceQuota
metadata:
  name: namespace-quota
  namespace: production
spec:
  hard:
    requests.cpu: "100"
    requests.memory: 200Gi
    limits.cpu: "200"
    limits.memory: 400Gi
    persistentvolumeclaims: "10"
    pods: "100"
    services: "20"
    secrets: "10"

```

### 5.2 Limit Range

```yaml
apiVersion: v1
kind: LimitRange
metadata:
  name: default-limits
  namespace: production
spec:
  limits:
  - default:
      cpu: "1"
      memory: 1Gi
    defaultRequest:
      cpu: 100m
      memory: 256Mi
    max:
      cpu: "4"
      memory: 8Gi
    min:
      cpu: 50m
      memory: 128Mi
```

## 7. 大数据平台资源隔离

### 7.1 YARN Node Labels

```bash
# 创建节点标签
yarn rmadmin -addToClusterNodeLabels "GPU(exclusive=false)"
yarn rmadmin -addToClusterNodeLabels "HIGH_MEMORY(exclusive=false)"

# 给节点添加标签
yarn rmadmin -replaceLabelsOnNode "node1=GPU node2=HIGH_MEMORY"

# 提交到指定标签队列
spark-submit \
    --queue production \
    --conf spark.yarn.am.nodeLabelExpression="GPU" \
    --conf spark.executor.nodeLabelExpression="GPU" \
    app.jar
```

## 9. 总结

资源隔离技术对比：

| 技术 | 粒度 | 适用场景 |
|------|------|----------|
| **Cgroup** | 进程组 | 资源限制和统计 |
| **Namespace** | 进程 | 环境隔离 |
| **Docker** | 容器 | 应用打包和部署 |
| **Kubernetes** | Pod/Namespace | 容器编排和资源管理 |

最佳实践：

1. 多层隔离结合使用
2. 设置合理的资源限制
3. 监控实际资源使用
4. 预留系统资源缓冲
