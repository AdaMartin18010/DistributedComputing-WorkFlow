# 一致性状态机图

> 🔄 使用Mermaid StateDiagram展示分布式系统中的关键状态流转

---

## 1️⃣ 副本状态机

```mermaid
stateDiagram-v2
    [*] --> Follower: 启动/初始化

    state "Follower状态" as F {
        [*] --> Idle: 初始化完成
        Idle --> Receiving: 收到AppendEntries
        Idle --> Voting: 选举超时
        Receiving --> Idle: 处理完成
        Receiving --> Applying: 提交日志
        Applying --> Idle: 应用到状态机
    }

    state "Candidate状态" as C {
        [*] --> Requesting: 增加Term
        Requesting --> VoteCounting: 发送RequestVote
        VoteCounting --> Won: 获得多数票
        VoteCounting --> Lost: 发现更高Term
        VoteCounting --> Timeout: 选举超时
        Timeout --> Requesting: 重新选举
    }

    state "Leader状态" as L {
        [*] --> Establishing: 发送心跳
        Establishing --> Leading: 确认领导地位
        Leading --> Replicating: 接收客户端请求
        Replicating --> Committing: 复制到多数派
        Committing --> Leading: 提交确认
        Leading --> SteppingDown: 发现更高Term
    }

    F --> C: 选举超时
    C --> F: 发现更高Term/新Leader
    C --> L: 选举成功
    L --> F: 失去领导地位

    F --> [*]: 优雅关闭
    C --> [*]: 优雅关闭
    L --> [*]: 优雅关闭

    note right of F
        被动状态
        - 响应Leader请求
        - 检测Leader失效
    end note

    note right of C
        过渡状态
        - 短暂存在
        - 争取选票
    end note

    note right of L
        主动状态
        - 处理客户端请求
        - 复制日志
    end note
```

**状态说明**：

| 状态 | 职责 | 转换条件 |
|------|------|----------|
| **Follower** | 被动接收日志，响应Leader | 选举超时→Candidate |
| **Candidate** | 发起选举，争取成为Leader | 获得多数票→Leader |
| **Leader** | 处理请求，复制日志到Followers | 发现更高Term→Follower |

## 2️⃣ 视图变更状态机 (PBFT)

```mermaid
stateDiagram-v2
    [*] --> Normal: 系统启动

    state "正常状态" as Normal {
        [*] --> Processing
        Processing --> Checkpointing: 执行K个请求
        Checkpointing --> Processing: 生成检查点
        Processing --> Suspecting: 检测主节点故障
    }

    state "视图变更状态" as ViewChange {
        [*] --> ViewChangeRequest
        ViewChangeRequest --> CollectingVC: 广播View-Change
        CollectingVC --> NewViewReady: 收集2f个VC
        NewViewReady --> SendingNV: 新主节点发送New-View
        SendingNV --> ViewChangeConfirm: 副本验证NV
        ViewChangeConfirm --> ViewChangeComplete: 准备就绪
    }

    Normal --> ViewChange: 主节点故障
    ViewChange --> Normal: 视图变更完成

    state ViewChange {
        ViewChangeRequest: 发起视图变更
        note right of ViewChangeRequest
            原因：
            - 主节点无响应
            - 超时检测
            - 恶意行为检测
        end note
    }
```

**视图变更流程**：

```mermaid
sequenceDiagram
    participant R1 as Replica 1
    participant R2 as Replica 2
    participant R3 as Replica 3 (New Primary)
    participant R4 as Replica 4

    Note over R1,R4: 视图v，检测到主节点故障

    R1->>R1: 超时触发
    R1->>R2: VIEW-CHANGE(v+1)
    R1->>R3: VIEW-CHANGE(v+1)
    R1->>R4: VIEW-CHANGE(v+1)

    R2->>R2: 同样发送VIEW-CHANGE
    R4->>R4: 同样发送VIEW-CHANGE

    R3->>R3: 收到2f个VIEW-CHANGE<br/>成为新主节点

    R3->>R1: NEW-VIEW(v+1, V, O)
    R3->>R2: NEW-VIEW(v+1, V, O)
    R3->>R4: NEW-VIEW(v+1, V, O)

    R1->>R1: 验证并进入视图v+1
    R2->>R2: 验证并进入视图v+1
    R4->>R4: 验证并进入视图v+1
```

## 3️⃣ 事务状态流转

### 2PC事务状态机

```mermaid
stateDiagram-v2
    [*] --> Active: 事务开始

    state "活动状态" as Active {
        [*] --> Executing
        Executing --> Preparing: 准备提交
        Preparing --> Prepared: 预写日志完成
        Preparing --> PreparingFailed: 准备失败
    }

    state "决策状态" as Decision {
        [*] --> Voting
        Voting --> Committing: 全部Yes
        Voting --> Aborting: 任一No/超时
    }

    state "终止状态" as Terminal {
        state "提交路径" as CommitPath {
            Committing --> Committed: 写Commit记录
            Committed --> Complete: 释放锁
        }
        state "回滚路径" as AbortPath {
            Aborting --> Aborted: 写Abort记录
            Aborted --> Complete: 释放锁
        }
        Complete --> [*]
    }

    Active --> Decision: 准备阶段完成
    PreparingFailed --> Aborting: 直接回滚
    Decision --> Terminal: 决策执行

    note right of Active
        协调者状态机
        参与者有独立状态机
    end note
```

### 参与者状态机

```mermaid
stateDiagram-v2
    [*] --> Idle

    Idle --> Working: 收到子事务
    Working --> Prepared: 执行成功<br/>写Prepare记录
    Working --> Aborted: 执行失败

    Prepared --> Committed: 收到Commit
    Prepared --> Aborted: 收到Abort/超时

    Aborted --> Idle: 清理完成
    Committed --> Idle: 清理完成

    note right of Prepared
        关键点：
        必须能提交或回滚
        不能丢失决策
    end note
```

## 4️⃣ Saga事务状态机

```mermaid
stateDiagram-v2
    [*] --> Started: Saga开始

    state "执行阶段" as Execution {
        [*] --> Step1
        Step1 --> Step2: 成功
        Step1 --> Compensating1: 失败

        Step2 --> Step3: 成功
        Step2 --> Compensating2: 失败

        Step3 --> Completed: 成功
        Step3 --> Compensating3: 失败
    }

    state "补偿阶段" as Compensation {
        Compensating3 --> Compensating2: 补偿完成
        Compensating2 --> Compensating1: 补偿完成
        Compensating1 --> Compensated: 补偿完成

        Compensating1 --> CompensationFailed: 补偿失败
        Compensating2 --> CompensationFailed: 补偿失败
        Compensating3 --> CompensationFailed: 补偿失败
    }

    Compensating1 --> Compensation
    Compensating2 --> Compensation
    Compensating3 --> Compensation

    Completed --> [*]: Saga成功
    Compensated --> [*]: Saga回滚成功
    CompensationFailed --> [*]: 需人工介入

    note right of Execution
        每个步骤T：
        - 正向服务
        - 补偿服务C
    end note
```

**Saga执行示例**：

```mermaid
sequenceDiagram
    participant O as Orchestrator
    participant S1 as 订单服务
    participant S2 as 库存服务
    participant S3 as 支付服务

    O->>S1: T1: 创建订单
    S1-->>O: 成功

    O->>S2: T2: 扣减库存
    S2-->>O: 成功

    O->>S3: T3: 处理支付
    S3--xO: 失败

    Note over O,S3: 开始补偿

    O->>S2: C2: 恢复库存
    S2-->>O: 补偿成功

    O->>S1: C1: 取消订单
    S1-->>O: 补偿成功

    O->>O: Saga完成(已回滚)
```

## 5️⃣ 一致性级别状态机

```mermaid
stateDiagram-v2
    [*] --> Linearizable: 线性一致性

    Linearizable --> Sequential: 放宽实时性
    Sequential --> Causal: 放宽全局顺序
    Causal --> Session: 限制为会话
    Session --> MonotonicReads: 放宽写入可见
    MonotonicReads --> ReadYourWrites: 进一步放宽
    ReadYourWrites --> Eventual: 最终一致

    Eventual --> ReadYourWrites: 增强保证
    ReadYourWrites --> MonotonicReads: 增强保证
    MonotonicReads --> Session: 增强保证
    Session --> Causal: 增强保证
    Causal --> Sequential: 增强保证
    Sequential --> Linearizable: 增强保证

    state Linearizable {
        note right of Linearizable
            最强一致性
            每次读取都看到最新写入
        end note
    }

    state Eventual {
        note right of Eventual
            最弱一致性
            最终所有副本一致
        end note
    }

    Linearizable --> [*]: 最高一致性要求
    Eventual --> [*]: 最高可用性要求
```

## 6️⃣ 故障检测状态机

```mermaid
stateDiagram-v2
    [*] --> Unsuspected: 健康

    Unsuspected --> Suspected: 心跳超时
    Suspected --> Unsuspected: 收到心跳
    Suspected --> Failed: 确认超时

    Failed --> Unsuspected: 节点恢复

    state "健康状态" as Unsuspected {
        [*] --> Monitoring
        Monitoring --> HeartbeatReceived: 收到心跳
        HeartbeatReceived --> Monitoring: 更新lastSeen
    }

    state "怀疑状态" as Suspected {
        [*] --> Waiting
        Waiting --> ConfirmTimeout: 等待确认
        ConfirmTimeout --> ConfirmFailed: 二次超时
    }

    note right of Suspected
        Phi累积故障检测器
        自适应超时阈值
    end note
```

## 📊 状态机复杂度对比

| 状态机类型 | 状态数 | 复杂度 | 主要用途 |
|------------|--------|--------|----------|
| **Raft副本** | 3 | ⭐⭐ | 领导者选举 |
| **PBFT视图** | 4+ | ⭐⭐⭐⭐ | 拜占庭容错 |
| **2PC事务** | 6+ | ⭐⭐⭐ | 分布式事务 |
| **Saga** | 2n+ | ⭐⭐⭐ | 长事务 |
| **一致性级别** | 6 | ⭐⭐ | 模型选择 |
| **故障检测** | 3 | ⭐⭐ | 故障识别 |
