## 一、核心概念

### 1.1 电商系统架构定义

电商系统是典型的分布式高并发系统，需要处理海量用户请求、保证数据一致性、支持高可用和弹性扩展。其核心挑战包括：

- **高并发访问**：秒杀活动峰值可达百万级QPS
- **分布式事务**：跨服务数据一致性保障
- **数据一致性**：库存扣减、支付状态等强一致要求
- **系统可用性**：7×24小时不间断服务

### 1.2 关键特性

- **水平扩展**：支持按需扩容的计算和存储能力
- **最终一致性**：通过异步消息保证数据最终一致
- **多级缓存**：Redis、CDN、本地缓存多层架构
- **熔断降级**：故障隔离和优雅降级能力

### 1.3 适用场景

| 场景 | 适用性 | 说明 |
|------|--------|------|
| B2C电商平台 | ⭐⭐⭐⭐⭐ | 天猫、京东模式 |
| C2C电商平台 | ⭐⭐⭐⭐⭐ | 淘宝、拼多多模式 |
| O2O电商 | ⭐⭐⭐⭐ | 线上线下融合 |
| 社交电商 | ⭐⭐⭐⭐ | 直播带货、拼团 |

### 2.2 Saga模式实现

#### 2.2.1 编排式Saga（Orchestration）

**状态机定义**：

```java
public enum OrderSagaState {
    STARTED,           // 开始
    ORDER_CREATED,     // 订单已创建
    STOCK_DEDUCTED,    // 库存已扣减
    PAYMENT_CREATED,   // 支付单已创建
    PAYMENT_COMPLETED, // 支付完成
    ORDER_COMPLETED,   // 订单完成
    COMPENSATING,      // 补偿中
    COMPENSATED        // 已补偿
}
```

**正向流程**：

| 步骤 | 服务 | 操作 | 超时时间 |
|------|------|------|----------|
| 1 | 订单服务 | 创建订单 | 2s |
| 2 | 库存服务 | 预扣库存 | 3s |
| 3 | 支付服务 | 创建支付单 | 2s |
| 4 | 物流服务 | 创建运单 | 3s |

**补偿流程**：

```
补偿顺序（与正向相反）：
1. 补偿物流服务 → 取消运单
2. 补偿支付服务 → 退款/取消支付单
3. 补偿库存服务 → 释放库存
4. 补偿订单服务 → 关闭订单
```

#### 2.2.2 异常处理策略

| 异常类型 | 处理策略 | 说明 |
|----------|----------|------|
| 服务超时 | 重试+补偿 | 指数退避重试3次 |
| 业务异常 | 立即补偿 | 触发完整补偿流程 |
| 网络故障 | 定时检查 | 定时任务兜底补偿 |

### 3.2 库存扣减策略

#### 3.2.1 三级库存架构

| 层级 | 存储 | 用途 | 一致性 |
|------|------|------|--------|
| L1 | 本地缓存 | 热点数据读取 | 最终一致 |
| L2 | Redis | 库存扣减/预扣 | 强一致（Lua） |
| L3 | MySQL | 持久化存储 | 强一致 |

#### 3.2.2 Redis库存扣减（Lua脚本）

```lua
-- stock_deduct.lua
local key = KEYS[1]          -- 库存key: stock:sku:{sku_id}
local deductQty = tonumber(ARGV[1])  -- 扣减数量
local orderId = ARGV[2]      -- 订单ID

-- 1. 获取当前库存
local currentStock = redis.call('GET', key)
if not currentStock then
    return {-1, "库存key不存在"}
end

currentStock = tonumber(currentStock)

-- 2. 检查库存是否充足
if currentStock < deductQty then
    return {-2, "库存不足"}
end

-- 3. 扣减库存
local newStock = currentStock - deductQty
redis.call('SET', key, newStock)

-- 4. 记录预扣日志（用于幂等和回滚）
local preDeductKey = "pre_deduct:" .. orderId
redis.call('HSET', preDeductKey, "sku_id", key, "qty", deductQty)
redis.call('EXPIRE', preDeductKey, 3600)  -- 1小时过期

return {0, newStock}
```

#### 3.2.3 库存同步策略

```
同步方式对比：

┌─────────────┬───────────┬───────────┬───────────┐
│   策略      │   延迟    │   一致性   │   适用场景 │
├─────────────┼───────────┼───────────┼───────────┤
│ 实时同步    │   毫秒级   │   强一致   │  核心SKU   │
│ 异步MQ      │   秒级     │  最终一致  │  一般SKU   │
│ 定时任务    │   分钟级   │  最终一致  │  全量对账  │
└─────────────┴───────────┴───────────┴───────────┘
```

### 3.3 库存防超卖方案

| 方案 | 实现方式 | 优缺点 |
|------|----------|--------|
| 数据库乐观锁 | version字段 | 简单，但高并发性能差 |
| Redis原子操作 | Lua脚本/Redisson | 性能好，需处理同步 |
| 令牌桶限流 | 限制请求速率 | 保护系统，不直接防超卖 |

### 4.3 幂等性保障

#### 4.3.1 幂等键设计

```java
public class PaymentIdempotency {
    // 幂等键 = 业务类型:业务单号:操作类型
    // 示例：PAYMENT:ORDER_123456:PAY

    public String generateKey(String bizType, String bizNo, String action) {
        return String.format("%s:%s:%s", bizType, bizNo, action);
    }
}
```

#### 4.3.2 幂等控制表

```sql
CREATE TABLE t_idempotency (
    idempotency_key VARCHAR(128) PRIMARY KEY,
    biz_type        VARCHAR(32),
    biz_no          VARCHAR(64),
    action          VARCHAR(32),
    status          VARCHAR(32), -- PROCESSING/SUCCESS/FAILED
    result_code     VARCHAR(32),
    result_msg      VARCHAR(256),
    response_data   JSON,
    expire_time     DATETIME,    -- 幂等记录过期时间
    create_time     DATETIME
);

-- 唯一索引确保幂等
CREATE UNIQUE INDEX uk_idempotency_key ON t_idempotency(idempotency_key);
```

### 5.3 秒杀核心流程

#### 5.3.1 秒杀令牌机制

```java
@Service
public class SeckillTokenService {

    @Autowired
    private StringRedisTemplate redisTemplate;

    // 活动开始前预热：将库存量转为令牌放入Redis队列
    public void preheatTokens(Long activityId, Long skuId, Integer stock) {
        String tokenKey = "seckill:tokens:" + activityId + ":" + skuId;

        // 使用Redis List作为令牌桶
        for (int i = 0; i < stock; i++) {
            redisTemplate.opsForList().leftPush(tokenKey,
                UUID.randomUUID().toString());
        }

        // 设置过期时间
        redisTemplate.expire(tokenKey, Duration.ofHours(2));
    }

    // 获取秒杀令牌（原子操作）
    public String acquireToken(Long activityId, Long skuId) {
        String tokenKey = "seckill:tokens:" + activityId + ":" + skuId;
        // 从队列右侧弹出令牌
        return redisTemplate.opsForList().rightPop(tokenKey);
    }
}
```
