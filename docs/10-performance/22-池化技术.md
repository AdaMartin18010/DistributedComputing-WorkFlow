# 池化技术：连接池与对象池

## 核心概念

### 1. 池化原理

池化技术通过复用昂贵的资源，避免频繁创建和销毁的开销。

**资源生命周期对比**：

```
┌─────────────────────────────────────────────────────────────┐
│                   资源生命周期对比                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   无池化（每次创建销毁）：                                   │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  创建 ──→ 使用 ──→ 销毁                             │  │
│   │  10ms     5ms      5ms                              │  │
│   │  单次开销: 20ms                                     │  │
│   │  1000次: 20秒                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   池化（复用）：                                            │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  初始化: 创建N个资源 ──→ 放入池                      │  │
│   │           N × 10ms                                  │  │
│   │                                                     │  │
│   │  使用: 从池获取 ──→ 使用 ──→ 归还                   │  │
│   │        <1ms        5ms                              │  │
│   │        单次开销: 5ms                                │  │
│   │        1000次: 5秒 + 初始化                         │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   收益：                                                     │
│   - 资源复用，减少创建销毁开销                              │
│   - 控制资源数量，防止资源耗尽                              │
│   - 提高响应速度                                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**适用池化资源**：

| 资源类型 | 创建开销 | 池化收益 | 常见实现 |
|----------|----------|----------|----------|
| 数据库连接 | 高（网络+认证） | 高 | HikariCP、Druid |
| HTTP连接 | 中（TCP握手） | 高 | HttpClient连接池 |
| 线程 | 高（OS资源） | 高 | 线程池 |
| 对象 | 低-中 | 中 | ObjectPool |
| 内存 | 低 | 低 | 内存池 |

## 实践方法

### 连接池配置

**1. HikariCP最佳实践**：

```yaml
spring:
  datasource:
    hikari:
      # 核心参数
      maximum-pool-size: 20
      minimum-idle: 5
      connection-timeout: 30000
      idle-timeout: 600000
      max-lifetime: 1800000

      # 性能优化
      data-source-properties:
        cachePrepStmts: true
        prepStmtCacheSize: 250
        prepStmtCacheSqlLimit: 2048
```

**2. HTTP连接池**：

```java
// Apache HttpClient
PoolingHttpClientConnectionManager cm =
    new PoolingHttpClientConnectionManager();
cm.setMaxTotal(200);        // 最大连接数
cm.setDefaultMaxPerRoute(20);  // 每个路由最大连接

CloseableHttpClient httpClient = HttpClients.custom()
    .setConnectionManager(cm)
    .build();

// 或使用OkHttp
OkHttpClient client = new OkHttpClient.Builder()
    .connectionPool(new ConnectionPool(10, 5, TimeUnit.MINUTES))
    .build();
```

### 对象池实现

**1. Apache Commons Pool**：

```java
public class ObjectPoolExample {

    public static void main(String[] args) throws Exception {
        // 对象工厂
        PooledObjectFactory<MyObject> factory =
            new BasePooledObjectFactory<MyObject>() {
            @Override
            public MyObject create() {
                return new MyObject();  // 创建对象
            }

            @Override
            public PooledObject<MyObject> wrap(MyObject obj) {
                return new DefaultPooledObject<>(obj);
            }
        };

        // 配置
        GenericObjectPoolConfig<MyObject> config =
            new GenericObjectPoolConfig<>();
        config.setMaxTotal(100);
        config.setMaxIdle(10);
        config.setMinIdle(5);

        // 创建池
        ObjectPool<MyObject> pool = new GenericObjectPool<>(factory, config);

        // 使用
        MyObject obj = pool.borrowObject();
        try {
            obj.doSomething();
        } finally {
            pool.returnObject(obj);
        }
    }
}
```

**2. Go对象池**：

```go
type ObjectPool struct {
    pool sync.Pool
}

func NewObjectPool(newFunc func() interface{}) *ObjectPool {
    return &ObjectPool{
        pool: sync.Pool{
            New: newFunc,
        },
    }
}

func (p *ObjectPool) Get() interface{} {
    return p.pool.Get()
}

func (p *ObjectPool) Put(obj interface{}) {
    p.pool.Put(obj)
}

// 使用
var bufferPool = NewObjectPool(func() interface{} {
    return make([]byte, 4096)
})

func process() {
    buf := bufferPool.Get().([]byte)
    defer bufferPool.Put(buf)

    // 使用buf
}
```

## 案例数据

### 连接池优化效果

**场景**：微服务调用下游服务

**无连接池**：

```
每次调用：建立TCP连接 → TLS握手 → HTTP请求 → 关闭连接
耗时：50ms
QPS：20/秒
```

**有连接池**：

```
复用连接：直接发送HTTP请求
耗时：5ms
QPS：200/秒
性能提升：10倍
```

## 优化前后对比

| 指标 | 无池化 | 池化 | 改进 |
|------|--------|------|------|
| 平均延迟 | 50ms | 5ms | 90%↓ |
| 吞吐量 | 20 QPS | 200 QPS | 10倍↑ |
| 资源消耗 | 高 | 低 | 显著降低 |

---

**总结**：池化技术是性能优化的基础手段。对于创建开销大的资源，使用池化可以显著提升性能，降低资源消耗。需要合理配置池大小，避免资源耗尽或浪费。
