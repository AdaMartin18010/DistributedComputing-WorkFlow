## 1. 数据倾斜处理

数据倾斜是批处理作业中最常见的问题，会导致部分任务执行时间过长，拖慢整体作业进度。

### 1.1 两阶段聚合（加盐法）

```python
# PySpark 两阶段聚合解决数据倾斜
from pyspark.sql import SparkSession
from pyspark.sql.functions import spark_partition_id, col, rand, concat, lit

spark = SparkSession.builder.appName("SkewHandling").getOrCreate()

# 原始数据
df = spark.read.parquet("skewed_data")

# 第一阶段：加盐局部聚合
salt_count = 10
salted_df = df.withColumn("salt", (rand() * salt_count).cast("int")) \
              .withColumn("salted_key", concat(col("key"), lit("_"), col("salt")))

# 局部聚合
local_agg = salted_df.groupBy("salted_key").agg(sum("value").alias("local_sum"))

# 第二阶段：去盐全局聚合
from pyspark.sql.functions import split, element_at

global_agg = local_agg.withColumn("original_key",
                                   split(col("salted_key"), "_")[0]) \
                      .groupBy("original_key") \
                      .agg(sum("local_sum").alias("total_sum"))
```

### 1.2 Hive 数据倾斜处理

```sql
-- Hive 中处理 Join 倾斜

-- 方法1: 使用 Skew Join
set hive.optimize.skewjoin=true;
set hive.skewjoin.key=100000;

SELECT /*+ SKEWJOIN(a) */ *
FROM large_table a
JOIN small_table b ON a.key = b.key;

-- 方法2: 拆分热点 key
CREATE TEMPORARY TABLE hot_keys AS
SELECT key
FROM large_table
GROUP BY key
HAVING COUNT(*) > 100000;

-- 非热点数据
SELECT a.*, b.*
FROM large_table a
JOIN small_table b ON a.key = b.key
LEFT SEMI JOIN hot_keys h ON a.key = h.key
WHERE h.key IS NULL;

-- 热点数据单独处理（使用 MapJoin）
SELECT /*+ MAPJOIN(b) */ a.*, b.*
FROM large_table a
JOIN small_table b ON a.key = b.key
WHERE a.key IN (SELECT key FROM hot_keys);
```

### 2.1 Spark 小文件处理

```python
# 写入时控制文件大小
df.write \
  .option("maxRecordsPerFile", 1000000) \
  .partitionBy("date") \
  .parquet("output_path")

# 使用 coalesce 减少分区数
df.coalesce(10).write.parquet("output_path")

# 使用 repartition 增加分区并均匀分布
df.repartition(10, "key").write.parquet("output_path")

# 批量合并已有小文件
def compact_files(spark, input_path, output_path, partition_count=10):
    df = spark.read.parquet(input_path)
    df.coalesce(partition_count).write \
      .mode("overwrite") \
      .parquet(output_path)
```

## 6. Join 优化

### 6.1 Broadcast Join

```python
# Spark Broadcast Join
from pyspark.sql.functions import broadcast

# 小表广播到大表侧
large_df = spark.read.parquet("large_table")
small_df = spark.read.parquet("small_table")  # < 10MB

# 显式广播
result = large_df.join(broadcast(small_df), "key")

# 自动广播阈值配置
spark.conf.set("spark.sql.autoBroadcastJoinThreshold", "100MB")
```

### 6.2 Sort-Merge Join 优化

```python
# 确保 Join key 分布均匀
spark.conf.set("spark.sql.adaptive.skewJoin.enabled", "true")

# 使用 hint 指定 Join 策略
```

## 8. 监控与诊断

```python
# Spark UI 关键指标监控
def analyze_stage_metrics(spark, stage_id):
    """分析 Stage 执行指标"""
    status = spark.sparkContext.statusTracker()

    # 检查是否有数据倾斜
    task_infos = status.getStageInfo(stage_id).taskInfos
    durations = [t.duration for t in task_infos]

    max_duration = max(durations)
    min_duration = min(durations)
    avg_duration = sum(durations) / len(durations)

    skew_ratio = max_duration / avg_duration

    if skew_ratio > 3:
        print(f"⚠️ 数据倾斜检测: 最大/平均耗时比 = {skew_ratio:.2f}")
        print(f"   最大任务耗时: {max_duration}s")
        print(f"   平均任务耗时: {avg_duration}s")

# 使用 SparkListener 监控
class PerformanceListener(spark.SparkListener):
    def onStageCompleted(self, stageCompleted):
        stage_info = stageCompleted.stageInfo()
        print(f"Stage {stage_info.stageId()} completed in {stage_info.completionTime() - stage_info.submissionTime()}ms")
```

## 9. 总结

批处理优化核心要点：

| 优化类别 | 关键技术 | 预期收益 |
|---------|---------|---------|
| 数据倾斜 | 加盐、拆分、AQE | 减少 50-90% 执行时间 |
| 小文件 | 合并、合理分区 | 减少 NameNode 压力 |
| 存储格式 | Parquet/ORC + 压缩 | 减少 70% 存储，加速查询 |
| 并行度 | 动态分配、AQE | 资源利用率提升 |
| Join 优化 | Broadcast、SMJ | 减少 Shuffle |
| 内存管理 | G1GC、内存配比 | 减少 GC 停顿 |

优化建议执行顺序：

1. 使用列式存储格式（Parquet/ORC）
2. 启用数据压缩
3. 处理数据倾斜
4. 调优并行度和资源分配
5. 优化 Join 策略
6. 细调 JVM 参数
