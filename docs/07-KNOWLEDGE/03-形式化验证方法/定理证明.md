# 定理证明

## 概述

**定理证明（Theorem Proving）** 是一种形式化验证技术，使用严格的数学推理来证明系统满足给定的规约。与模型检验不同，定理证明可以处理无限状态系统，适用于更复杂的验证场景。

### 核心思想

定理证明基于三个核心思想：

1. **形式化规约**：
   - 使用形式化语言描述系统和性质
   - 明确系统行为和预期性质
   - 建立严格的数学基础

2. **逻辑推理**：
   - 从公理和已知定理出发
   - 使用推理规则推导新结论
   - 确保证明的严格性和正确性

3. **交互式验证**：
   - 人机协作完成复杂证明
   - 自动化处理常规推理步骤
   - 人工指导处理复杂推理

### 与模型检验的对比

| 特性 | 定理证明 | 模型检验 |
|------|----------|----------|
| 状态空间 | 无限状态 | 有限状态 |
| 自动化程度 | 半自动/交互式 | 全自动 |
| 可扩展性 | 高 | 有限（状态爆炸） |
| 反例生成 | 一般不提供 | 自动生成 |
| 适用系统 | 复杂系统、无限系统 | 有限状态系统 |
| 学习曲线 | 陡峭 | 相对平缓 |

## Coq 交互式定理证明器

### 简介

**Coq** 是一个基于构造演算（Calculus of Constructions）的交互式定理证明器，由法国INRIA开发。

**核心特性**：

- **构造性类型理论**：基于依赖类型
- **Curry-Howard对应**：命题即类型，证明即程序
- **程序提取**：从证明中提取可执行代码

### 类型理论基础

#### 构造演算（Calculus of Constructions）

类型判断：
$$ \Gamma \vdash t : T $$

其中：

- $\Gamma$ 是类型上下文
- $t$ 是项（程序或表达式）
- $T$ 是类型

**类型层次**：

- $\text{Prop}$：命题类型
- $\text{Set}$：数据类型
- $\text{Type}_i$：类型类型

#### Curry-Howard 对应

| 逻辑概念 | 类型理论概念 |
|---------|-------------|
| 命题 | 类型 |
| 证明 | 程序/项 |
| 蕴含 | 函数类型 |
| 合取 | 积类型（Product） |
| 析取 | 和类型（Sum） |
| 全称量词 | 依赖函数类型 |
| 存在量词 | 依赖积类型 |

### Coq 语法和示例

#### 基本结构

```coq
(* 定义归纳类型 *)
Inductive nat : Type :=
  | O : nat
  | S : nat -> nat.

(* 定义函数 *)
Fixpoint add (n m : nat) : nat :=
  match n with
  | O => m
  | S n' => S (add n' m)
  end.

(* 定理陈述和证明 *)
Theorem add_comm : forall n m : nat, add n m = add m n.
Proof.
  intros n m.          (* 引入变量 *)
  induction n.         (* 对 n 归纳 *)
  - simpl.             (* 化简 *)
    rewrite add_0_r.   (* 使用引理 *)
    reflexivity.       (* 自反性 *)
  - simpl.
    rewrite IHn.       (* 使用归纳假设 *)
    rewrite add_succ_r.
    reflexivity.
Qed.
```

#### 证明策略（Tactics）

| 策略 | 功能 |
|------|------|
| `intros` | 引入假设和变量 |
| `simpl` | 化简表达式 |
| `rewrite` | 使用等式重写 |
| `apply` | 应用定理或假设 |
| `induction` | 归纳法 |
| `reflexivity` | 自反性（证明 t = t） |
| `auto` | 自动尝试简单策略 |
| `omega` | 线性算术自动化 |

### Coq 示例：列表操作

```coq
(* 定义多态列表 *)
Inductive list (A : Type) : Type :=
  | nil : list A
  | cons : A -> list A -> list A.

Arguments nil {A}.
Arguments cons {A} _ _.

Infix "::" := cons (at level 60, right associativity).

(* 列表连接 *)
Fixpoint app {A : Type} (l1 l2 : list A) : list A :=
  match l1 with
  | nil => l2
  | x :: l1' => x :: app l1' l2
  end.

Infix "++" := app (right associativity, at level 60).

(* 证明列表连接的结合律 *)
Theorem app_assoc : forall A (l1 l2 l3 : list A),
  (l1 ++ l2) ++ l3 = l1 ++ (l2 ++ l3).
Proof.
  intros A l1 l2 l3.
  induction l1 as [| x l1' IH].
  - (* l1 = nil *)
    reflexivity.
  - (* l1 = x :: l1' *)
    simpl.
    rewrite IH.
    reflexivity.
Qed.
```

## Isabelle/HOL 定理证明器

### 简介

**Isabelle** 是一个通用的定理证明器，Isabelle/HOL是其最流行的逻辑（高阶逻辑）。

**核心特性**：

- **高阶逻辑（HOL）**：支持函数和谓词量化
- **强大的自动化**：auto、simp、blast等策略
- **可读性强的证明**：Isar证明语言

### 高阶逻辑（HOL）

#### 语法

**项（Terms）**：

- 变量：$x, y, z$
- 函数应用：$f(t_1, ..., t_n)$
- $\lambda$抽象：$\lambda x. t$

**公式（Formulas）**：

- 原子公式：$P(t_1, ..., t_n)$
- 逻辑连接词：$\neg, \land, \lor, \rightarrow, \leftrightarrow$
- 量词：$\forall x. \phi, \exists x. \phi$（可以量化函数和谓词）

#### 公理和规则

**外延性公理**：
$$ \forall f\ g. (\forall x. f\ x = g\ x) \rightarrow f = g $$

**推理规则**：

- **Modus Ponens**：$\frac{\phi \rightarrow \psi \quad \phi}{\psi}$
- **全称引入**：$\frac{\phi}{\forall x. \phi}$（$x$不在$\phi$的自由变量中）
- **全称消除**：$\frac{\forall x. \phi}{\phi[t/x]}$

### Isabelle 语法和示例

#### 基本结构

```isabelle
(* 理论声明 *)
theory MyTheory
  imports Main
begin

(* 类型定义 *)
datatype nat = Zero | Suc nat

(* 函数定义 *)
fun add :: "nat ⇒ nat ⇒ nat" where
  "add Zero n = n" |
  "add (Suc m) n = Suc (add m n)"

(* 定理证明 *)
theorem add_comm: "add m n = add n m"
  apply (induct m)
   apply auto
  apply (induct n)
   apply auto
  done

(* 使用Isar的结构化证明 *)
theorem add_assoc: "add (add l m) n = add l (add m n)"
proof (induct l)
  case Zero
  show ?case by simp
next
  case (Suc l)
  show ?case
  proof -
    have "add (add (Suc l) m) n = add (Suc (add l m)) n" by simp
    also have "... = Suc (add (add l m) n)" by simp
    also have "... = Suc (add l (add m n))" by (simp add: Suc)
    also have "... = add (Suc l) (add m n)" by simp
    finally show ?case .
  qed
qed

end
```

#### 证明方法

| 方法 | 功能 |
|------|------|
| `simp` | 简化，使用simp规则 |
| `auto` | 自动证明简单目标 |
| `blast` | 经典逻辑自动证明 |
| `induct` | 归纳法 |
| `rule` | 应用规则 |
| `subst` | 使用等式替换 |

## 霍尔逻辑（Hoare Logic）

### 基本概念

**霍尔三元组**：
$$ \{P\}\ C\ \{Q\} $$

含义：如果前置条件 $P$ 成立，执行程序 $C$ 后，后置条件 $Q$ 成立。

### 推理规则

#### 赋值规则

$$ \frac{}{\{Q[e/x]\}\ x := e\ \{Q\}} $$

#### 顺序规则

$$ \frac{\{P\}\ C_1\ \{R\} \quad \{R\}\ C_2\ \{Q\}}{\{P\}\ C_1; C_2\ \{Q\}} $$

#### 条件规则

$$ \frac{\{P \land B\}\ C_1\ \{Q\} \quad \{P \land \neg B\}\ C_2\ \{Q\}}{\{P\}\ \text{if } B \text{ then } C_1 \text{ else } C_2\ \{Q\}} $$

#### 循环规则

$$ \frac{\{I \land B\}\ C\ \{I\}}{\{I\}\ \text{while } B \text{ do } C\ \{I \land \neg B\}} $$

其中 $I$ 是**循环不变式**。

### 霍尔逻辑示例

```
{ n ≥ 0 }
    i := 0;
    s := 0;
    while i < n do
        s := s + i;
        i := i + 1
    end
{ s = n * (n-1) / 2 }

循环不变式：s = i * (i-1) / 2 ∧ i ≤ n
```

## 不变式证明

### 基本概念

**不变式（Invariant）**：在程序执行过程中始终保持成立的性质。

**类型**：

- **类型不变式**：变量类型相关的性质
- **结构不变式**：数据结构相关的性质
- **语义不变式**：程序语义相关的性质

### 不变式证明方法

#### 归纳证明框架

要证明 $I$ 是不变式，需要证明：

1. **初始条件**：$\text{Init} \implies I$
2. **保持条件**：$I \land \text{Next} \implies I'$

#### 示例：计数器不变式

```tla
MODULE Counter

VARIABLES count

Init == count = 0

Increment == count < 10 /\ count' = count + 1

Reset == count >= 10 /\ count' = 0

Next == Increment \/ Reset

Spec == Init /\ [][Next]_count

(* 不变式 *)
TypeInvariant == count \in 0..10

(* 不变式证明 *)
THEOREM Invariant == Spec => []TypeInvariant
```

证明：

1. **Init => TypeInvariant**：count = 0 ∈ 0..10 ✓
2. **TypeInvariant ∧ Next => TypeInvariant'**：
   - Increment: count < 10 ∧ count' = count + 1 ⇒ count' ∈ 1..10 ✓
   - Reset: count >= 10 ∧ count' = 0 ⇒ count' = 0 ∈ 0..10 ✓

## 类型理论

### 简单类型

**基本类型**：

- 基本类型：$\text{int}, \text{bool}, \text{string}$
- 函数类型：$\tau_1 \rightarrow \tau_2$
- 积类型：$\tau_1 \times \tau_2$

### 依赖类型

**定义**：类型可以依赖于值。

**示例**：

```
Vector n  -- 长度为 n 的向量
Matrix m n  -- m × n 的矩阵
```

### 类型安全

**类型安全定理**：
$$ \text{WellTyped}(e) \implies \text{NoRuntimeError}(e) $$

**类型可靠性**：

- **进展性（Progress）**：良类型的非值项可以继续规约
- **保持性（Preservation）**：规约保持类型

## 程序验证

### 验证流程

```
1. 程序规约 → 2. 不变式识别 → 3. 验证条件生成 → 4. 定理证明
     ↓              ↓                  ↓                ↓
 霍尔三元组    循环不变式        VC生成器        证明器
```

### 验证条件生成（VCG）

**功能**：自动生成需要证明的验证条件。

**示例**：

```
程序：
    { x ≥ 0 }
    y := 0;
    while x > 0 do
        y := y + x;
        x := x - 1
    end
    { y = x₀ * (x₀ + 1) / 2 }

验证条件：
1. x ≥ 0 ∧ y = 0 → I(x, y, x₀)
2. I(x, y, x₀) ∧ x > 0 → I(x-1, y+x, x₀)
3. I(x, y, x₀) ∧ x ≤ 0 → y = x₀ * (x₀ + 1) / 2
```

## 总结

定理证明是一种强大的形式化验证技术，适用于复杂系统和无限状态系统的验证。Coq和Isabelle是目前最流行的交互式定理证明器，分别基于构造演算和高阶逻辑。通过霍尔逻辑和不变式证明，可以验证程序的正确性。虽然定理证明的学习曲线较陡，但它在安全关键系统的验证中具有不可替代的作用。

**关键优势**：

- 可以处理无限状态系统
- 提供数学级别的正确性保证
- 可验证复杂算法和系统

**主要挑战**：

- 需要专业的数学和逻辑知识
- 证明过程通常需要人工指导
- 时间和成本较高
