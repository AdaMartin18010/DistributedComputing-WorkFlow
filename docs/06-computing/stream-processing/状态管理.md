# 流处理状态管理详解

## 1. 状态管理概述

状态管理是流处理系统的核心能力，用于在持续的数据流处理中维护和访问中间计算结果。良好的状态管理支持容错恢复、扩展性和复杂的有状态计算。

```

│                   流处理状态类型                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                    Operator State                            │   │
│  │  (算子状态，与特定算子实例绑定)                                │   │
│  │                                                             │   │
│  │  List State         ──▶ 列表形式的状态，均匀分布              │   │
│  │  Union List State   ──▶ 列表形式，广播到所有子任务            │   │
│  │  Broadcast State    ──▶ 广播状态，所有子任务相同              │   │
│  │                                                             │   │
```

## 3. Keyed State 使用

### 3.1 ValueState

```java
public class CountFunction extends KeyedProcessFunction<String, Event, Result> {
    
    private ValueState<Long> countState;
    private ValueState<Long> lastModifiedState;
    
    @Override
    public void open(Configuration parameters) {
        // 定义状态描述符
        ValueStateDescriptor<Long> countDescriptor = 
            new ValueStateDescriptor<>("count", Types.LONG);
        ValueStateDescriptor<Long> timeDescriptor = 
            new ValueStateDescriptor<>("lastModified", Types.LONG);
        
        // 获取状态句柄
        countState = getRuntimeContext().getState(countDescriptor);
        lastModifiedState = getRuntimeContext().getState(timeDescriptor);
    }
    
    @Override
    public void processElement(Event value, Context ctx, Collector<Result> out) 
            throws Exception {
        
        // 读取状态
        Long current = countState.value();
        if (current == null) {
            current = 0L;
        }
        
        // 更新状态
        current++;
        countState.update(current);
        lastModifiedState.update(ctx.timestamp());
        
        out.collect(new Result(value.getKey(), current));
    }
}
```

### 3.2 ListState

```java
public class TopNFunction extends KeyedProcessFunction<String, Event, List<Event>> {
    
    private ListState<Event> eventListState;
    private final int N;
    
    public TopNFunction(int n) {
        this.N = n;
    }
    
    @Override
    public void open(Configuration parameters) {
        ListStateDescriptor<Event> descriptor = 
            new ListStateDescriptor<>("events", Event.class);
        eventListState = getRuntimeContext().getListState(descriptor);
    }
    
    @Override
    public void processElement(Event value, Context ctx, Collector<List<Event>> out) 
            throws Exception {
        
        // 添加元素
        eventListState.add(value);
        
        // 获取所有元素
        Iterable<Event> events = eventListState.get();
        List<Event> sortedEvents = new ArrayList<>();
        events.forEach(sortedEvents::add);
        
        // 只保留 Top N
        if (sortedEvents.size() > N) {
            sortedEvents.sort(Comparator.comparing(Event::getScore).reversed());
            eventListState.clear();
            for (int i = 0; i < N; i++) {
                eventListState.add(sortedEvents.get(i));
            }
        }
        
        out.collect(sortedEvents);
    }
}
```

### 3.3 MapState

```java
public class UserSessionFunction extends KeyedProcessFunction<String, Event, Session> {
    
    private MapState<String, Long> pageVisitState;
    
    @Override
    public void open(Configuration parameters) {
        MapStateDescriptor<String, Long> descriptor = 
            new MapStateDescriptor<>("pageVisits", String.class, Long.class);
        pageVisitState = getRuntimeContext().getMapState(descriptor);
    }
    
    @Override
    public void processElement(Event value, Context ctx, Collector<Session> out) 
            throws Exception {
        
        String page = value.getPage();
        Long count = pageVisitState.get(page);
        
        if (count == null) {
            count = 0L;
        }
        
        pageVisitState.put(page, count + 1);
        
        // 遍历 MapState
        Map<String, Long> allVisits = new HashMap<>();
        for (Map.Entry<String, Long> entry : pageVisitState.entries()) {
            allVisits.put(entry.getKey(), entry.getValue());
        }
        
        out.collect(new Session(value.getUserId(), allVisits));
    }
}
```

## 4. Operator State

```java
// CheckpointedFunction 接口实现 Operator State
public class BufferingSink implements SinkFunction<Tuple2<String, Integer>>, 
                                     CheckpointedFunction {
    
    private final int threshold;
    private List<Tuple2<String, Integer>> bufferedElements;
    
    // 算子状态
    private ListState<Tuple2<String, Integer>> checkpointState;
    
    public BufferingSink(int threshold) {
        this.threshold = threshold;
        this.bufferedElements = new ArrayList<>();
    }
    
    @Override
    public void invoke(Tuple2<String, Integer> value, Context context) {
        bufferedElements.add(value);
        
        if (bufferedElements.size() >= threshold) {
            flush();
        }
    }
    
    private void flush() {
        // 批量写入外部系统
        for (Tuple2<String, Integer> element : bufferedElements) {
            // send to external system
        }
        bufferedElements.clear();
    }
    
    @Override
    public void snapshotState(FunctionSnapshotContext context) throws Exception {
        checkpointState.clear();
        for (Tuple2<String, Integer> element : bufferedElements) {
            checkpointState.add(element);
        }
    }
    
    @Override
    public void initializeState(FunctionInitializationContext context) throws Exception {
        ListStateDescriptor<Tuple2<String, Integer>> descriptor = 
            new ListStateDescriptor<>("buffered-elements", 
                TypeInformation.of(new TypeHint<Tuple2<String, Integer>>() {}));
        
        checkpointState = context.getOperatorStateStore().getListState(descriptor);
        
        // 从 Checkpoint 恢复
        if (context.isRestored()) {
            for (Tuple2<String, Integer> element : checkpointState.get()) {
                bufferedElements.add(element);
            }
        }
    }
}
```

## 5. 广播状态

```java
// 广播流：配置更新
BroadcastStream<Config> broadcastConfigStream = configStream
    .broadcast(new MapStateDescriptor<>("config", String.class, Config.class));

// 主处理流连接广播流
DataStream<Result> result = mainStream
    .connect(broadcastConfigStream)
    .process(new BroadcastProcessFunction<Event, Config, Result>() {
        
        private MapStateDescriptor<String, Config> configStateDescriptor = 
            new MapStateDescriptor<>("config", String.class, Config.class);
        
        @Override
        public void processElement(Event value, ReadOnlyContext ctx, 
                                  Collector<Result> out) throws Exception {
            // 只读访问广播状态
            ReadOnlyBroadcastState<String, Config> configState = 
                ctx.getBroadcastState(configStateDescriptor);
            
            Config config = configState.get(value.getType());
            if (config != null) {
                out.collect(processWithConfig(value, config));
            }
        }
        
        @Override
        public void processBroadcastElement(Config value, Context ctx, 
                                           Collector<Result> out) throws Exception {
            // 更新广播状态
            BroadcastState<String, Config> configState = 
                ctx.getBroadcastState(configStateDescriptor);
            configState.put(value.getType(), value);
        }
    });
```

```java
StateTtlConfig ttlConfig = StateTtlConfig
    .newBuilder(Time.minutes(10))  // 过期时间
    .setUpdateType(StateTtlConfig.UpdateType.OnCreateAndWrite)  // 更新策略

    new ValueStateDescriptor<>("myState", String.class);
```

## 9. 不同框架状态管理对比

| 特性 | Flink | Spark Streaming | Kafka Streams |
|------|-------|-----------------|---------------|
| **状态类型** | Keyed/Operator | DStream 状态 | KTable/KStream |
| **状态后端** | Memory/FS/RocksDB | Memory | RocksDB |
| **Checkpoint** | 原生支持 | RDD 检查点 | 日志压缩 |
| **TTL 支持** | 完整 | 有限 | 有限 |
| **增量 Checkpoint** | 支持 | 不支持 | 支持 |
| **Queryable State** | 支持 | 不支持 | 支持 (IQ) |

## 10. 最佳实践

1. **选择合适的状态后端**：小状态用 Memory，大状态用 RocksDB
2. **启用增量 Checkpoint**：减少 Checkpoint 时间和存储
3. **设置合理的 TTL**：避免状态无限增长
4. **使用状态描述符**：统一管理状态配置
5. **监控状态大小**：及时发现状态膨胀问题
6. **定期清理过期状态**：使用 TTL 或定时器
