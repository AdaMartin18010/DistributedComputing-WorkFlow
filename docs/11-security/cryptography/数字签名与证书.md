# 数字签名与证书

## 概述

数字签名是公钥密码学的重要应用，用于验证消息的真实性、完整性和不可否认性。证书则是将公钥与身份信息绑定的数据结构，由可信的证书颁发机构（CA）签发。这两者构成了现代互联网信任基础设施的基石。

---

## 一、数字签名原理

### 1.1 数字签名 vs 手写签名

```
┌─────────────────────────────────────────────────────────────┐
│              数字签名 vs 手写签名对比                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  特性              手写签名          数字签名                 │
│  ─────────────────────────────────────────────────────────  │
│  验证方式          人工比对          数学算法验证             │
│  绑定内容          纸质文档          任意数字数据             │
│  复制难度          容易伪造          几乎不可伪造             │
│  传输方式          物理传递          网络传输                 │
│  不可否认性        可否认            密码学保证               │
│  完整性保护        无                内置篡改检测             │
│                                                             │
│  数字签名核心特性：                                           │
│  ✓ 真实性 - 确认签名者身份                                    │
│  ✓ 完整性 - 检测数据是否被篡改                                │
│  ✓ 不可否认性 - 签名者无法否认签名                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 数字签名工作原理

```
数字签名流程：

签名过程：
┌─────────────┐
│   消息 M    │
└──────┬──────┘
       │
       ▼
┌─────────────┐     ┌─────────────┐
│   哈希函数   │────▶│  消息摘要 H  │
│  (SHA-256)  │     │  (固定长度)  │
└─────────────┘     └──────┬──────┘
                           │
                    ┌──────┴──────┐
                    │  私钥签名    │
                    │  Sign(H,    │
                    │  PrivateKey)│
                    └──────┬──────┘
                           │
                           ▼
                    ┌─────────────┐
                    │   数字签名 S │
                    └─────────────┘

传输：发送 M + S

验证过程：
                    ┌─────────────┐
                    │   消息 M'   │ (可能已被篡改)
                    └──────┬──────┘
                           │
       ┌───────────────────┼───────────────────┐
       │                   │                   │
       ▼                   ▼                   ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   哈希函数   │     │   数字签名 S │     │  公钥验证   │
│  (SHA-256)  │     │             │────▶│Verify(S,   │
└──────┬──────┘     └─────────────┘     │PublicKey)   │
       │                                └──────┬──────┘
       ▼                                       │
┌─────────────┐                                │
│  消息摘要 H' │◀───────────────────────────────┘
│  (计算得到)  │       比较 H' == H?
└─────────────┘

如果 H' == H 且签名验证通过，则：
• 消息未被篡改（完整性）
• 签名者持有对应私钥（真实性）
```

### 1.3 数字签名算法

```
┌─────────────────────────────────────────────────────────────┐
│                    数字签名算法对比                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  RSA签名                                                     │
│  ─────────────────────────────────────────────────────────  │
│  签名: S = M^d mod n  (使用私钥)                             │
│  验证: M = S^e mod n  (使用公钥)                             │
│                                                             │
│  优点：历史悠久，广泛支持                                     │
│  缺点：签名和密钥较大，速度较慢                               │
│  推荐：RSA-PSS with SHA-256, 2048+ bits                     │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ECDSA (椭圆曲线数字签名算法)                                 │
│  ─────────────────────────────────────────────────────────  │
│  基于椭圆曲线离散对数难题                                     │
│                                                             │
│  优点：签名小（64字节），速度快，安全性高                      │
│  缺点：实现复杂，随机数重用会导致私钥泄露                      │
│  推荐：ECDSA with P-256 curve, SHA-256                       │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Ed25519                                                     │
│  ─────────────────────────────────────────────────────────  │
│  基于Curve25519的现代签名方案                                  │
│                                                             │
│  优点：设计安全（不依赖随机数），速度快，实现简单               │
│  缺点：相对较新，旧系统可能不支持                             │
│  推荐：首选方案，除非需要兼容旧系统                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 二、数字签名实现

### 2.1 Ed25519签名实现（推荐）

```python
import nacl.signing
import nacl.encoding
import hashlib
import base64
from typing import Tuple

class Ed25519Signature:
    """
    Ed25519数字签名实现
    现代、安全、高效的签名方案
    """

    def __init__(self):
        self.signing_key = None
        self.verify_key = None

    def generate_keypair(self) -> Tuple[str, str]:
        """
        生成Ed25519密钥对

        Returns:
            (私钥_base64, 公钥_base64)
        """
        # 生成签名密钥对
        self.signing_key = nacl.signing.SigningKey.generate()
        self.verify_key = self.signing_key.verify_key

        # 编码为可传输格式
        private_key_b64 = self.signing_key.encode(
            encoder=nacl.encoding.Base64Encoder
        ).decode()
        public_key_b64 = self.verify_key.encode(
            encoder=nacl.encoding.Base64Encoder
        ).decode()

        return private_key_b64, public_key_b64

    def load_private_key(self, private_key_b64: str):
        """加载私钥"""
        self.signing_key = nacl.signing.SigningKey(
            private_key_b64,
            encoder=nacl.encoding.Base64Encoder
        )
        self.verify_key = self.signing_key.verify_key

    def load_public_key(self, public_key_b64: str):
        """加载公钥"""
        self.verify_key = nacl.signing.VerifyKey(
            public_key_b64,
            encoder=nacl.encoding.Base64Encoder
        )

    def sign(self, message: bytes) -> bytes:
        """
        对消息进行签名

        Args:
            message: 待签名的原始消息

        Returns:
            签名（64字节）
        """
        if not self.signing_key:
            raise ValueError("Private key not loaded")

        signed = self.signing_key.sign(message)
        # signed包含原始消息+签名，我们只返回签名部分
        return signed.signature

    def sign_detached(self, message: bytes) -> str:
        """
        对消息签名并返回base64编码的签名
        """
        signature = self.sign(message)
        return base64.b64encode(signature).decode()

    def verify(self, message: bytes, signature: bytes) -> bool:
        """
        验证签名

        Args:
            message: 原始消息
            signature: 签名

        Returns:
            验证成功返回True
        """
        if not self.verify_key:
            raise ValueError("Public key not loaded")

        try:
            self.verify_key.verify(message, signature)
            return True
        except nacl.exceptions.BadSignatureError:
            return False

    def verify_b64(self, message: bytes, signature_b64: str) -> bool:
        """验证base64编码的签名"""
        signature = base64.b64decode(signature_b64)
        return self.verify(message, signature)


# 扩展：带时间戳的签名（防止重放攻击）
import time
import json

class TimestampedSignature(Ed25519Signature):
    """
    带时间戳的数字签名
    防止签名重放攻击
    """

    def __init__(self, ttl_seconds: int = 300):
        super().__init__()
        self.ttl = ttl_seconds  # 签名有效期

    def sign_with_timestamp(self, message: bytes) -> dict:
        """
        创建带时间戳的签名

        Returns:
            {
                'message': base64编码的消息,
                'timestamp': 时间戳,
                'signature': 对(消息+时间戳)的签名
            }
        """
        timestamp = int(time.time())

        # 将时间戳附加到消息
        message_with_ts = json.dumps({
            'msg': base64.b64encode(message).decode(),
            'ts': timestamp
        }).encode()

        signature = self.sign(message_with_ts)

        return {
            'message': base64.b64encode(message).decode(),
            'timestamp': timestamp,
            'signature': base64.b64encode(signature).decode()
        }

    def verify_with_timestamp(self, signed_data: dict) -> tuple[bool, str]:
        """
        验证带时间戳的签名

        Returns:
            (是否有效, 原因)
        """
        # 检查时间戳
        current_time = int(time.time())
        timestamp = signed_data['timestamp']

        if current_time - timestamp > self.ttl:
            return False, "Signature expired"

        if timestamp > current_time + 60:  # 允许1分钟时钟偏差
            return False, "Timestamp in future"

        # 重构消息
        message_with_ts = json.dumps({
            'msg': signed_data['message'],
            'ts': timestamp
        }).encode()

        signature = base64.b64decode(signed_data['signature'])

        if not self.verify(message_with_ts, signature):
            return False, "Invalid signature"

        return True, "Valid"


# 使用示例
if __name__ == "__main__":
    print("=== Ed25519 Digital Signature ===\n")

    # 创建签名实例
    signer = Ed25519Signature()

    # 生成密钥对
    private_key, public_key = signer.generate_keypair()
    print(f"Private Key: {private_key[:30]}...")
    print(f"Public Key:  {public_key}")

    # 签名消息
    message = b"Transfer $1000 from Alice to Bob"
    signature = signer.sign_detached(message)
    print(f"\nMessage: {message.decode()}")
    print(f"Signature: {signature}")
    print(f"Signature length: {len(base64.b64decode(signature))} bytes")

    # 验证签名
    is_valid = signer.verify_b64(message, signature)
    print(f"\nSignature valid: {is_valid}")

    # 验证篡改后的消息
    tampered_message = b"Transfer $10000 from Alice to Bob"
    is_invalid = signer.verify_b64(tampered_message, signature)
    print(f"Tampered message valid: {is_invalid}")

    # 时间戳签名示例
    print("\n=== Timestamped Signature ===")
    ts_signer = TimestampedSignature(ttl_seconds=60)
    ts_signer.load_private_key(private_key)

    ts_signed = ts_signer.sign_with_timestamp(message)
    print(f"Signed data: {json.dumps(ts_signed, indent=2)}")

    # 验证
    ts_signer.load_public_key(public_key)
    valid, reason = ts_signer.verify_with_timestamp(ts_signed)
    print(f"\nVerification: {valid} ({reason})")
```

### 2.2 ECDSA签名实现

```python
from cryptography.hazmat.primitives.asymmetric import ec
from cryptography.hazmat.primitives import hashes, serialization
from cryptography.hazmat.backends import default_backend
import base64

class ECDSASignature:
    """
    ECDSA数字签名实现
    使用P-256曲线，广泛兼容
    """

    def __init__(self):
        self.private_key = None
        self.public_key = None

    def generate_keypair(self):
        """生成ECDSA密钥对（P-256曲线）"""
        self.private_key = ec.generate_private_key(
            ec.SECP256R1(),
            default_backend()
        )
        self.public_key = self.private_key.public_key()

        # 导出密钥
        private_pem = self.private_key.private_bytes(
            encoding=serialization.Encoding.PEM,
            format=serialization.PrivateFormat.PKCS8,
            encryption_algorithm=serialization.NoEncryption()
        )

        public_pem = self.public_key.public_bytes(
            encoding=serialization.Encoding.PEM,
            format=serialization.PublicFormat.SubjectPublicKeyInfo
        )

        return private_pem, public_pem

    def load_private_key(self, private_pem: bytes):
        """加载私钥"""
        self.private_key = serialization.load_pem_private_key(
            private_pem,
            password=None,
            backend=default_backend()
        )
        self.public_key = self.private_key.public_key()

    def load_public_key(self, public_pem: bytes):
        """加载公钥"""
        self.public_key = serialization.load_pem_public_key(
            public_pem,
            backend=default_backend()
        )

    def sign(self, message: bytes) -> bytes:
        """
        ECDSA签名（使用确定性签名防止随机数重用攻击）
        """
        if not self.private_key:
            raise ValueError("Private key not loaded")

        # 使用ECDSA with SHA-256
        signature = self.private_key.sign(
            message,
            ec.ECDSA(hashes.SHA256())
        )
        return signature

    def verify(self, message: bytes, signature: bytes) -> bool:
        """验证ECDSA签名"""
        if not self.public_key:
            raise ValueError("Public key not loaded")

        try:
            self.public_key.verify(
                signature,
                message,
                ec.ECDSA(hashes.SHA256())
            )
            return True
        except Exception:
            return False
```

### 3.2 证书链与信任模型

```
PKI证书链结构：

根CA（自签名）
    │
    │ 签名
    ▼
┌──────────────────────────────────────────────────────┐
│ Root CA Certificate                                  │
│ Subject: CN=Global Root CA                           │
│ Issuer:  CN=Global Root CA  ← 自签名                 │
│ Trust: 系统内置信任                                   │
└──────────────────────────────────────────────────────┘
    │
    │ 签名中间CA证书
    ▼
┌──────────────────────────────────────────────────────┐
│ Intermediate CA Certificate                          │
│ Subject: CN=Secure Intermediate CA                   │
│ Issuer:  CN=Global Root CA                           │
│ Trust: 由根CA签名验证                                 │
└──────────────────────────────────────────────────────┘
    │
    │ 签名叶子证书
    ▼
┌──────────────────────────────────────────────────────┐
│ End-Entity Certificate                               │
│ Subject: CN=www.example.com                          │
│ Issuer:  CN=Secure Intermediate CA                   │
│ Trust: 由中间CA签名验证                               │
└──────────────────────────────────────────────────────┘

浏览器/操作系统内置数百个根CA证书
                 │
                 ▼
           预置信任存储
     (Trusted Root Store)

证书验证流程：
1. 验证叶子证书签名（使用中间CA公钥）
2. 验证中间CA签名（使用根CA公钥）
3. 验证根CA在信任存储中
4. 检查证书有效期
5. 检查证书吊销状态（CRL/OCSP）
6. 验证域名匹配
```

### 3.3 证书操作实现

```python
from cryptography import x509
from cryptography.x509.oid import NameOID
from cryptography.hazmat.primitives import hashes, serialization
from cryptography.hazmat.primitives.asymmetric import rsa
from cryptography.hazmat.backends import default_backend
import datetime
import ipaddress

class CertificateManager:
    """
    X.509证书管理
    支持生成、签名、验证证书
    """

    def __init__(self):
        self.private_key = None
        self.certificate = None

    def generate_ca_certificate(self,
                                 common_name: str,
                                 validity_days: int = 3650) -> tuple:
        """
        生成自签名CA证书

        Args:
            common_name: CA名称
            validity_days: 有效期（天）

        Returns:
            (私钥PEM, 证书PEM)
        """
        # 生成RSA密钥对
        self.private_key = rsa.generate_private_key(
            public_exponent=65537,
            key_size=4096,
            backend=default_backend()
        )

        # 构建主题和颁发者（自签名，两者相同）
        subject = issuer = x509.Name([
            x509.NameAttribute(NameOID.COUNTRY_NAME, u"CN"),
            x509.NameAttribute(NameOID.ORGANIZATION_NAME, u"Test CA"),
            x509.NameAttribute(NameOID.COMMON_NAME, common_name),
        ])

        # 构建证书
        self.certificate = x509.CertificateBuilder().subject_name(
            subject
        ).issuer_name(
            issuer
        ).public_key(
            self.private_key.public_key()
        ).serial_number(
            x509.random_serial_number()
        ).not_valid_before(
            datetime.datetime.utcnow()
        ).not_valid_after(
            datetime.datetime.utcnow() + datetime.timedelta(days=validity_days)
        ).add_extension(
            # CA基本约束
            x509.BasicConstraints(ca=True, path_length=None),
            critical=True
        ).add_extension(
            # 密钥用途
            x509.KeyUsage(
                digital_signature=True,
                content_commitment=False,
                key_encipherment=False,
                data_encipherment=False,
                key_agreement=False,
                key_cert_sign=True,  # 可以签名证书
                crl_sign=True,       # 可以签名CRL
                encipher_only=False,
                decipher_only=False
            ),
            critical=True
        ).sign(self.private_key, hashes.SHA256(), default_backend())

        # 导出PEM格式
        private_pem = self.private_key.private_bytes(
            encoding=serialization.Encoding.PEM,
            format=serialization.PrivateFormat.TraditionalOpenSSL,
            encryption_algorithm=serialization.NoEncryption()
        )

        cert_pem = self.certificate.public_bytes(
            encoding=serialization.Encoding.PEM
        )

        return private_pem, cert_pem

    def sign_server_certificate(self,
                                 ca_private_key,
                                 ca_certificate,
                                 common_name: str,
                                 san_dns_names: list = None,
                                 san_ip_addresses: list = None,
                                 validity_days: int = 365) -> bytes:
        """
        使用CA证书签名服务器证书

        Args:
            ca_private_key: CA私钥
            ca_certificate: CA证书
            common_name: 服务器域名
            san_dns_names: 主题备用名称（DNS）
            san_ip_addresses: 主题备用名称（IP）
            validity_days: 有效期
        """
        # 生成服务器密钥对
        server_key = rsa.generate_private_key(
            public_exponent=65537,
            key_size=2048,
            backend=default_backend()
        )

        # 主题
        subject = x509.Name([
            x509.NameAttribute(NameOID.COUNTRY_NAME, u"CN"),
            x509.NameAttribute(NameOID.ORGANIZATION_NAME, u"Test Server"),
            x509.NameAttribute(NameOID.COMMON_NAME, common_name),
        ])

        # 构建SAN列表
        san_list = []
        if san_dns_names:
            for dns in san_dns_names:
                san_list.append(x509.DNSName(dns))
        if san_ip_addresses:
            for ip in san_ip_addresses:
                san_list.append(x509.IPAddress(ipaddress.ip_address(ip)))

        # 构建证书
        cert_builder = x509.CertificateBuilder().subject_name(
            subject
        ).issuer_name(
            ca_certificate.subject
        ).public_key(
            server_key.public_key()
        ).serial_number(
            x509.random_serial_number()
        ).not_valid_before(
            datetime.datetime.utcnow()
        ).not_valid_after(
            datetime.datetime.utcnow() + datetime.timedelta(days=validity_days)
        ).add_extension(
            x509.BasicConstraints(ca=False, path_length=None),
            critical=True
        ).add_extension(
            x509.KeyUsage(
                digital_signature=True,
                key_encipherment=True,
                key_cert_sign=False,
                crl_sign=False,
                content_commitment=False,
                data_encipherment=False,
                key_agreement=False,
                encipher_only=False,
                decipher_only=False
            ),
            critical=True
        ).add_extension(
            x509.ExtendedKeyUsage([
                x509.ExtendedKeyUsageOID.SERVER_AUTH,
                x509.ExtendedKeyUsageOID.CLIENT_AUTH
            ]),
            critical=False
        )

        # 添加SAN扩展
        if san_list:
            cert_builder = cert_builder.add_extension(
                x509.SubjectAlternativeName(san_list),
                critical=False
            )

        server_cert = cert_builder.sign(
            ca_private_key,
            hashes.SHA256(),
            default_backend()
        )

        # 导出服务器私钥和证书
        server_key_pem = server_key.private_bytes(
            encoding=serialization.Encoding.PEM,
            format=serialization.PrivateFormat.TraditionalOpenSSL,
            encryption_algorithm=serialization.NoEncryption()
        )

        server_cert_pem = server_cert.public_bytes(
            encoding=serialization.Encoding.PEM
        )

        return server_key_pem + server_cert_pem

    @staticmethod
    def verify_certificate_chain(cert_pem: bytes,
                                  trusted_cas: list) -> bool:
        """
        验证证书链

        Args:
            cert_pem: 待验证证书
            trusted_cas: 受信任的CA证书列表
        """
        try:
            cert = x509.load_pem_x509_certificate(cert_pem, default_backend())

            # 检查有效期
            now = datetime.datetime.utcnow()
            if now < cert.not_valid_before or now > cert.not_valid_after:
                return False

            # 验证签名
            issuer_found = False
            for ca_pem in trusted_cas:
                ca_cert = x509.load_pem_x509_certificate(ca_pem, default_backend())
                if ca_cert.subject == cert.issuer:
                    # 验证签名
                    ca_cert.public_key().verify(
                        cert.signature,
                        cert.tbs_certificate_bytes,
                        cert.signature_hash_algorithm
                    )
                    issuer_found = True
                    break

            return issuer_found

        except Exception as e:
            print(f"Verification failed: {e}")
            return False
```

## 五、最佳实践

### 5.1 证书管理清单

- [ ] 使用2048位+ RSA或P-256+ ECC密钥
- [ ] 使用SHA-256或更强的哈希算法
- [ ] 正确配置SAN扩展（替代CN字段）
- [ ] 设置合理的证书有效期（建议不超过13个月）
- [ ] 启用证书透明度（CT）
- [ ] 配置OCSP Stapling提高性能
- [ ] 实施证书固定（Certificate Pinning）用于敏感应用
- [ ] 建立证书过期监控和自动续期流程

```python
# 使用HSM或KMS保护私钥的安全实践

class SecureKeyStorage:
    """
    私钥安全存储最佳实践
    """

    @staticmethod
    def encrypt_private_key(private_key_pem: bytes,
                           password: str) -> bytes:
        """
        使用强加密保护私钥文件
        """
        from cryptography.hazmat.primitives import serialization

        private_key = serialization.load_pem_private_key(
            private_key_pem, password=None
        )

        # 使用PBKDF2 + AES-256-CBC加密
        encrypted_pem = private_key.private_bytes(
            encoding=serialization.Encoding.PEM,
            format=serialization.PrivateFormat.PKCS8,
            encryption_algorithm=serialization.BestAvailableEncryption(
                password.encode()
            )
        )

        return encrypted_pem

    @staticmethod
    def generate_csr(private_key, common_name: str,
                     san_dns_names: list = None) -> bytes:
        """
        生成证书签名请求（CSR）
        私钥不离开本地，只提交CSR给CA
        """
        from cryptography import x509
        from cryptography.x509.oid import NameOID

        csr_builder = x509.CertificateSigningRequestBuilder()
        csr_builder = csr_builder.subject_name(x509.Name([
            x509.NameAttribute(NameOID.COUNTRY_NAME, u"CN"),
            x509.NameAttribute(NameOID.COMMON_NAME, common_name),
        ]))

        if san_dns_names:
            csr_builder = csr_builder.add_extension(
                x509.SubjectAlternativeName(
                    [x509.DNSName(name) for name in san_dns_names]
                ),
                critical=False
            )

        csr = csr_builder.sign(private_key, hashes.SHA256())
        return csr.public_bytes(serialization.Encoding.PEM)
```
