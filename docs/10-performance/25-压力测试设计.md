# 压力测试设计

## 核心概念

### 1. 压力测试类型

**测试类型对比**：

| 类型 | 目标 | 方法 | 关注指标 |
|------|------|------|----------|
| 负载测试 | 验证正常负载 | 逐步加压 | 响应时间、吞吐量 |
| 压力测试 | 找到极限 | 超负载运行 | 错误率、恢复时间 |
| 稳定性测试 | 长时间稳定 | 持续运行 | 内存泄漏、性能衰减 |
| 容量测试 | 确定上限 | 阶梯加压 | 最大承载量 |
| 峰值测试 | 验证突发 | 瞬间加压 | 弹性、降级 |

### 2. 压力测试目标

### 压力测试方案设计

**1. 测试场景设计**：

```yaml
# 压测场景配置
scenarios:
  - name: normal_load
    description: 正常负载测试
    stages:
      - duration: 5m
        target: 1000    # 1000并发
      - duration: 30m
        target: 1000    # 稳定运行
      - duration: 5m
        target: 0       # 逐步减压

  - name: stress_test
    description: 压力测试
    stages:
      - duration: 2m
        target: 1000
      - duration: 2m
        target: 2000
      - duration: 2m
        target: 4000   # 超负载
      - duration: 2m
        target: 8000   # 极限负载
      - duration: 5m
        target: 0      # 恢复

  - name: spike_test
    description: 峰值测试
    stages:
      - duration: 1m
        target: 1000
      - duration: 10s
        target: 10000  # 瞬间加压
      - duration: 5m
        target: 1000   # 恢复正常
```

**2. 测试脚本设计**：

```javascript
// k6 压测脚本
import http from 'k6/http';
import { check, sleep, group } from 'k6';

export const options = {
  scenarios: {
    browse: {
      executor: 'constant-vus',
      vus: 100,
      duration: '30m',
      exec: 'browseProducts',
    },
    order: {
      executor: 'ramping-vus',
      startVUs: 0,
      stages: [
        { duration: '5m', target: 100 },
        { duration: '10m', target: 100 },
        { duration: '5m', target: 0 },
      ],
      exec: 'placeOrder',
    },
  },
  thresholds: {
    http_req_duration: ['p(95)<200', 'p(99)<500'],
    http_req_failed: ['rate<0.01'],
  },
};

export function browseProducts() {
  let res = http.get('https://api.example.com/products');
  check(res, {
    'status is 200': (r) => r.status === 200,
    'response time < 200ms': (r) => r.timings.duration < 200,
  });
  sleep(1);
}

export function placeOrder() {
  let payload = JSON.stringify({
    productId: Math.floor(Math.random() * 10000),
    quantity: 1,
  });

  let res = http.post('https://api.example.com/orders', payload, {
    headers: { 'Content-Type': 'application/json' },
  });

  check(res, {
    'status is 200 or 201': (r) => r.status === 200 || r.status === 201,
    'response time < 500ms': (r) => r.timings.duration < 500,
  });
  sleep(3);
}
```

class LoadTestMonitor:
    """压测监控"""

    def __init__(self):
        self.metrics = {
            'qps': [],
            'latency_p99': [],
            'error_rate': [],
            'cpu_usage': [],
            'memory_usage': [],
        }

    def collect(self, interval=10):
        """定期采集指标"""
        while self.running:
            self.metrics['qps'].append(self.get_qps())
            self.metrics['latency_p99'].append(self.get_latency_p99())
            self.metrics['error_rate'].append(self.get_error_rate())
            self.metrics['cpu_usage'].append(self.get_cpu_usage())
            self.metrics['memory_usage'].append(self.get_memory_usage())

            # 异常检测

                self.alert('Error rate exceeded 1%')

            if self.metrics['latency_p99'][-1] > 1000:
                self.alert('P99 latency exceeded 1s')

            time.sleep(interval)

## 案例数据

### 电商平台压测案例

**场景**：双11大促压测

**压测方案**：

```
目标：验证系统支持10万QPS

测试阶段：
1. 基线测试：1000 QPS，验证基础功能
2. 负载测试：逐步加压到5万QPS
3. 压力测试：8万QPS持续10分钟
4. 峰值测试：瞬间加压到10万QPS
5. 稳定性测试：5万QPS运行24小时

测试结果：
- 最大承载：8.5万QPS（CPU饱和）
- 响应时间P99：200ms @ 5万QPS
- 错误率：<0.1% @ 8万QPS
- 恢复时间：2分钟

发现的问题：
1. 数据库连接池不足 → 连接池扩容
2. Redis热点key → 本地缓存优化
3. 日志输出阻塞 → 异步日志改造
```

## 优化前后对比

| 指标 | 压测前 | 压测后优化 | 改进 |
|------|--------|------------|------|
| 最大QPS | 5万 | 10万 | 100%↑ |
| P99延迟 | 500ms | 150ms | 70%↓ |
| 错误率 | 1% | 0.01% | 99%↓ |
| 稳定性 | 8小时 | 7×24小时 | 显著提升 |
