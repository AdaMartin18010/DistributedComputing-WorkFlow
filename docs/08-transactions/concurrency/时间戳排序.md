# 时间戳排序（Timestamp Ordering）

## 概述

时间戳排序（Timestamp Ordering）是一种基于时间戳的并发控制协议，由Phil Bernstein和Nathan Goodman在1980年提出。它为每个事务分配唯一的时间戳，通过比较时间戳来决定事务操作的执行顺序，从而保证事务的可串行化。

时间戳排序协议简单易实现，是许多数据库系统中实现可串行化的基础算法，特别是其变种Thomas写规则（Thomas Write Rule）在实际应用中有着重要价值。

## 一、基本概念

### 1.1 核心思想

```
┌─────────────────────────────────────────────────────────────┐
│               时间戳排序核心思想                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   基本假设：                                                 │
│   - 每个事务Ti在启动时被分配一个唯一的时间戳TS(Ti)            │
│   - 时间戳是单调递增的                                       │
│                                                             │
│   串行化顺序：                                               │
│   如果 TS(Ti) < TS(Tj)，则数据库系统等效于先执行Ti再执行Tj    │
│                                                             │
│   冲突处理：                                                 │
│   - 当操作冲突时，比较时间戳决定如何处理                      │
│   - 老事务的请求通常被优先满足                                │
│   - 违反顺序的事务需要回滚重试                                │
│                                                             │
│   示例：                                                     │
│   时间 ────────────────────────────────────────────────→   │
│                                                             │
│   T1开始 (TS=1) ──→ T2开始 (TS=2) ──→ T3开始 (TS=3)         │
│                                                             │
│   等效串行化顺序：T1 → T2 → T3                               │
│                                                             │
│   如果T3要修改T1已读取的数据：                                │
│   - T3的时间戳(3) > T1的时间戳(1)                            │
│   - 这是允许的，因为等效顺序中T3在T1之后                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 关键数据结构

```
┌─────────────────────────────────────────────────────────────┐
│                 时间戳排序的数据结构                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   每个数据项X维护两个时间戳：                                  │
│                                                             │
│   ┌─────────────────────────────────────────┐              │
│   │              数据项 X                   │              │
│   │  ┌─────────────┐  ┌─────────────┐       │              │
│   │  │  W-timestamp │  │  R-timestamp │       │              │
│   │  │   (W-TS)    │  │   (R-TS)    │       │              │
│   │  │  最后写入   │  │  最后读取   │       │              │
│   │  │  时间戳     │  │  时间戳     │       │              │
│   │  └─────────────┘  └─────────────┘       │              │
│   │                                         │              │
│   │  数据值: ...                            │              │
│   └─────────────────────────────────────────┘              │
│                                                             │
│   W-timestamp(X): 成功执行write(X)的最大事务时间戳            │
│   R-timestamp(X): 成功执行read(X)的最大事务时间戳             │
│                                                             │
│   初始值：W-TS(X) = R-TS(X) = 0                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 二、基本时间戳排序协议（Basic TO）

### 2.1 读操作规则

```
┌─────────────────────────────────────────────────────────────┐
│                  基本TO协议 - 读规则                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   事务Ti发起 read(X) 操作：                                  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  if TS(Ti) < W-TS(X) then                           │  │
│   │      // 需要读取的数据已经被更晚的事务修改             │  │
│   │      // 违反串行化顺序                               │  │
│   │      reject read()                                  │  │
│   │      rollback Ti;  // 回滚Ti，分配新时间戳重试         │  │
│   │  else                                               │  │
│   │      // 可以安全读取                                  │  │
│   │      execute read(X);                               │  │
│   │      R-TS(X) = max(R-TS(X), TS(Ti));                │  │
│   │  end if                                             │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   示例1（允许）：                                            │
│   TS(T1)=10, W-TS(X)=5                                     │
│   TS(T1) > W-TS(X) → 允许读取，R-TS(X)更新为10              │
│                                                             │
│   示例2（拒绝）：                                            │
│   TS(T1)=10, W-TS(X)=15                                    │
│   TS(T1) < W-TS(X) → 拒绝读取，T1回滚                       │
│   原因：T1想要读旧值，但等效顺序要求T1在写X的事务之后执行      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 写操作规则

```
┌─────────────────────────────────────────────────────────────┐
│                  基本TO协议 - 写规则                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   事务Ti发起 write(X) 操作：                                 │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  if TS(Ti) < R-TS(X) then                           │  │
│   │      // 数据已经被更晚的事务读取                      │  │
│   │      // 不能覆盖（否则违反串行化）                    │  │
│   │      reject write();                                │  │
│   │      rollback Ti;                                   │  │
│   │  else if TS(Ti) < W-TS(X) then                      │  │
│   │      // 数据已经被更晚的事务修改                      │  │
│   │      // 这是过时的写操作                             │  │
│   │      reject write();                                │  │
│   │      rollback Ti;                                   │  │
│   │  else                                               │  │
│   │      // 可以安全写入                                  │  │
│   │      execute write(X);                              │  │
│   │      W-TS(X) = TS(Ti);                              │  │
│   │  end if                                             │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   示例1（允许）：                                            │
│   TS(T1)=10, R-TS(X)=5, W-TS(X)=5                          │
│   10 >= 5 且 10 >= 5 → 允许写入，W-TS(X)更新为10            │
│                                                             │
│   示例2（拒绝 - 被读过）：                                    │
│   TS(T1)=10, R-TS(X)=15                                    │
│   10 < 15 → 拒绝写入，T1回滚                                │
│                                                             │
│   示例3（拒绝 - 被写过）：                                    │
│   TS(T1)=10, W-TS(X)=15                                    │
│   10 < 15 → 拒绝写入，T1回滚                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

│              基本TO协议执行示例                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   初始状态：X=100, Y=200                                    │
│   W-TS(X)=0, R-TS(X)=0, W-TS(Y)=0, R-TS(Y)=0               │

│                                                             │

│                                                             │
│   T1开始执行：                                               │
│   1. T1 read(X):                                            │
│      TS(T1)=1 >= W-TS(X)=0  ✓ 允许                         │

│      结果: X=100                                           │
│                                                             │
│   2. T1 write(Y):                                           │
│      TS(T1)=1 >= R-TS(Y)=0  ✓                             │
│      TS(T1)=1 >= W-TS(Y)=0  ✓ 允许                         │
│      W-TS(Y) = 1                                           │
│      结果: Y=新值                                           │
│                                                             │
│   T2开始执行：                                               │
│   3. T2 read(Y):                                            │
│      TS(T2)=2 >= W-TS(Y)=1  ✓ 允许                         │
│      R-TS(Y) = max(0, 2) = 2                               │
│      结果: Y=新值(T1写入的)                                  │
│                                                             │
│   4. T2 write(X):                                           │
│      TS(T2)=2 >= R-TS(X)=1  ✓                             │
│      TS(T2)=2 >= W-TS(X)=0  ✓ 允许                         │
│      W-TS(X) = 2                                           │
│      结果: X=新值                                           │
│                                                             │
│   等效串行化顺序: T1 → T2 (符合时间戳顺序)                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 三、Thomas写规则（Thomas Write Rule）

### 3.1 改进动机

```
┌─────────────────────────────────────────────────────────────┐
│                Thomas写规则的动机                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   问题场景：                                                 │
│   事务T1 (TS=1): write(X)                                  │
│   事务T2 (TS=2): write(X)  // 在T1之后执行                   │
│                                                             │
│   基本TO协议的处理：                                         │
│   - T1先执行: W-TS(X) = 1                                  │
│   - T2执行时: TS(T2)=2 >= W-TS(X)=1 ✓ 允许                 │
│   - T2写入，W-TS(X) = 2                                    │
│   - 结果正常                                               │
│                                                             │
│   另一个场景：                                               │
│   事务T1 (TS=2): write(X)                                  │
│   事务T2 (TS=1): write(X)  // 在T1之后启动，但时间戳更小      │
│                                                             │
│   基本TO协议的处理：                                         │
│   - T1先执行: W-TS(X) = 2                                  │
│   - T2执行时: TS(T2)=1 < W-TS(X)=2 ✗ 拒绝，T2回滚           │
│                                                             │
│   观察：T2的写操作是"过时的"，因为等效顺序中T2在T1之前，      │
│         T1的写入会覆盖T2的写入。                              │
│         那么T2的写操作其实是可以忽略的！                       │
│                                                             │
│   Thomas写规则：                                             │
│   如果 write(X) 是过时的（TS(Ti) < W-TS(X)），               │
│   不需要回滚，直接忽略这个写操作即可！                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 Thomas写规则定义

```
┌─────────────────────────────────────────────────────────────┐
│                  Thomas写规则                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   事务Ti发起 write(X) 操作：                                 │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  if TS(Ti) < R-TS(X) then                           │  │
│   │      // 数据已经被更晚的事务读取                      │  │
│   │      reject write();                                │  │
│   │      rollback Ti;                                   │  │
│   │  else if TS(Ti) < W-TS(X) then                      │  │
│   │      // 数据已经被更晚的事务修改                      │  │
│   │      // Thomas写规则: 忽略这个过时的写操作             │  │
│   │      // 不执行write，也不回滚！                       │  │
│   │      skip write();  ← 关键区别！                      │  │
│   │  else                                               │  │
│   │      execute write(X);                              │  │
│   │      W-TS(X) = TS(Ti);                              │  │
│   │  end if                                             │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   核心思想：                                                 │
│   - 如果Ti的时间戳小于W-TS(X)，说明有一个更晚的事务已经       │
│     写入了X，并且等效顺序中那个事务在Ti之后。                 │
│   - Ti的写入会被覆盖，所以可以直接忽略，不需要回滚Ti。         │
│                                                             │
│   效果：                                                     │
│   - 减少不必要的事务回滚                                     │
│   - 提高并发度                                              │
│   - 仍保证可串行化                                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

│   T1 (TS=2): write(X)                                      │

│                                                             │
│   步骤2: T2执行 read(Y)                                     │

│   步骤3: T2执行 write(X)                                    │
│   - TS(T2)=1 < W-TS(X)=2                                   │
│                                                             │
│   基本TO协议：                                               │
│   - 拒绝写入                                               │
│   - 回滚T2，分配新时间戳（如3）                              │

│                                                             │

│   - 忽略这个写操作                                          │
│   - T2继续执行，最终提交                                     │
│   - 结果：减少回滚，提高性能                                 │
│                                                             │
│   正确性分析：                                               │
│   - 等效串行化顺序是 T2 → T1 (因为TS(T2)=1 < TS(T1)=2)       │
│   - 在这个顺序中，T1的写入会覆盖T2的写入                     │
│   - 所以T2的写入实际上是无效的，忽略它是正确的！               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 四、时间戳排序的Java实现

### 4.1 基础实现

```java
/**
 * 时间戳排序管理器
 */
public class TimestampOrderingManager {
    
    private final AtomicLong timestampGenerator = new AtomicLong(1);
    
    // 数据项的时间戳信息
    private final ConcurrentHashMap<String, DataItemTimestamps> timestampsMap = 
        new ConcurrentHashMap<>();
    
    @Data
    public static class DataItemTimestamps {
        private final String key;
        private volatile long rTimestamp = 0;  // 读时间戳
        private volatile long wTimestamp = 0;  // 写时间戳
        
        public DataItemTimestamps(String key) {
            this.key = key;
        }
    }
    
    /**
     * 获取数据项的时间戳信息
     */
    private DataItemTimestamps getTimestamps(String key) {
        return timestampsMap.computeIfAbsent(key, DataItemTimestamps::new);
    }
    
    /**
     * 分配新时间戳
     */
    public long allocateTimestamp() {
        return timestampGenerator.getAndIncrement();
    }
}

/**
 * 基本TO协议实现
 */
@Component
public class BasicTOProtocol {
    
    @Autowired
    private TimestampOrderingManager toManager;
    
    @Autowired
    private DataRepository dataRepository;
    
    /**
     * 读操作
     */
    public DataItem read(TransactionContext tx, String key) {
        long ts = tx.getTimestamp();
        TimestampOrderingManager.DataItemTimestamps timestamps = 
            toManager.getTimestamps(key);
        
        // 检查: TS(Ti) < W-TS(X)?
        if (ts < timestamps.getWTimestamp()) {
            // 拒绝读取，需要回滚
            throw new TimestampOrderingException(
                "Read rejected: TS(" + ts + ") < W-TS(" + timestamps.getWTimestamp() + ")"
            );
        }
        
        // 允许读取，更新R-TS
        synchronized (timestamps) {
            if (ts > timestamps.getRTimestamp()) {
                timestamps.setRTimestamp(ts);
            }
        }
        
        // 读取数据
        return dataRepository.findByKey(key);
    }
    
    /**
     * 写操作（基本TO）
     */
    public void write(TransactionContext tx, String key, DataItem value) {
        long ts = tx.getTimestamp();
        TimestampOrderingManager.DataItemTimestamps timestamps = 
            toManager.getTimestamps(key);
        
        // 检查: TS(Ti) < R-TS(X)?
        if (ts < timestamps.getRTimestamp()) {
            throw new TimestampOrderingException(
                "Write rejected: TS(" + ts + ") < R-TS(" + timestamps.getRTimestamp() + ")"
            );
        }
        
        // 检查: TS(Ti) < W-TS(X)?
        if (ts < timestamps.getWTimestamp()) {
            throw new TimestampOrderingException(
                "Write rejected: TS(" + ts + ") < W-TS(" + timestamps.getWTimestamp() + ")"
            );
        }
        
        // 允许写入，更新W-TS
        timestamps.setWTimestamp(ts);
        
        // 写入数据
        dataRepository.save(key, value);
    }
}
```

### 4.2 Thomas写规则实现

```java
/**
 * Thomas写规则实现
 */
@Component
public class ThomasWriteRuleProtocol {
    
    @Autowired
    private TimestampOrderingManager toManager;
    
    @Autowired
    private DataRepository dataRepository;
    
    private static final Logger logger = LoggerFactory.getLogger(ThomasWriteRuleProtocol.class);
    
    /**
     * 读操作（与基本TO相同）
     */
    public DataItem read(TransactionContext tx, String key) {
        long ts = tx.getTimestamp();
        TimestampOrderingManager.DataItemTimestamps timestamps = 
            toManager.getTimestamps(key);
        
        if (ts < timestamps.getWTimestamp()) {
            throw new TimestampOrderingException(
                "Read rejected: TS(" + ts + ") < W-TS(" + timestamps.getWTimestamp() + ")"
            );
        }
        
        synchronized (timestamps) {
            if (ts > timestamps.getRTimestamp()) {
                timestamps.setRTimestamp(ts);
            }
        }
        
        return dataRepository.findByKey(key);
    }
    
    /**
     * 写操作（Thomas写规则）
     */
    public WriteResult write(TransactionContext tx, String key, DataItem value) {
        long ts = tx.getTimestamp();
        TimestampOrderingManager.DataItemTimestamps timestamps = 
            toManager.getTimestamps(key);
        
        // 检查: TS(Ti) < R-TS(X)?
        if (ts < timestamps.getRTimestamp()) {
            // 被读过，必须回滚
            throw new TimestampOrderingException(
                "Write rejected: TS(" + ts + ") < R-TS(" + timestamps.getRTimestamp() + ")"
            );
        }
        
        // 检查: TS(Ti) < W-TS(X)?
        if (ts < timestamps.getWTimestamp()) {
            // Thomas写规则: 忽略这个过时的写操作
            logger.info("Thomas Write Rule applied: Ignoring outdated write by TS={} on key={}", 
                ts, key);
            return WriteResult.IGNORED;
        }
        
        // 允许写入
        timestamps.setWTimestamp(ts);
        dataRepository.save(key, value);
        
        return WriteResult.SUCCESS;
    }
    
    public enum WriteResult {
        SUCCESS,   // 写入成功
        IGNORED    // 被Thomas写规则忽略
    }
}
```

### 4.3 使用示例

```java
/**
 * 时间戳排序使用示例
 */
@Service
public class BankServiceWithTO {
    
    @Autowired
    private ThomasWriteRuleProtocol toProtocol;
    
    @Autowired
    private TimestampOrderingManager toManager;
    
    /**
     * 转账操作
     */
    public void transfer(String fromAccount, String toAccount, 
                        BigDecimal amount) {
        // 分配时间戳
        long timestamp = toManager.allocateTimestamp();
        TransactionContext tx = new TransactionContext(timestamp);
        
        try {
            // 1. 读取转出账户
            Account from = toProtocol.read(tx, fromAccount);
            if (from.getBalance().compareTo(amount) < 0) {
                throw new InsufficientBalanceException("余额不足");
            }
            
            // 2. 读取转入账户
            Account to = toProtocol.read(tx, toAccount);
            
            // 3. 更新转出账户
            from.setBalance(from.getBalance().subtract(amount));
            WriteResult result1 = toProtocol.write(tx, fromAccount, from);
            
            // 4. 更新转入账户
            to.setBalance(to.getBalance().add(amount));
            WriteResult result2 = toProtocol.write(tx, toAccount, to);
            
            logger.info("Transfer completed: {} -> {}, amount={}, results={}/{}",
                fromAccount, toAccount, amount, result1, result2);
                
        } catch (TimestampOrderingException e) {
            // 时间戳排序冲突，需要重试
            logger.warn("TO conflict, retrying: {}", e.getMessage());
            retryTransfer(fromAccount, toAccount, amount);
        }
    }
    
    private void retryTransfer(String fromAccount, String toAccount, 
                               BigDecimal amount) {
        // 获取新时间戳重试
        // 实际应用中应有最大重试次数限制
        transfer(fromAccount, toAccount, amount);
    }
}
```

## 五、优缺点分析

### 5.1 优点

| 优点 | 说明 |
|-----|------|
| **无死锁** | 不需要等待锁，不会出现死锁 |
| **无饥饿** | 老事务优先，不会出现事务饥饿 |
| **简单易实现** | 只需维护两个时间戳 |
| **Thomas写规则优化** | 减少不必要的事务回滚 |

### 5.2 缺点

| 缺点 | 说明 |
|-----|------|
| **冲突回滚** | 冲突时必须回滚，开销较大 |
| **长事务不利** | 长事务容易被短事务导致回滚 |
| **时间戳分配** | 需要全局唯一的时间戳分配机制 |
| **读可能拒绝** | 基本TO中读操作也可能被拒绝 |

### 5.3 适用场景

## 六、总结

时间戳排序是一种基于时间戳的乐观并发控制协议，通过比较事务时间戳和数据项的读写时间戳来决定操作的执行顺序。

**核心要点**：
1. 每个事务分配唯一的时间戳
2. 每个数据项维护R-TS和W-TS
3. 老事务优先，违反顺序的操作被拒绝
4. Thomas写规则可以优化过时的写操作

**最佳实践**：
- 适用于冲突较少的场景
- 配合重试机制处理回滚
- 考虑使用Thomas写规则减少回滚
- 避免设计过长的业务事务
