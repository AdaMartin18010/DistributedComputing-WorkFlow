## 📋 执行摘要

边缘计算是一种分布式计算范式，将计算、存储和网络资源部署在靠近数据源的边缘位置，以降低延迟、减少带宽消耗、提升隐私安全性。它与云计算形成互补，共同构成云边端协同的完整计算架构。

## 一、核心概念

### 1.1 定义与原理

**边缘计算（Edge Computing）** 是指在靠近物或数据源头的网络边缘侧，融合网络、计算、存储、应用核心能力的开放平台，就近提供边缘智能服务。

**核心驱动力**：
| 驱动力 | 说明 |
|--------|------|
| **延迟敏感** | 自动驾驶、工业控制需<10ms响应 |
| **带宽限制** | 减少回传流量，降低带宽成本 |
| **隐私合规** | 敏感数据本地处理，不出境 |
| **可靠性** | 弱网/断网环境下持续运行 |
| **实时性** | 本地决策，无需等待云端响应 |

### 1.3 关键特性

| 特性 | 说明 |
|------|------|
| **低延迟** | 数据处理在本地完成，毫秒级响应 |
| **带宽优化** | 仅传输聚合结果到云端 |
| **离线自治** | 断网时仍可独立运行 |
| **数据主权** | 敏感数据本地处理，符合法规 |
| **异构支持** | 支持ARM/x86/GPU等多种硬件 |
| **安全隔离** | 多层安全机制，设备认证 |

## 二、雾计算（Fog Computing）

### 2.1 雾计算定义

**雾计算**是Cisco提出的概念，是云计算向网络边缘的延伸，位于云端和设备端之间，形成"云-雾-端"三层架构。

**边缘计算 vs 雾计算**：
| 维度 | 边缘计算 | 雾计算 |
|------|----------|--------|
| **定义来源** | 通用术语 | Cisco提出 |
| **侧重点** | 靠近数据源 | 网络基础设施层 |
| **位置** | 设备到接入网 | 接入网到核心网 |
| **范围** | 更广（含终端） | 网络设备层 |
| **关系** | 包含雾计算 | 边缘计算的子集 |

## 三、MEC（移动边缘计算）

### 3.1 MEC 定义与架构

**MEC（Multi-access Edge Computing）** 是由ETSI标准化的边缘计算架构，最初针对移动网络，现已扩展到多接入（5G、WiFi、固网）。

**MEC 关键组件**：

| 组件 | 功能 | 部署位置 |
|------|------|----------|
| **MEC主机** | 运行MEC应用的平台 | 基站/接入点/边缘DC |
| **MEC平台** | 提供服务注册、发现、能力开放 | MEC主机内 |
| **MEC应用** | 边缘业务应用 | 部署在MEC主机 |
| **MEC编排器** | 应用生命周期管理 | 运营商核心网 |
| **OSS/BSS** | 运营支撑系统 | 运营商云端 |

### 3.3 MEC 应用场景

| 场景 | 需求 | MEC方案 |
|------|------|---------|
| **车联网** | <10ms延迟 | 路侧单元RSU部署MEC |
| **AR/VR** | 大带宽、低延迟 | 5G基站侧MEC渲染 |
| **工业互联网** | 数据不出厂 | 园区级MEC部署 |
| **智能视频** | 实时分析 | 边缘AI推理 |
| **云游戏** | 流式传输 | 边缘渲染推流 |

### 5.2 数据流与决策流

| 流类型 | 方向 | 内容 | 带宽要求 |
|--------|------|------|----------|
| **数据上行** | 边缘→云 | 聚合数据、告警、日志 | 低 |
| **模型/配置下行** | 云→边缘 | AI模型、配置更新、策略 | 中 |
| **控制指令** | 云→边缘 | 远程控制、调度指令 | 低 |
| **实时监控** | 边缘→云 | 指标、心跳、状态 | 低 |
| **原始数据** | 设备→边缘 | 传感器数据、视频流 | 高 |

### 5.3 协同计算挑战

| 挑战 | 说明 | 解决方案 |
|------|------|----------|
| **网络不稳定** | 边缘网络抖动、断网 | 断点续传、本地缓存、异步同步 |
| **资源受限** | 边缘设备CPU/内存有限 | 模型压缩、量化、轻量运行时 |
| **安全边界** | 边缘设备物理暴露 | 设备认证、加密传输、TEE |
| **管理复杂** | 海量异构边缘节点 | 统一管理平面、自动化运维 |
| **一致性** | 分布式状态同步 | 最终一致性、CRDT、状态版本 |

# edgecore.yaml 边缘节点配置
apiVersion: edgecore.config.kubeedge.io/v1alpha2
kind: EdgeCore
modules:
  edged:
    nodeIP: 192.168.1.100
    podSandboxImage: kubeedge/pause:3.6

    remoteImageEndpoint: unix:///run/containerd/containerd.sock

    token: "<edge-node-token>"

      enable: true  # 启用边缘API server

      internal: true  # 内置MQTT broker

### 7.2 边缘设备管理示例

```yaml
# device-instance.yaml - KubeEdge设备定义
apiVersion: devices.kubeedge.io/v1alpha2
kind: Device
metadata:
  name: temperature-sensor-01
  labels:
    model: sensor
spec:
  deviceModelRef:
    name: temperature-model
  protocol:
    mqtt:
      client:
        server: tcp://127.0.0.1:1883  # 本地MQTT broker
  nodeSelector:
    nodeSelectorTerms:
      - matchExpressions:
          - key: node-role.kubernetes.io/edge
            operator: Exists
  propertyVisitors:
    - propertyName: temperature
      mqtt:
        topic: /devices/sensor01/temperature
        contentType: json
status:
  twins:
    - propertyName: temperature
      desired:
        value: "25.0"
      reported:
        value: "24.5"
```
