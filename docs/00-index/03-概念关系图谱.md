## 关系类型定义

本知识库使用以下标准关系类型描述概念间关系：

| 关系类型 | 符号 | 说明 | 示例 |
|:---|:---:|:---|:---|
| is-a | ⊆ | 是一种（层次关系） | 线性一致性 is-a 一致性模型 |
| has-a | ◇ | 具有（组成关系） | 分布式系统 has-a 一致性 |
| depends-on | → | 依赖（依赖关系） | 共识 depends-on 多数派 |
| implements | ⊢ | 实现（实现关系） | etcd implements Raft |
| uses | ⊙ | 使用（使用关系） | Cassandra uses 一致性哈希 |
| part-of | ⊂ | 部分（部分整体） | 领导者选举 part-of Raft |
| equivalent | ≡ | 等价（同义关系） | Master ≡ Primary |
| conflicts-with | ⊥ | 冲突（互斥关系） | 强一致性 conflicts-with 高延迟（某些场景） |
| enables | ⇒ | 使能（因果关系） | 分区容忍性 enables 高可用 |
| requires | ⇐ | 需要（前提条件） | 线性一致性 requires 时钟同步 |

## 核心关系图谱

### 一致性模型关系

```
一致性模型
├── is-a → 强一致性
│   ├── is-a → 线性一致性
│   ├── is-a → 严格可串行化
│   └── is-a → 顺序一致性
├── is-a → 弱一致性
│   ├── is-a → 因果一致性
│   ├── is-a → 会话一致性
│   │   ├── has-a → 读己之写
│   │   ├── has-a → 单调读
│   │   ├── has-a → 单调写
│   │   └── has-a → 读后写
│   └── is-a → 最终一致性
└── conflicts-with → 可用性（在网络分区时）

线性一致性
├── requires → 全局时钟/实时钟
├── implements → 原子操作语义
└── uses → 共识算法

因果一致性
├── depends-on → Happens-Before关系
├── uses → 向量时钟
└── is-a → 可捕获并发
```

### 共识算法关系

```
共识问题
├── has-a → 终止性（活性）
├── has-a → 一致性（安全性）
├── has-a → 有效性
├── depends-on → 故障模型
└── conflicts-with → FLP不可能性（在异步系统）

Paxos
├── implements → 共识
├── has-a → Prepare阶段
├── has-a → Accept阶段
├── requires → 多数派Quorum
├── is-a → Basic Paxos（单值）
├── is-a → Multi-Paxos（多值）
│   └── optimizes → 领导者稳定时跳过Prepare
├── is-a → Fast Paxos
│   └── reduces → 延迟（特定条件下）
└── equivalent → ZAB（功能等价）

Raft
├── implements → 共识
├── is-a → Multi-Paxos变体
├── has-a → 领导者选举
│   └── uses → 超时机制
├── has-a → 日志复制
│   └── uses → 强领导者
├── has-a → 安全性
│   └── requires → 多数派匹配
├── easier-than → Paxos（可理解性）
└── used-by → etcd, Consul, TiKV

BFT算法
├── handles → 拜占庭故障
├── requires → n ≥ 3f + 1
├── is-a → PBFT
│   └── uses → 三阶段提交
├── is-a → HotStuff
│   └── has → 线性通信复杂度
└── used-by → 区块链系统
```

## 关键概念三元组

以下是RDF风格的核心概念关系（主语-谓语-宾语）：

### 一致性相关
```
(线性一致性, is-a, 强一致性)
(最终一致性, is-a, 弱一致性)
(因果一致性, depends-on, Happens-Before关系)
(向量时钟, enables, 因果一致性检测)
(会话保证, part-of, 弱一致性模型)
(读己之写, is-a, 会话保证)
(单调读, is-a, 会话保证)
(CAP定理, states, 一致性-可用性-分区容忍性权衡)
```

## 概念冲突与权衡矩阵

| 概念A | 关系 | 概念B | 说明 |
|:---|:---:|:---|:---|
| 强一致性 | conflicts-with | 高可用性 | 网络分区时需选择 |
| 强一致性 | requires | 高延迟 | 同步复制的代价 |
| 最终一致性 | enables | 高可用性 | 牺牲即时一致性 |
| 安全性 | requires | 同步通信 | 拜占庭共识成本高 |
| 2PC | has-problem | 阻塞 | 协调者故障时阻塞 |
| 3PC | solves | 2PC阻塞 | 但增加复杂度 |
| 批处理 | suitable-for | 大数据量 | 吞吐优化 |
| 流处理 | suitable-for | 低延迟 | 实时性要求 |
| 有状态 | harder-than | 无状态 | 扩展和恢复复杂 |
| 微服务 | increases | 系统复杂度 | 但提高可维护性 |
