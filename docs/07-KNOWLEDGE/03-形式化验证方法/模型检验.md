# 模型检验

## 概述

**模型检验（Model Checking）** 是一种自动验证技术，用于验证有限状态系统是否满足给定的形式化规约。它由Edmund Clarke、Allen Emerson和Joseph Sifakis在1980年代初期独立提出，并于2007年获得图灵奖。

### 核心思想

模型检验基于三个核心思想：

1. **自动验证**：
   - 给定系统模型和性质规约
   - 自动检查模型是否满足性质
   - 如果性质不满足，生成反例

2. **穷举状态空间**：
   - 遍历系统的所有可能状态
   - 检查性质在所有状态下是否成立
   - 处理可达状态空间

3. **反例生成**：
   - 当性质不满足时，生成反例路径
   - 反例帮助调试和理解问题
   - 提供具体的违反场景

### 基本流程

```
系统建模 → 性质规约 → 模型检验 → 结果分析
   ↓           ↓           ↓
 有限状态   时序逻辑    自动验证
 自动机    (CTL/LTL)    算法
```

## 状态空间探索

### 基本概念

**状态空间**：系统所有可能状态的集合。

**可达状态**：从初始状态通过状态转换可以到达的所有状态。

**状态爆炸问题**：状态空间随系统复杂度指数增长。

$$ |S| = O(2^n) $$

其中 $n$ 是系统变量数。

### 状态空间搜索算法

#### 广度优先搜索（BFS）

```
BFS(Model, InitialState, Property):
    Queue ← [InitialState]
    Visited ← {InitialState}

    while Queue ≠ ∅:
        State ← Queue.dequeue()

        if not CheckProperty(State, Property):
            return FALSE, State

        for each NextState in Successors(State):
            if NextState ∉ Visited:
                Visited.add(NextState)
                Queue.enqueue(NextState)

    return TRUE
```

**复杂度**：

- 时间复杂度：$O(|S| + |T|)$
- 空间复杂度：$O(|S|)$
- 其中 $|S|$ 是状态数，$|T|$ 是转换数

#### 深度优先搜索（DFS）

```
DFS(Model, State, Property, Visited, Stack):
    if not CheckProperty(State, Property):
        return FALSE, State

    Visited.add(State)
    Stack.push(State)

    for each NextState in Successors(State):
        if NextState ∉ Visited:
            result = DFS(Model, NextState, Property, Visited, Stack)
            if not result:
                return FALSE

    Stack.pop()
    return TRUE
```

**复杂度**：

- 时间复杂度：$O(|S| + |T|)$
- 空间复杂度：$O(|S|)$（最坏情况）或 $O(h)$（平均，$h$ 是深度）

## 反例生成

### 基本概念

**反例**：当系统不满足性质时，一个具体的执行路径（状态序列）展示了性质的违反。

**反例类型**：

- **路径反例**：线性路径，展示从初始状态到违反状态的路径
- **循环反例**：包含循环的路径，展示无限行为的违反

### 反例生成算法

#### CTL反例生成

对于 $EG\phi$ 不满足（即不存在路径使得 $\phi$ 总是成立）：

1. 计算满足 $\neg\phi$ 的状态集合 $S_{\neg\phi}$
2. 检查从初始状态是否可达 $S_{\neg\phi}$
3. 如果可达，生成从初始状态到 $S_{\neg\phi}$ 的最短路径

#### LTL反例生成

通过LTL到Büchi自动机的转换：

1. 将LTL公式 $\phi$ 转换为Büchi自动机 $A_\phi$
2. 构建系统 $M$ 与 $A_\phi$ 的乘积自动机
3. 检查乘积自动机是否存在接受路径
4. 如果存在，映射回系统 $M$ 的具体路径

## 抽象技术

### 动机

**状态空间爆炸**：实际系统的状态空间可能非常大甚至无限。

**解决方案**：使用抽象技术减少状态空间。

### 抽象类型

#### 1. 数据抽象（Data Abstraction）

**思想**：将具体数据值映射到抽象域。

**示例**：

```
具体域：整数 (..., -2, -1, 0, 1, 2, ...)
抽象域：{负数, 零, 正数}
```

**形式化**：
$$ \alpha: D \to D^{\#} $$

其中 $\alpha$ 是抽象函数，$D$ 是具体域，$D^{\#}$ 是抽象域。

#### 2. 谓词抽象（Predicate Abstraction）

**思想**：使用谓词将状态空间划分为等价类。

**定义**：给定谓词集合 $P = \{p_1, p_2, ..., p_n\}$，两个状态 $s_1$ 和 $s_2$ 等价，当且仅当它们满足相同的谓词集合。

**抽象状态**：
$$ \alpha(s) = (p_1(s), p_2(s), ..., p_n(s)) $$

#### 3. 反例引导的抽象精化（CEGAR - Counter-Example Guided Abstraction Refinement）

**流程**：

```
1. 初始抽象 → 2. 模型检验 → 3. 分析反例
      ↑                            ↓
      ←←← 4. 精化抽象 ←←← 4. 验证反例
```

**步骤**：

1. 创建初始（粗粒度）抽象
2. 在抽象模型上运行模型检验
3. 如果验证通过，原系统满足性质
4. 如果产生反例，检查反例是否在具体系统上可行
5. 如果不可行（虚假反例），精化抽象，回到步骤2

## 符号模型检验（BDD）

### 二叉决策图（BDD - Binary Decision Diagram）

**定义**：BDD是表示布尔函数的规范数据结构。

**特性**：

- **规范性**：对于给定的变量序，BDD是唯一的
- **紧凑性**：可以紧凑表示大型布尔函数
- **高效操作**：支持高效的布尔运算

### 符号模型检验原理

**核心思想**：使用BDD表示状态集合和转换关系，而不是显式枚举状态。

**状态集合表示**：
$$ S = \{s \in \{0,1\}^n : f_S(s) = 1\} $$

其中 $f_S$ 是表示状态集合的布尔函数。

**转换关系表示**：
$$ R(s, s') = \bigvee_{t \in T} \text{Guard}_t(s) \land \text{Update}_t(s, s') $$

### 不动点计算

**前向可达性分析**：

$$ S_0 = I $$
$$ S_{i+1} = S_i \cup \text{Post}(S_i) $$
$$ S_{\infty} = \mu Z. I \lor \text{Post}(Z) $$

其中：

- $I$ 是初始状态集合
- $\text{Post}(S) = \{s' : \exists s \in S. R(s, s')\}$
- $\mu$ 表示最小不动点

**后向可达性分析**：

$$ B_0 = P $$
$$ B_{i+1} = B_i \cup \text{Pre}(B_i) $$
$$ B_{\infty} = \mu Z. P \lor \text{Pre}(Z) $$

其中：

- $P$ 是目标状态集合
- $\text{Pre}(B) = \{s : \exists s' \in B. R(s, s')\}$

### BDD复杂度分析

**优点**：

- 可以处理更大的状态空间（$10^{20}$ 甚至更大）
- 符号表示，避免显式枚举

**缺点**：

- 最坏情况仍是指数复杂度
- BDD大小对变量序敏感
- 某些函数无法用紧凑BDD表示

## 有界模型检验（BMC - Bounded Model Checking）

### 基本概念

**思想**：将有界长度的路径验证转换为SAT问题。

**原理**：检查是否存在长度不超过 $k$ 的反例路径。

### BMC算法

```
BMC(Model, Property, k_max):
    for k = 1 to k_max:
        # 展开模型 k 步
        Formula = I(s_0) ∧ ⋀_{i=0}^{k-1} R(s_i, s_{i+1}) ∧ ⋁_{i=0}^{k} ¬Property(s_i)

        # 使用SAT求解器求解
        result = SAT_Solve(Formula)

        if result == SAT:
            return FALSE, Extract_Counterexample(result)

    return TRUE, "No counterexample found up to length k_max"
```

### BMC公式构建

**状态展开**：
$$ I(s_0) \land R(s_0, s_1) \land R(s_1, s_2) \land ... \land R(s_{k-1}, s_k) $$

**性质违反**：
$$ \neg \text{Property}(s_0) \lor \neg \text{Property}(s_1) \lor ... \lor \neg \text{Property}(s_k) $$

**完整公式**：
$$ I(s_0) \land \bigwedge_{i=0}^{k-1} R(s_i, s_{i+1}) \land \bigvee_{i=0}^{k} \neg \text{Property}(s_i) $$

### BMC vs BDD模型检验

| 特性 | BDD模型检验 | BMC |
|------|-------------|-----|
| 状态空间 | 符号表示 | 展开为SAT |
| 完整验证 | 可以 | 只能验证有界路径 |
| 反例 | 可以生成任意长度 | 只能生成有界长度 |
| 适用场景 | 小规模系统 | 大规模系统 |
| 工具 | NuSMV | CBMC, NuSMV |

## 工具使用指南

### NuSMV

**NuSMV** 是支持BDD和SAT模型检验的工具。

#### 基本使用

```smv
MODULE main
VAR
    request : boolean;
    state : {ready, busy};

ASSIGN
    init(state) := ready;
    next(state) := case
        state = ready & request : busy;
        state = busy : ready;
        TRUE : state;
    esac;

-- CTL 性质
SPEC AG (request -> AF state = busy)

-- LTL 性质
LTLSPEC G (request -> F state = busy)
```

运行：

```bash
NuSMV -bmc model.smv      # 使用BMC
NuSMV model.smv           # 使用BDD
```

### SPIN

**SPIN** 是支持LTL模型检验的工具。

```bash
spin -a model.pml         # 生成验证器
gcc -o pan pan.c          # 编译
./pan -a                  # 运行验证
./pan -t                  # 生成反例轨迹
```

### CBMC

**CBMC** 是C语言的有界模型检验工具。

```bash
cbmc --trace program.c    # 验证C程序
cbmc --unwind 10 program.c  # 设置展开界限
```

## 总结

模型检验是一种强大的自动验证技术，通过穷举状态空间来验证系统性质。符号模型检验（BDD）和有界模型检验（SAT）解决了状态空间爆炸问题，使得模型检验可以应用于更大规模的系统。反例生成和抽象精化技术进一步增强了模型检验的实用性。

**关键技术**：

- 状态空间探索（BFS/DFS）
- 符号模型检验（BDD）
- 有界模型检验（SAT）
- 抽象与精化（CEGAR）

**适用场景**：

- 硬件验证
- 协议验证
- 嵌入式系统
- 并发程序
