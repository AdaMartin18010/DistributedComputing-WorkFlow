# 流处理窗口操作详解

## 1. 窗口概述

窗口是流处理中将无限数据流切分为有限数据块进行聚合计算的核心机制。不同类型的窗口适用于不同的业务场景和分析需求。

```
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │

│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  滚动窗口 (Tumbling Window)                                  │   │
│  │  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐                       │   │
```

## 3. Flink 窗口操作

### 3.1 滚动窗口 (Tumbling Window)

```java
import org.apache.flink.streaming.api.windowing.assigners.TumblingEventTimeWindows;
import org.apache.flink.streaming.api.windowing.time.Time;

// 基于 Event Time 的滚动窗口
DataStream<Tuple2<String, Integer>> result = stream
    .keyBy(value -> value.f0)
    .window(TumblingEventTimeWindows.of(Time.minutes(5)))
    .aggregate(new MyAggregateFunction());

// 基于 Processing Time 的滚动窗口
DataStream<Tuple2<String, Integer>> result2 = stream
    .keyBy(value -> value.f0)
    .window(TumblingProcessingTimeWindows.of(Time.seconds(10)))
    .aggregate(new MyAggregateFunction());

// 使用偏移量 (处理时区问题)
DataStream<Tuple2<String, Integer>> result3 = stream
    .keyBy(value -> value.f0)
    .window(TumblingEventTimeWindows.of(Time.days(1), Time.hours(-8)))
    .aggregate(new MyAggregateFunction());
```

### 3.2 滑动窗口 (Sliding Window)

```java
// 滑动窗口：窗口大小 10 分钟，每 5 分钟滑动一次
DataStream<Tuple2<String, Integer>> result = stream
    .keyBy(value -> value.f0)
    .window(SlidingEventTimeWindows.of(Time.minutes(10), Time.minutes(5)))
    .aggregate(new MyAggregateFunction());

// PyFlink 示例
result = data_stream \
    .key_by(lambda x: x[0]) \
    .window(SlidingEventTimeWindows.of(Time.minutes(10), Time.minutes(5))) \
    .aggregate(CountAggregateFunction())
```

### 3.3 会话窗口 (Session Window)

```java
// 会话窗口：5 分钟空闲时间
DataStream<Tuple2<String, Integer>> result = stream
    .keyBy(value -> value.f0)
    .window(EventTimeSessionWindows.withGap(Time.minutes(5)))
    .allowedLateness(Time.minutes(2))  // 允许 2 分钟延迟
    .aggregate(new MyAggregateFunction());

// 动态会话间隔
DataStream<Tuple2<String, Integer>> result2 = stream
    .keyBy(value -> value.f0)
    .window(EventTimeSessionWindows.withDynamicGap((element) -> Time.minutes(10)))
    .aggregate(new MyAggregateFunction());
```

### 3.4 全局窗口 (Global Window)

```java
// 全局窗口 + 触发器
DataStream<Tuple2<String, Integer>> result = stream
    .keyBy(value -> value.f0)
    .window(GlobalWindows.create())
    .trigger(CountTrigger.of(1000))  // 每 1000 条触发一次
    .evictor(CountEvictor.of(500))   // 保留最近 500 条
    .aggregate(new MyAggregateFunction());
```

## 4. Spark Structured Streaming 窗口

```python
from pyspark.sql.functions import window, col

# 滚动窗口
windowedCounts = events \
    .withWatermark("timestamp", "10 minutes") \
    .groupBy(
        window(events.timestamp, "10 minutes"),
        events.action
    ) \
    .count()

# 滑动窗口
windowedCounts = events \
    .withWatermark("timestamp", "10 minutes") \
    .groupBy(
        window(events.timestamp, "10 minutes", "5 minutes"),  # 窗口大小, 滑动间隔
        events.action
    ) \
    .count()

# 会话窗口 (Spark 3.2+)
sessionCounts = events \
    .withWatermark("timestamp", "10 minutes") \
    .groupBy(
        session_window(events.timestamp, "5 minutes"),  # gap duration
        events.user_id
    ) \
    .agg(count("*").alias("event_count"))
```

## 5. Kafka Streams 窗口

```java
import org.apache.kafka.streams.kstream.TimeWindows;
import org.apache.kafka.streams.kstream.SessionWindows;
import org.apache.kafka.streams.kstream.SlidingWindows;

// 滚动窗口
KStream<String, String> stream = builder.stream("input-topic");
stream.groupByKey()
    .windowedBy(TimeWindows.of(Duration.ofMinutes(5)))
    .count()
    .toStream()
    .to("output-topic");

// 滑动窗口 (Hopping)
stream.groupByKey()
    .windowedBy(TimeWindows.of(Duration.ofMinutes(10))
        .advanceBy(Duration.ofMinutes(2)))
    .aggregate(
        () -> 0L,
        (key, value, aggregate) -> aggregate + 1,
        Materialized.with(Serdes.String(), Serdes.Long())
    );

// 会话窗口
stream.groupByKey()
    .windowedBy(SessionWindows.with(Duration.ofMinutes(5)))
    .count();
```

## 6. Watermark 与迟到数据

```

│                                                                     │
│  Max Event Time = 12:00:00                                          │

│                                                                     │
│  数据流：                                                           │
│  ┌───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┐                │

│   │   │   │   │   │   │   │   │   │   │   │                       │
│   ■   ■   ■   ■   ■   ■   □   ■   ■   □   ■                       │
│       │                   │       │       │                        │
```

```java
DataStream<Event> withTimestamps = stream
    .assignTimestampsAndWatermarks(
        WatermarkStrategy
            .<Event>forBoundedOutOfOrderness(Duration.ofMinutes(2))
            .withTimestampAssigner((event, timestamp) -> event.getEventTime())
```

## 7. 窗口函数

### 7.1 增量聚合

```java
// AggregateFunction：增量聚合
public class AverageAggregate implements 
    AggregateFunction<Tuple2<String, Long>, AverageAccumulator, Double> {
    
    @Override
    public AverageAccumulator createAccumulator() {
        return new AverageAccumulator();
    }
    
    @Override
    public AverageAccumulator add(Tuple2<String, Long> value, AverageAccumulator accumulator) {
        accumulator.sum += value.f1;
        accumulator.count++;
        return accumulator;
    }
    
    @Override
    public Double getResult(AverageAccumulator accumulator) {
        return accumulator.sum / (double) accumulator.count;
    }
    
    @Override
    public AverageAccumulator merge(AverageAccumulator a, AverageAccumulator b) {
        a.sum += b.sum;
        a.count += b.count;
        return a;
    }
}
```

### 7.2 全窗口函数

```java
// ProcessWindowFunction：访问窗口元数据
public class MyWindowFunction extends 
    ProcessWindowFunction<Tuple2<String, Long>, String, String, TimeWindow> {
    
    @Override
    public void process(String key, Context context, 
                       Iterable<Tuple2<String, Long>> elements, 
                       Collector<String> out) {
        
        long windowStart = context.window().getStart();
        long windowEnd = context.window().getEnd();
        
        long count = 0;
        for (Tuple2<String, Long> element : elements) {
            count++;
        }
        
        out.collect("Window [" + windowStart + " - " + windowEnd + 
                   ") for key " + key + " has " + count + " elements");
    }
}

// 组合使用：增量聚合 + 窗口处理
DataStream<Result> result = stream
    .keyBy(value -> value.f0)
    .window(TumblingEventTimeWindows.of(Time.minutes(5)))
    .aggregate(new AverageAggregate(), new MyWindowFunction());
```

## 8. 窗口触发器

```java
// 自定义触发器
public class CustomTrigger extends Trigger<Object, TimeWindow> {
    
    private final long count;
    private final ReducingStateDescriptor<Long> stateDesc;
    
    private CustomTrigger(long count) {
        this.count = count;
        this.stateDesc = new ReducingStateDescriptor<>("count", 
            Long::sum, Types.LONG);
    }
    
    @Override
    public TriggerResult onElement(Object element, long timestamp, 
                                   TimeWindow window, TriggerContext ctx) {
        ReducingState<Long> countState = ctx.getPartitionedState(stateDesc);
        countState.add(1L);
        
        if (countState.get() >= count) {
            return TriggerResult.FIRE_AND_PURGE;
        }
        return TriggerResult.CONTINUE;
    }
    
    @Override
    public TriggerResult onProcessingTime(long time, TimeWindow window, 
                                         TriggerContext ctx) {
        return TriggerResult.CONTINUE;
    }
    
    @Override
    public TriggerResult onEventTime(long time, TimeWindow window, 
                                    TriggerContext ctx) {
        return TriggerResult.FIRE;
    }
    
    @Override
    public void clear(TimeWindow window, TriggerContext ctx) {
        ctx.getPartitionedState(stateDesc).clear();
    }
}
```

## 9. 窗口对比总结

| 窗口类型 | 特点 | 适用场景 | Flink | Spark | Kafka Streams |
|---------|------|---------|-------|-------|---------------|
| **滚动窗口** | 固定大小，不重叠 | 定期统计 | ✓ | ✓ | ✓ |
| **滑动窗口** | 固定大小，有重叠 | 移动平均 | ✓ | ✓ | ✓ |
| **会话窗口** | 动态大小，活动间隙 | 用户行为分析 | ✓ | ✓ (3.2+) | ✓ |
| **全局窗口** | 单一窗口，自定义触发 | 全局统计 | ✓ | ✗ | ✗ |

## 10. 最佳实践

1. **选择合适的窗口类型**：根据业务需求选择最合适的窗口
2. **设置合理的 Watermark**：平衡延迟和准确性
3. **配置 allowedLateness**：处理迟到数据
4. **使用增量聚合**：减少内存占用和计算量
5. **监控窗口状态**：避免窗口状态过大导致 OOM
