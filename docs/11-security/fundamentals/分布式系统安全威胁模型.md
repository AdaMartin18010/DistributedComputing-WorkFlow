# 分布式系统安全威胁模型

## 概述

分布式系统安全威胁模型是理解和防御安全攻击的基础框架。它帮助系统架构师识别潜在威胁、评估风险并设计相应的防御机制。在分布式环境中，由于系统的复杂性、网络边界模糊以及多参与方的特点，威胁建模变得尤为重要。

---

## 一、分布式系统安全挑战

### 1.1 网络边界模糊

与传统单体应用不同，分布式系统的组件分布在不同网络位置：

- **跨数据中心通信**：服务可能部署在不同地理位置
- **多云/混合云环境**：部分服务在公有云，部分在私有云
- **边缘计算节点**：IoT设备和边缘服务器加入系统
- **微服务间通信**：大量服务间调用增加攻击面

### 1.2 信任边界复杂

### 1.3 数据分散性

- 数据在多个节点间复制和分片
- 敏感数据可能在传输、存储、处理各阶段暴露
- 数据一致性协议（如Raft、Paxos）本身成为攻击目标

## 二、STRIDE威胁分类模型

STRIDE是微软提出的威胁分类框架，适用于分布式系统：

| 威胁类型 | 描述 | 分布式系统示例 |
|---------|------|---------------|
| **S**poofing（欺骗） | 冒充他人身份 | 伪造服务身份调用其他微服务 |
| **T**ampering（篡改） | 修改数据或代码 | 篡改服务间通信消息 |
| **R**epudiation（抵赖） | 否认执行过的操作 | 否认发起过交易请求 |
| **I**nformation Disclosure（信息泄露） | 暴露敏感信息 | 未加密的配置中心数据泄露 |
| **D**enial of Service（拒绝服务） | 使服务不可用 | 针对服务发现的DDoS攻击 |
| **E**levation of Privilege（权限提升） | 获得未授权权限 | 利用服务间调用链提升权限 |

## 三、常见分布式系统攻击

### 3.1 网络层攻击

#### 3.1.1 中间人攻击（MITM）

```
攻击前正常通信：
客户端 ──────────────► 服务端
         加密通道

中间人攻击后：
客户端 ──► 攻击者 ──► 服务端
         解密    重新加密
         窃听/篡改
```

**攻击原理**：
1. 攻击者截获客户端与服务端的通信
2. 向客户端伪装成服务端，向服务端伪装成客户端
3. 转发并可能篡改通信内容

**防御措施**：
- 强制TLS 1.3通信
- 证书固定（Certificate Pinning）
- 双向TLS认证（mTLS）

#### 3.1.2 DNS劫持与投毒

攻击者篡改DNS解析结果，将流量导向恶意服务器。

```
┌─────────────┐     DNS查询      ┌─────────────┐
│   客户端    │ ───────────────► │  被污染DNS   │
│  (受害者)   │                  │   服务器     │
└─────────────┘                  └──────┬──────┘
       │                                │
       │                                ▼
       │                         ┌─────────────┐
       │                         │  返回恶意IP  │
       │                         │ (攻击者控制) │
       │                         └──────┬──────┘
       │                                │
       ▼                                │
┌─────────────┐    连接到恶意IP         │
│  数据泄露   │ ◄───────────────────────┘
└─────────────┘
```

**防御措施**：
- DNSSEC（DNS安全扩展）
- DNS over HTTPS (DoH)
- DNS over TLS (DoT)

### 3.2 应用层攻击

#### 3.2.1 服务发现攻击

在微服务架构中，服务发现组件是核心：

```python
# 恶意服务注册示例
import requests

# 攻击者注册与合法服务同名但恶意实现的服务
def malicious_service_registration():
    service_registry = "http://consul:8500/v1/agent/service/register"
    
    payload = {
        "ID": "payment-service-999",
        "Name": "payment-service",  # 与合法服务同名
        "Tags": ["production"],
        "Address": "10.0.0.99",  # 攻击者控制的服务器
        "Port": 8080,
        "Check": {
            "HTTP": "http://10.0.0.99:8080/health",
            "Interval": "10s"
        }
    }
    
    # 如果注册中心缺乏认证，攻击者可成功注册
    response = requests.put(service_registry, json=payload)
    return response.status_code == 200
```

**攻击影响**：
- 流量被劫持到恶意服务
- 数据泄露或篡改
- 服务降级

**防御措施**：
- 服务注册认证（mTLS）
- 服务身份验证
- 网络隔离（服务网格）

#### 3.2.2 重放攻击

```
┌────────────┐                    ┌────────────┐
│   攻击者    │                    │   服务端    │
│            │                    │            │
│  拦截请求   │ ◄────────────────  │   转账API   │
│  req(A→B,  │                    │            │
│     $1000) │                    │            │
│            │                    │            │
│  重放请求   │ ────────────────►  │  重复执行   │
│  (相同req) │                    │  转账操作   │
└────────────┘                    └────────────┘
```

**防御代码示例**：

```python
import hashlib
import time
from typing import Optional
import redis

class ReplayAttackProtection:
    def __init__(self, redis_client: redis.Redis, 
                 window_seconds: int = 300):
        self.redis = redis_client
        self.window = window_seconds
    
    def _generate_nonce_key(self, nonce: str) -> str:
        return f"nonce:{nonce}"
    
    def validate_request(self, client_id: str, 
                        timestamp: int,
                        nonce: str,
                        signature: str) -> tuple[bool, Optional[str]]:
        """
        验证请求，防止重放攻击
        
        Returns:
            (is_valid, error_message)
        """
        current_time = int(time.time())
        
        # 1. 检查时间戳（防止过期请求重放）
        if abs(current_time - timestamp) > self.window:
            return False, "Request timestamp too old"
        
        # 2. 检查nonce是否已被使用
        nonce_key = self._generate_nonce_key(nonce)
        if self.redis.exists(nonce_key):
            return False, "Nonce already used - possible replay attack"
        
        # 3. 存储nonce，设置过期时间
        self.redis.setex(nonce_key, self.window, "1")
        
        # 4. 验证签名（包含client_id, timestamp, nonce）
        expected_sig = self._generate_signature(
            client_id, timestamp, nonce
        )
        if not self._secure_compare(signature, expected_sig):
            return False, "Invalid signature"
        
        return True, None
    
    def _generate_signature(self, client_id: str, 
                           timestamp: int, nonce: str) -> str:
        """生成HMAC签名"""
        import hmac
        message = f"{client_id}:{timestamp}:{nonce}"
        return hmac.new(
            self.secret_key.encode(),
            message.encode(),
            hashlib.sha256
        ).hexdigest()
    
    def _secure_compare(self, a: str, b: str) -> bool:
        """时序安全比较，防止时序攻击"""
        return hmac.compare_digest(a.encode(), b.encode())
```

### 3.3 分布式特有攻击

#### 3.3.1 时钟攻击

分布式系统依赖时钟同步，攻击者可以：
- 篡改NTP服务器响应
- 利用时钟偏差绕过基于时间的安全机制
- 影响分布式事务和共识算法

```python
# 安全的时钟同步验证
def validate_clock_skew(received_timestamp: int, 
                       max_skew_seconds: int = 30) -> bool:
    """
    验证接收时间戳与本地时间的偏差是否在允许范围内
    """
    import ntplib
    import time
    
    # 获取多个NTP源的时间
    ntp_servers = ['pool.ntp.org', 'time.google.com', 'time.apple.com']
    times = []
    
    for server in ntp_servers:
        try:
            client = ntplib.NTPClient()
            response = client.request(server, timeout=2)
            times.append(response.tx_time)
        except:
            continue
    
    if len(times) < 2:
        raise Exception("Cannot reach enough NTP servers")
    
    # 使用中位数时间，抵抗异常值
    median_time = sorted(times)[len(times) // 2]
    local_time = time.time()
    
    # 验证本地时间与NTP时间的偏差
    if abs(local_time - median_time) > max_skew_seconds:
        raise Exception("Local clock significantly out of sync")
    
    # 验证消息时间戳
    return abs(received_timestamp - median_time) <= max_skew_seconds
```

#### 3.3.2 拜占庭将军问题

分布式系统中节点可能出现任意故障（包括恶意行为）：

```
                    司令官
                      │
          ┌───────────┼───────────┐
          ▼           ▼           ▼
       ┌─────┐    ┌─────┐    ┌─────┐
       │副官1 │    │副官2 │    │副官3 │
       │(诚实) │    │(叛徒) │    │(诚实) │
       └──┬──┘    └──┬──┘    └──┬──┘
          │          │          │
          ▼          ▼          ▼
        "进攻"     "撤退"     "进攻"
          │          │          │
          └──────────┴──────────┘
                     │
            副官3收到矛盾信息
            需要判断谁是叛徒
```

**解决方案**：
- 使用BFT（拜占庭容错）共识算法
- PBFT、HotStuff、Tendermint等
- 3f+1节点容忍f个拜占庭节点

## 六、最佳实践

### 6.1 安全设计原则

1. **最小权限原则**：每个服务只拥有完成工作所需的最小权限
2. **纵深防御**：多层安全控制，单层失效不影响整体安全
3. **零信任**：永不信任，始终验证
4. **安全默认**：默认配置应该是安全的
5. **失败安全**：系统故障时应进入安全状态

### 6.2 实施建议

- **安全左移**：在设计和开发阶段考虑安全
- **自动化安全测试**：集成SAST/DAST到CI/CD
- **漏洞管理**：建立漏洞响应流程
- **安全培训**：提升团队安全意识
- **持续监控**：实时威胁检测和响应
