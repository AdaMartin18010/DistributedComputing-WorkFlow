# 时序逻辑

## 概述

**时序逻辑（Temporal Logic）** 是用于描述和验证系统随时间变化的性质的逻辑系统。它在形式化验证、程序验证和系统设计中具有重要应用。

### 主要分支

1. **线性时序逻辑（LTL - Linear Temporal Logic）**
   - 1977年由Amir Pnueli提出
   - 基于线性时间模型，考虑系统的一条执行路径
   - 适合表达路径上的时序性质

2. **计算树逻辑（CTL - Computation Tree Logic）**
   - 1981年由Clarke和Emerson提出
   - 基于分支时间模型，考虑系统所有可能的执行路径
   - 适合表达分支性质

3. **CTL*（统一逻辑）**
   - 1986年由Emerson和Halpern提出
   - 统一了CTL和LTL
   - 具有更强的表达能力

## 线性时序逻辑（LTL）

### 语法定义

LTL公式的语法由以下BNF定义：

$$ \phi ::= \text{true} | \text{false} | p | \neg \phi | \phi \land \psi | \phi \lor \psi | X\phi | F\phi | G\phi | \phi U \psi | \phi W \psi $$

其中：

- $p$ 是原子命题
- **时序运算符**：
  - $X\phi$：**下一步（Next）**，下一步满足 $\phi$
  - $F\phi$：**最终（Finally/Future）**，最终满足 $\phi$
  - $G\phi$：**总是（Globally/Always）**，总是满足 $\phi$
  - $\phi U \psi$：**直到（Until）**，$\phi$ 直到 $\psi$ 成立
  - $\phi W \psi$：**弱直到（Weak Until）**，等价于 $G\phi \lor (\phi U \psi)$

### 语义定义

**路径**：路径是状态的无限序列，表示系统的一个可能执行。

$$ \pi = s_0, s_1, s_2, ... $$

对于路径 $\pi = s_0, s_1, s_2, ...$ 和位置 $i \ge 0$：

- $\pi, i \models X\phi$ 当且仅当 $\pi, i+1 \models \phi$
- $\pi, i \models F\phi$ 当且仅当 $\exists j \ge i: \pi, j \models \phi$
- $\pi, i \models G\phi$ 当且仅当 $\forall j \ge i: \pi, j \models \phi$
- $\pi, i \models \phi U \psi$ 当且仅当 $\exists j \ge i: (\pi, j \models \psi \land \forall k \in [i, j): \pi, k \models \phi)$

### 常用性质表达

| 性质类型 | LTL公式 | 含义 |
|---------|---------|------|
| 安全性 | $G \neg P$ | 坏事永远不会发生 |
| 活性 | $F P$ | 好事最终会发生 |
| 响应性 | $G(P \implies F Q)$ | 每当P发生，最终Q会发生 |
| 公平性 | $GF P$ | P无限次发生 |
| 稳定性 | $FG P$ | 最终P永远成立 |

### LTL示例

```
互斥性质：G(¬(critical1 ∧ critical2))
无死锁：G((wait1 ∧ ¬critical2) → F critical1)
响应性：G(request → F response)
公平性：GF enabled → GF executed
```

## 计算树逻辑（CTL）

### 语法定义

CTL公式由路径量词（A表示"所有路径"，E表示"存在路径"）和时序运算符组合而成：

$$ \phi ::= \text{true} | \text{false} | p | \neg \phi | \phi \land \psi | \phi \lor \psi | AX\phi | EX\phi | AF\phi | EF\phi | AG\phi | EG\phi | A[\phi U \psi] | E[\phi U \psi] $$

**路径量词**：

- $A\phi$：所有路径都满足 $\phi$
- $E\phi$：存在路径满足 $\phi$

**时序运算符**：

- $X\phi$：下一步满足 $\phi$
- $F\phi$：最终满足 $\phi$
- $G\phi$：总是满足 $\phi$
- $\phi U \psi$：$\phi$ 直到 $\psi$ 成立

**组合运算符**：

- $AX\phi$：所有路径的下一步都满足 $\phi$
- $EX\phi$：存在路径的下一步满足 $\phi$
- $AF\phi$：所有路径最终满足 $\phi$
- $EF\phi$：存在路径最终满足 $\phi$
- $AG\phi$：所有路径总是满足 $\phi$
- $EG\phi$：存在路径总是满足 $\phi$
- $A[\phi U \psi]$：所有路径上 $\phi$ 直到 $\psi$ 成立
- $E[\phi U \psi]$：存在路径上 $\phi$ 直到 $\psi$ 成立

### 语义定义

CTL的语义基于**Kripke结构**：

$$ M = (S, S_0, R, L) $$

其中：

- $S$ 是状态集合
- $S_0 \subseteq S$ 是初始状态集合
- $R \subseteq S \times S$ 是转换关系
- $L: S \to 2^{AP}$ 是标记函数，$AP$ 是原子命题集合

对于Kripke结构 $M = (S, S_0, R, L)$ 和状态 $s \in S$：

- $M, s \models AX\phi$ 当且仅当对所有 $(s, s') \in R$，$M, s' \models \phi$
- $M, s \models EX\phi$ 当且仅当存在 $(s, s') \in R$，$M, s' \models \phi$
- $M, s \models AF\phi$ 当且仅当所有从 $s$ 开始的路径最终满足 $\phi$
- $M, s \models EF\phi$ 当且仅当存在从 $s$ 开始的路径最终满足 $\phi$
- $M, s \models AG\phi$ 当且仅当所有从 $s$ 开始的路径总是满足 $\phi$
- $M, s \models EG\phi$ 当且仅当存在从 $s$ 开始的路径总是满足 $\phi$

### CTL示例

```
安全性：AG(¬(critical1 ∧ critical2))
可达性：EF goal
死锁自由：AG(EX true)
活性：AG(start → AF finish)
公平性：AF AG enabled
```

## 路径量词和时序算子

### 对偶性

**LTL对偶性**：

- $\neg X\phi \equiv X\neg\phi$
- $\neg F\phi \equiv G\neg\phi$
- $\neg G\phi \equiv F\neg\phi$
- $\neg(\phi U \psi) \equiv (\neg\psi) W (\neg\phi \land \neg\psi)$

**CTL对偶性**：

- $AX\phi \equiv \neg EX\neg\phi$
- $AF\phi \equiv \neg EG\neg\phi$
- $AG\phi \equiv \neg EF\neg\phi$
- $A[\phi U \psi] \equiv \neg E[\neg\psi U (\neg\phi \land \neg\psi)] \land \neg EG\neg\psi$

### 时序算子关系

```
F φ ≡ true U φ
G φ ≡ ¬F¬φ ≡ φ W false
φ W ψ ≡ G φ ∨ (φ U ψ)
φ U ψ ≡ (φ W ψ) ∧ F ψ
```

## 安全性与活性

### 安全性（Safety Property）

**定义**：安全性性质表示"坏事永远不会发生"。

**特征**：

- 如果违反，可以在有限步骤内检测
- 形式上：$G \neg P$ 或 $AG \neg P$

**示例**：

- 互斥：两个进程不会同时进入临界区
- 数组越界检查：不会访问超出数组范围的索引
- 资源约束：不会分配超过可用数量的资源

### 活性（Liveness Property）

**定义**：活性性质表示"好事最终会发生"。

**特征**：

- 不能在有限步骤内验证或证伪
- 形式上：$F P$ 或 $AF P$

**示例**：

- 请求响应：如果发送请求，最终会收到响应
- 终止：程序最终会终止
- 公平性：所有进程最终都会获得资源

### 公平性（Fairness）

#### 无条件公平性（Unconditional Fairness）

$$ GF \text{ enabled} $$

无限经常启用某个转换。

#### 弱公平性（Weak Fairness / Justice）

$$ GF \text{ enabled} \implies GF \text{ executed} $$

如果一个转换持续被启用，它将无限经常被执行。

#### 强公平性（Strong Fairness / Compassion）

$$ FG \text{ enabled} \implies GF \text{ executed} $$

如果一个转换无限经常被启用，它将无限经常被执行。

## CTL与LTL的关系

### 表达能力比较

$$ \text{CTL} \subsetneq \text{CTL}^* \supsetneq \text{LTL} $$

$$ \text{CTL} \not\subseteq \text{LTL} \land \text{LTL} \not\subseteq \text{CTL} $$

**LTL可表达但CTL不可表达**：

- $A(F p \land F q)$：在所有路径上，p和q最终都成立

**CTL可表达但LTL不可表达**：

- $EF(AG p)$：存在一条路径，使得从该路径开始的所有后续路径上p总是成立

### 选择原则

| 场景 | 推荐逻辑 |
|------|----------|
| 路径性质验证 | LTL |
| 分支性质验证 | CTL |
| 复杂性质 | CTL* |
| 实时系统 | TCTL |
| 概率系统 | PCTL |

## 工具使用指南

### SPIN / Promela

**SPIN** 是支持LTL模型检验的工具。

#### 基本使用

```promela
// Promela 示例：互斥算法
#define N 2
bool critical[N];
int counter = 0;

active [N] proctype P() {
    do
    :: atomic {
        counter == 0 ->
        critical[_pid] = true;
        counter++;
    }
    // 临界区
    critical[_pid] = false;
    counter--;
    od
}
```

**LTL验证**：

```
ltl mutex { [](!(critical[0] && critical[1])) }
ltl progress { []<>(counter == 0) }
```

运行：

```bash
spin -a mutex.pml    # 生成验证器
gcc -o pan pan.c     # 编译
./pan -a             # 运行验证
```

### NuSMV

**NuSMV** 是支持CTL和LTL模型检验的工具。

#### 基本使用

```smv
MODULE main
VAR
    state: {ready, running, waiting};
    request: boolean;

ASSIGN
    init(state) := ready;
    next(state) := case
        state = ready & request: running;
        state = running: waiting;
        state = waiting: ready;
        TRUE: state;
    esac;

SPEC AG (state = running -> AF state = waiting)
SPEC EF state = running
```

运行：

```bash
NuSMV model.smv
```

## 总结

时序逻辑是形式化验证的核心工具，LTL和CTL各有优势，适用于不同的验证场景。安全性、活性和公平性是系统验证中最常见的性质类型。通过模型检验工具（如SPIN、NuSMV），可以自动验证系统是否满足给定的时序逻辑性质。

**关键要点**：

- LTL适合路径性质，CTL适合分支性质
- 安全性 = "坏事永远不会发生"
- 活性 = "好事最终会发生"
- 公平性确保系统不会忽视某些行为
